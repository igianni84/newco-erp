# Filament — Rules (apply by default)

> Promoted from `hypotheses.md` (3 dated confirmations) or derived directly from a canonical decision/arch-test. A contradiction demotes a rule back to `hypotheses.md`.

## An operator-console slice = a read-projection Resource + write-through pages driving domain Actions (never Eloquent), on the shared `OperatorConsole*` kit

**Rule.** An operator surface over a domain module is built from the shared kit in `app/Modules/OperatorPanel/Filament/Console/` — never an ad-hoc Filament Resource. The fixed recipe:

1. **Resource** `extends OperatorConsoleResource`, read-binds the module's Eloquent model (a pure read-projection). State/status columns + badges render through the model **cast** (`getStateUsing` + `instanceof \BackedEnum` → `->value`), **never** importing the module's `Enums` (operand-enum carve-out, ADR `2026-06-21`); any cross-module datum is a `{Models}` read, never a relation (invariant 10). `getPages()` = index/create/view; the table carries no edit/delete/bulk mutating action.
2. **Create page** `extends OperatorConsoleCreateRecord`: a form of within-module `Select`s whose `createViaAction()` routes to `app(CreateX::class)->handle(...)` and returns the model — **never** `$model->save()`. The domain's create rejection is surfaced on a named field via `createRejectionField()`.
3. **View page** `extends \Filament\Resources\Pages\ViewRecord` + `use SurfacesDomainActions` for the larger non-catalog FSMs (the fixed-5-verb `OperatorConsoleViewRecord` is the catalog submit/reject/activate/retire/reopen shape only — ADR `2026-06-20`). Each lifecycle transition is a **form-less** `lifecycleAction($verb, $successKey, $invoke)` header action, `->visible()`-gated as the **exact complement** of its domain Action's from-state guard, so the rejection branch is structurally unreachable through the UI. _Lone exception:_ a compound domain gate not expressible as a visibility predicate (e.g. renew-within-grace) stays UI-reachable and surfaces the danger notification via `surfaceLifecycleOutcome`.
4. Every write carries the `actor_role: newco_ops` envelope from `ActorContext` (the console constructs no envelope itself); **no Eloquent write under `App\Modules\OperatorPanel`** (`NoEloquentWriteInOperatorPanelRule`), **no cross-module import beyond `{Models, Actions}`** (`ModuleBoundariesTest`). Reference any `Illegal*Transition` / rejection exception in **prose/backticks only** — a `{@see FQCN}` / `@throws FQCN` makes Pint auto-add a boundary-violating `use` and reds `ModuleBoundariesTest` (lesson `2026-06-20`); catch base rejections by `\RuntimeException`.
5. **EN+IT i18n** for every user-facing string (no hardcoded copy), guarded by a per-slice i18n-completeness test reusing the suite-wide `scanOperatorConsoleHardcodedSinks` helper. Verify it via `--filter` or the full suite, **never** a bare single-file path — the shared helper is `function_exists`-guarded and otherwise loads undeclared → false red (lesson `2026-06-20`).
6. A **PG17 closing-chain** integration test drives the full FSM through the Filament pages (the `postgres:17` Docker recipe, GUIDE §2.7): SQLite-green is necessary, not sufficient (`knowledge/testing/rules.md`).

**Testing corollary.** A hidden (from-state-gated) action is undriveable through any Filament test helper — assert the domain rejection **directly** (`expect(fn () => app(X::class)->handle($id))->toThrow(...)`) **plus** `assertActionHidden(...)`; never assert an `action_failed` notification the widget can never send (lesson `2026-06-22`).

**Origin (dated).** 2026-06-20 `operator-console-catalog-spine` (tasks 1.1/1.2) — abstracted the `OperatorConsoleResource` / `OperatorConsoleCreateRecord` / `OperatorConsoleViewRecord` base classes + the `SurfacesDomainActions` trait out of the first Resource (`ProductMasterResource`, `operator-console-catalog-master`, same day) and retrofitted Master onto the kit.

**Confirmations (dated, cross-change).** The recipe then drove every subsequent operator surface unchanged: 2026-06-20 `operator-console-parties-producer` (Producer); 2026-06-21 `operator-console-parties-supply-side` (ProducerAgreement); 2026-06-22 `operator-console-parties-customer` (Customer status FSM + activation composite gate); 2026-06-22 `operator-console-parties-holds` (Hold place/lift + status coupling); 2026-06-23 `operator-console-parties-kyc-sanctions` (two independent compliance axes); 2026-06-24 `operator-console-parties-membership` (Profile 9-state FSM + orthogonal Account FSM). 12 Resources total (7 Catalog + 5 Parties); `ModuleBoundariesTest` + `NoEloquentWriteInOperatorPanelRuleTest` stayed green across all. Backing ADRs: `2026-06-19` read-binding/write-through, `2026-06-20` non-catalog lifecycle-trait reuse, `2026-06-21` operand-enum carve-out.

**Applies to.** Any new operator-console slice over a domain module. The Resource is always a read-projection; writes always route through a domain Action with the `newco_ops` envelope; copy is always EN+IT with a completeness guard; the FSM is always proven on PG17 through the pages. Sibling concerns: module-boundary law in `knowledge/architecture/`, cross-engine test portability in `knowledge/testing/`, Eloquent/runtime idioms in `knowledge/laravel/`.
