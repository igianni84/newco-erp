# Filament — Rules (apply by default)

> Promoted from `hypotheses.md` (3 dated confirmations) or derived directly from a canonical decision / arch-test / build finding. A contradiction demotes a rule back to `hypotheses.md`. Filament here = the **OperatorPanel** operator surface (module #9); the panel's read-bind + write-through architecture is decided in ADR `2026-06-19-operator-console-read-binding-write-through-actions.md`.

## An operator console reuses the `Filament/Console/` kit: read-only Resource + write-through Create/lifecycle Actions, cross-module surface fenced to `{Models, Actions(, Enums)}`

**Rule.** Every operator console over a shipped module is built from the shared kit under `app/Modules/OperatorPanel/Filament/Console/` — **never auto-discovered** (it lives outside `Filament/Resources`), so the concrete per-entity classes under `Filament/Resources/<Module>/` subclass it:
- **`OperatorConsoleResource`** (abstract) — the read projection: `table()`/`infolist()`/`getPages()`={index,view(,create)}, labels via `getModelLabel(): string` returning `(string) __()`, shared `versionColumn()`/`lifecycleStateColumn()`. Read-only = add nothing; state/enum columns render **through the Eloquent cast instance**, never by importing the module's `Enums`. `getPages()` eagerly references all registered page classes at registration, so a read surface must ship its Create + View pages **together as one green unit** or the panel fails to boot.
- **`OperatorConsoleCreateRecord`** (abstract) — `handleRecordCreation()` overridden to `app(<CreateAction>)->handle(...)`; narrow the `mixed` `$data` with `is_string`/`is_numeric` guards that throw; map a domain rejection to `ValidationException` on `data.<field>` (and a framework `UniqueConstraintViolationException` re-thrown as `ValidationException` for the create-collision case). Reach-the-create affordance is a plain **URL link**, never `CreateAction::make()` (which bypasses the domain action via an inline modal).
- **`OperatorConsoleViewRecord`** + the **`SurfacesDomainActions`** trait — lifecycle verbs as `Filament\Actions\Action`s in `getHeaderActions()`, each wired to `app(<TransitionAction>)`, `$record` injected by name, the outcome surfaced via a helper that catches the domain **base type** (`\RuntimeException`/`\Throwable`) → a Filament notification. The typing stays PHPStan-max clean with **no shared action interface**: per-entity typed invoke-closures + a `recordOf(class-string, Model)` narrowing helper.

Two hard constraints, both from the read-bind ADR: **(1)** the OperatorPanel's cross-module import surface is exactly `{Models, Actions}` — widened to `{Models, Actions, Enums}` by ADR `2026-06-21-operator-console-operand-enum-carveout` (operand enums an Action `handle()` param needs; **state enums stay rendered via the cast, never imported**). Enforced by `ModuleBoundariesTest`'s single-source `moduleBoundaryAllowedImports(Module)`; widening the list needs a new ADR. **(2)** Reference any cross-module `Exceptions\*`/`Enums\*` in a docblock only as **prose backticks**, never `{@see \FQCN}`/`@param FQCN` — Pint's `fully_qualified_strict_types` turns the type-ref into a `use` that breaches the boundary and reds the test (the `knowledge/laravel` Pint rule, escalated to a boundary breach here).

**Confirmations (dated, cross-change).** 2026-06-20 `operator-console-catalog-master` (origin — first console: read-only `ProductMasterResource`, write-through Create page + lifecycle Actions, the `{Models, Actions}` carve-out + the custom PHPStan write-ban rule); 2026-06-20 `operator-console-catalog-spine` (kit **extracted** to `Filament/Console/`, reused verbatim across Format/Variant/Case-Config/PR/Sellable/Composite consoles — pure reuse + a parent picker); 2026-06-20 `operator-console-parties-producer` (the **non-catalog** recipe — `extends OperatorConsoleResource` for labels but own `statusColumn()`/`kycStatusColumn()` via the cast; `SurfacesDomainActions` View page as the trait-reuse template, scaling to a second KYC FSM on the same page); 2026-06-21 `operator-console-parties-supply-side` (Club + ProducerAgreement consoles reuse the kit; the operand-enum carve-out lands).

**Applies to.** Every operator console still to come (A/D/S/B/C/E + the cross-module finance-ops / Logilize consoles). Pairs with the read-bind ADR (`$model->save()` is banned — a repo-local PHPStan rule asserts no Eloquent write inside `App\Modules\OperatorPanel\**`).

## A kit-based console's i18n completeness test = sink-anchored literal scan + kit-key enumeration via `Lang::has` + per-key EN fallback, run through `--filter` (never a bare path)

**Rule.** An operator console routes copy two ways, so its i18n "no hardcoded string / every key resolves" guard needs **both** halves:
- **Literal scan** — walk the console's PHP tokens and flag a bare user-facing string **only at a copy SINK method** (label/notification/heading builders), not every literal; prove non-vacuous with a **planted fixture** (a nowdoc with `<?php` at column 0). The suite-wide sink scanner (`scanOperatorConsoleHardcodedSinks`) is declared **once** and reused behind a `function_exists` guard — never redeclared (Pest loads every test file → a redeclare fatals the full suite).
- **Kit-key enumeration** — the kit builds ~17 keys by **string concatenation** (`i18nKey().'.actions.'.$verb`), so they never appear as literals a scan could catch; the only guard is to **enumerate the kit contract** (verbs + fixed suffixes + infolist-only keys like `fields.fee`) and assert `Lang::has(<key>, 'en', false)` for each, paired with a per-entity literal-key regex scan. Verify per-key **EN fallback** (DEC-127) is real via nested-dotted `Arr::get`, and pin `IT ⊆ EN`.

**Run it via `--filter=<TestName>` or the full suite, NEVER a bare file/directory path** — the shared sink scanner is declared in a sibling file, so a single-path pest run leaves it undeclared and the `function_exists` guard false-reds (a loading artifact, not a real failure; a directory-path PG17 gate command must **append the declaring file**). The five-guard test count is `|kit| + |differs| + 2 + 1 + 1`; the trailing dot in the `str_starts_with` key filter is load-bearing.

**Confirmations (dated, cross-change).** 2026-06-20 `operator-console-catalog-master` (`ProductMasterConsoleI18nTest` — origin sink-anchored scan + per-key EN fallback); 2026-06-20 `operator-console-catalog-spine` (`SpineConsoleI18nTest` — the kit-key `Lang::has` enumeration, the only way to catch a dropped concatenated key); 2026-06-20 `operator-console-parties-producer` (`ProducerConsoleI18nTest` — five guards off two dataset fns); 2026-06-21 `operator-console-parties-supply-side` (`ClubConsoleI18nTest` + `ProducerAgreementConsoleI18nTest` — twice-proven, `fields.fee` infolist-only key included).

**Applies to.** Every operator console's i18n capability-close task. Pairs with the `testing` Pest-global-namespace rule (the shared scanner is exactly such a declare-once helper) and invariant 12 (no hardcoded user-facing strings).
