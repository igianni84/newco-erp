## Open Patterns
- **Convenzioni F1 restano legge**: registry `Module::cases()`; red-proof per ogni regola arch NUOVA (i test di immutabilità 2.4 SONO red-path → non serve fixture separata); **verify-vendor-first** (provato di nuovo a 2.4: RAISE(ABORT) token nel messaggio PDO, `IS NOT` null-safe, before/after-only passa il WHEN, prepare+execute installa un CREATE TRIGGER con `;` interni); il runner emette **JSON** (no TTY Pest); per-file `uses(RefreshDatabase::class)`.
- **Raw DDL branch-ata = `DB::statement` (NON `DB::unprepared`)**: `unprepared()` è Larastan-typed `literal-string` → un'interpolazione (`{$values}`/`{$changed}` da implode/array_map) fa scattare `argument.type` a PHPStan max; `statement` non ha il vincolo e regge i bindings. Un comando per call, **nessun `;` finale** (evita il "multiple commands in a prepared statement" di PG; i `;` interni a `BEGIN…END`/`$$…$$` sono al sicuro). Idiom usato da TUTTE e 4 le migrazioni domain (CHECK actor_role solo-PG, partial index su entrambi gli engine, trigger su entrambi).
- **Trigger immutabilità cross-engine** (design D7): SQLite `CREATE TRIGGER … RAISE(ABORT,'…immutable…')` (single-event → due per tabella append-only; `RAISE(ABORT)` reverte solo lo statement → il test rilegge la riga invariata nella stessa wrapper-txn); PG funzione plpgsql `RAISE EXCEPTION '…immutable…'` + `BEFORE UPDATE OR DELETE … EXECUTE FUNCTION`. Immutabilità PARZIALE (solo alcune colonne mutabili, es. before/after) = UNA lista autorevole di colonne strutturali guida i due dialetti (`IS NOT` / `IS DISTINCT FROM`). Test SOLO comportamentali: `QueryException` + `toContain('immutable')` + riga invariata/esiste, MAI SQLSTATE → un file prova entrambe le lane. Tabella mutabile (event_deliveries) → guard "update riesce" così lo scope del trigger non si estende per errore.
- **Migrazione Postgres-truthful** (provata 4×): fluent types con fallback SQLite behavior-preserving (`jsonb→text`, `timestampTz→datetime text`, `uuid→varchar`); constraint solo-PG (CHECK) in branch `pgsql` via `DB::statement` DOPO `Schema::create`, valori da `Enum::cases()`; partial index raw `CREATE INDEX … WHERE` valido su entrambi; trigger immutabilità in branch ENTRAMBI implementati. **Assert schema portabili**: `Schema::hasColumns` (+ `->toBeFalse()` per pinnare l'ASSENZA) + `hasIndex($t,[col,...]|'name','unique'|null)` + prove comportamentali (`->throws(QueryException)` per NOT NULL/unique/FK, mai SQLSTATE). **PHPStan max nei test**: helper plain → `array<string,mixed>`; mai castare risultati query (`->value()` è `mixed` → asserisci con `->toEqual`/`->toContain`); `insertGetId` già `int` (PHPDoc).
