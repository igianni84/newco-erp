- **Registry is the single source of "the nine".** Every conformance/arch test AND the composition root iterate `Module::cases()` — never a hardcoded list. `->namespace()` FQCN root, `->value` table prefix, `->letter()` spec letter, `->providerClass()` standard provider FQCN. The ONE legitimate hardcoded list is `$platformNamespaces` (2.3) — the *complement* of the registry (non-module `App\*` roots), which `Module::cases()` cannot derive.
- **Three arch-test SHAPES now in `tests/Architecture/`:** (1) **conformance** (2.1) — `scandir` set-equality registry↔filesystem; (2) **import-boundary** (2.2/2.3) — pest-plugin-arch `not->toUse([deps])->ignoring([allowed])`, **arrays on the DEPENDENCY side, loops on the SOURCE side** (a source-side array under `not` is vacuously green — proven in 2.3); (3) **property-convention** (2.4) — hand-rolled reflection scan (walk `Module::cases()` dirs → FQCN → `class_exists`+`isSubclassOf`+`!isAbstract` → read `getDefaultProperties()`), proven-empty = assert the scan saw real files BEFORE asserting the law. Each arch task carries a mandatory **red-proof** (temp violating fixture → suite red → remove → green, both outputs in progress.md) — the only thing that catches a vacuously-green arch test.
- **The Architecture suite is boot-free, registry-located.** `tests/Architecture/` = own `<testsuite>` in `phpunit.xml` (→ default `php artisan test`; count delta is the proof) but NOT bound to `Tests\TestCase` in `tests/Pest.php`. Root via reflection on `Module::class`, never `app_path()`. Type-clean at level max: `(string)` on `getFileName(): string|false`, `?: []` on `scandir(): array|false`, `$item instanceof SplFileInfo` narrowing for iterator items. NEW `app/` fixtures need no `composer dump-autoload` (PSR-4 live); dump-autoload is only for MOVED/RENAMED `App\**` classes (stale classmap). Creating a sub-dir like `Catalog/Models/` does NOT trip the 2.1 conformance test (it scans only dirs DIRECTLY under `app/Modules/`).
