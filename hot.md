## Open Patterns
- **Command OUTPUT assertions a PHPStan max (5.1)**: `expect(Artisan::call('cmd'))->toBe(0)` poi `expect(Artisan::output())->toContain(...)` — il Pest `artisan()` (`@return PendingCommand|int`) è inusabile a max (sia `->expectsOutputToContain` che `->assertExitCode` colpiscono il ramo `int`). `Artisan::output()` = output bufferizzato dell'ultima `call()` (`Foundation\Console\Kernel::output()`).
- **Comando operator re-runnable su tabelle append-only (5.1, D9)**: ogni run APPENDE (UUIDv7 fresco per-evento → no collisione su `event_id` / `(domain_event_id,consumer)`); ogni STATE-write usa `DB::table(...)->updateOrInsert([key],[vals])` (upsert idempotente sul PK `cache`). Il consumer keya il marker per `event_id` e fa anch'esso upsert (modella l'obbligo di idempotenza). "2° run anche exit 0" = il failure/edge test. **Consumer demo-only registrati sul singleton `ConsumerRegistry` DENTRO `handle()`** (idempotente) — niente wiring demo nei provider always-on.
- **`DB::transaction(fn (): array => [...])` a PHPStan max (5.1)**: ritorna un array keyed e annota l'assegnazione `/** @var array{event: DomainEvent, audit: AuditRecord} $r */` (Larastan thread-a il return del closure ma l'`@var` rimuove ogni dubbio, sopravvive a Pint). Per UN valore solo tipizza il closure (`fn (): DomainEvent => …`) e Larastan inferisce diretto.
- **Post-commit hook scatta SINCRONO dentro il return di `DB::transaction()` a livello 0** (DatabaseMigrations / CLI reale): il comando legge una delivery `done` subito dopo il blocco transazione — niente polling, lo sweep non serve al demo (l'hook inline consegna).
- **Scheduling Laravel 13 + registrazione comando esplicita (4.2)**: schedule sub-minute in `routes/console.php` via `Schedule::command('name')->everyThirtySeconds()->withoutOverlapping()`; comandi-classe fuori da `app/Console/Commands` registrati via `withCommands([Cmd::class])` in `bootstrap/app.php`. Introspezione test: `app(Kernel::class)->bootstrap()` → `collect(app(Schedule::class)->events())->sole(fn (Event $e) => is_string($e->command) && str_contains($e->command,'name'))`.
