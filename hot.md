## Open Patterns
- **Provider-seam singleton** (3.2, ora in Codebase Patterns): classe con stato d'istanza, bind `$this->app->singleton(Foo::class)` in `AppServiceProvider::register()` (gira prima di ogni `boot()`); è il container singleton a far vedere UNA istanza a tutti i provider + al delivery path. `singleton($class)` senza closure è valido (ctor senza arg).
- **Unit non ha l'app Laravel**: `tests/Pest.php` lega `TestCase` solo `->in('Feature')` → `tests/Unit/**` è plain PHPUnit, niente `app()`/`$this->app`. Split: logica pura → Unit (new up diretto); qualsiasi risoluzione dal container (identità singleton, persistenza binding) → Feature test.
- **`is_subclass_of($x, Iface::class)` è un type-guard PHPStan**: dopo `if (! is_subclass_of($x, Foo::class)) { throw; }` il `string $x` si restringe a `class-string<Foo>` (storabile senza cast in `list<class-string<Foo>>`). Accetta nomi di classe anonima, ritorna false (no fatal) per classe mancante → UN guard copre "non implementa" + "non esiste". Tipa il param `string` (non `class-string<Foo>`) così un negative test può passare una classe non conforme senza errore statico.
- **Enum a caso singolo = gate compile-time**: niente runtime `if ($case !== Only) throw` (PHPStan max lo flagga always-false); pinna con `Enum::cases() === [Only]` + commento su cosa cambia all'aggiunta del caso. Param deferito come defaulted (`DeliveryMode $mode = Inline`); PHPStan non flagga un param di metodo inutilizzato.
- **Convenzioni F1 restano legge**: registry `Module::cases()`; **verify-vendor-first** (provato di nuovo a 3.2 su `is_subclass_of`/Pest-binding); il runner emette **JSON** (no TTY Pest); per-file `uses(RefreshDatabase::class)` per i Feature DB-touching (binding globale resta commentato); helper Pest GLOBALI tra file → prefissa. **Model platform** (3.1): `$table` esplicito, `$guarded=[]`, `casts()`; append-only → `$timestamps=false`; enum-cast = floor value-set (fuori-enum → `ValueError` non catturato = failure-case); `@property` espliciti per `timestamptz` (Larastan non lo inferisce). **Raw DDL branch-ata = `DB::statement`** (non `DB::unprepared`, vincolo `literal-string` di Larastan); trigger immutabilità rendono `domain_events`/`audit_records` insert-only (mai `update()`/`delete()`; solo `EventDelivery` round-trippa update a 4.1).
