Per the cut-sheet Q1 ("defer Direct Purchase *use* at launch; passive V1 + V2 only; the uniform flow + the retained `operator_initiated` / `ownership = NEWCO` enum values are the seam") + Phase C item I (confirmed — no launch deal; A/D/B/E/S idle in lockstep):

### §0.2 The `SupplierPaymentCompleted` posture (R1 + R4 — ⚠️ the trap)

The single highest-risk reconciliation. The v1.1 acceptance had **Module D *emitting* `SupplierPaymentCompleted`** as Module A's Direct-Purchase activation trigger. **Both halves are superseded:**

- **R1 (DEC-183) — financial-event-only, no FSM role.** Activation is **operator-publish post-PO-commit uniformly** (V1/V2/Direct Purchase). `SupplierPaymentCompleted` does **not** drive the Allocation FSM. **AC-D-J-3, AC-D-FSM-6, AC-D-XM-2, AC-D-BR-CrossModule-4** reconciled. *(Module A's side — that the event takes no FSM action — is verified in Module A acceptance AC-A-FSM-7 / AC-A-EVT-20 / AC-A-BR-CrossModule-4.)*
- **R4 (Phase C) — E-emits / D-consumes.** **Module E emits `SupplierPaymentCompleted`** on payment clearing (the payment executor — three-actor split DEC-119; symmetric with the customer-side `AirwallexChargeExecuted`). **Module D consumes it to settle/close the PO** (the PO `CLOSED` transition). **This corrects the v1.1 acceptance + the cut-sheet's D.24 "Module D emits."** **AC-D-EVT-10** moves from an emission criterion to a consumption criterion; **AC-D-FSM-3 / AC-D-EVT-16 / AC-D-BR-Separation-2 / AC-D-XM-13** drop `SupplierPaymentCompleted` from Module D's emitted set; **AC-D-MVP-5** verifies the E-emits/D-consumes contract. **Module D's *other* procurement financial events (`InboundEventCostFinalized`, `DiscrepancyResolutionRecorded`, `PurchaseOrderIssued`, `ConsignmentReceiptRecorded`, `ReverseInboundEventRecorded`, `POIssuedUnderNonActiveAgreement`) stay D-emitted.** *(Module E's emission side is verified in Module E acceptance; Module B's independent `ownership_flag` consume is verified in Module B acceptance.)*
