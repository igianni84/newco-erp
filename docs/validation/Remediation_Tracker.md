### F1 вЂ” DemoSeeder is SQLite-only (PG `TRUNCATE` rejects FK-referenced tables)  В·  рџџ§ В· рџ”ґ В· found 2026-07-01 (RM-07)
- **What:** `DemoSeeder::reset()` truncates FK-referenced tables one-by-one (masters, customers, and вЂ” new in RM-07 вЂ” `domain_events`/`audit_records`). PostgreSQL's `TRUNCATE` refuses a table referenced by a FK unless every referencer is truncated in the **same** statement or `CASCADE` is used; Laravel's PG `withoutForeignKeyConstraints` emits only `SET CONSTRAINTS ALL DEFERRED` (no effect on `TRUNCATE`). So `db:seed --class=DemoSeeder` would fail on Postgres (`domain_events` в†ђ `event_deliveries.domain_event_id` + self-FK `causation_id`).
- **Pre-existing, not a regression:** true of the existing truncations too вЂ” RM-07 only extended the sweep. Verified by reading Laravel's `PostgresGrammar::compileDisableForeignKeyConstraints` (= `SET CONSTRAINTS ALL DEFERRED`), not empirically on PG.
- **Why it's fine today:** dev + demo run on SQLite (`withoutForeignKeyConstraints` = `PRAGMA foreign_keys=OFF`, which allows it); suite green. No PG demo/staging env exists yet.
- **Disposition:** convert `reset()` to one `TRUNCATE вЂ¦ CASCADE` (or `delete()`s) **when a Postgres demo/staging env is stood up** вЂ” ties to the hosting/infra open-stack gate (`CLAUDE.md`). Small; promote to an RM row then.
