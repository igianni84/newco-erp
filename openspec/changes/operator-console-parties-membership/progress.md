- **ViewProfile lifecycle verb = `lifecycleAction($verb, $successKey, $invoke)->visible(fn (): bool => $this->stateIs('<from>'))`** appended to `getHeaderActions()` (groups 3–5 stack onto the same array). `$invoke = fn (Model $record, string $notes) => app(<Action>::class)->handle($this->recordOf(Profile::class, $record)->id)` (the `$notes` arg is the trait's uniform signature — unused for form-less verbs). The visibility predicate is the EXACT COMPLEMENT of the Action's from-state guard, factored into ONE parametric `private function stateIs(string $state): bool => $this->recordOf(Profile::class, $this->getRecord())->state->value === $state` — reads `state` through the `ProfileState` cast `->value` (non-nullable backed enum → plain string match), **no `Parties\Enums` import** (design D2; mirrors `ViewCustomer::kycPending()`). For a two-state gate (group-5 `cancel` = `active|lapsed`): `$this->stateIs('active') || $this->stateIs('lapsed')`. Reference any `Illegal*Transition` in **prose/backticks only** — a `{@see FQCN}`/`@throws` makes Pint add a boundary-breaching `Parties\Exceptions` use (lesson 2026-06-20; verified clean post-Pint here).
- **Gated-verb test = visibility-test + reject-floor split (the `CustomerKycSanctionsConsoleTest` shape, `DatabaseMigrations`).** (1) a parametrized visibility test over EVERY `ProfileState` (`assertActionVisible`/`assertActionHidden` per from-state); (2) a parametrized **reject-floor** test over the non-from states: `assertActionHidden('verb')` **+** `expect(fn () => app(<Action>::class)->handle($id))->toThrow(Illegal*Transition::class)` **+** state & `DomainEvent::count()` unchanged — a hidden verb is undriveable through `callAction` (asserts-visible-first), so prove the reject via the **domain throw**, never `assertNotified(action_failed)` (lesson 2026-06-22). Happy path: `callAction('verb')->assertNotified(__('…notifications.<successKey>'))` then assert state + event (`->sole()` for a singleton event; `actor_role` = `ActorRole::NewcoOps`, `actor_id` `->toEqual($operator->id)` loose for the PG bigint string). Approve's `OriginatingClubLocked` is a **Customer**-entity event (`entity_type` `'Customer'`), one-shot across a Customer's approvals.
  - **Multi-from-state corollary (group 4; reused by groups 5/6).** When ONE test file covers several verbs with DIFFERENT from-states (group 3's approve/decline both gate `applied` → one shared `bool`; group 4's activate/suspend/reactivate gate `approved`/`active`/`suspended`), drive BOTH sweeps off the full `ProfileState` list: the **visibility** test maps each state to its single `?string $visibleVerb` and loops the verbs asserting visible-iff-match; the **reject-floor** test builds a `$fromStateOf` map + an `$invokeOutOfBand` closure map, then `foreach … continue`s the verb whose legal-from === the case state and asserts `assertActionHidden` + `toThrow` on the rest, with one trailing state/`DomainEvent::count()`-unchanged assert. **PHPStan-max:** each closure MUST use a **literal** `app(ActivateProfile::class)` — never `app($actionClassVar)` (Larastan can't resolve `handle()` on a variable class-string → "call on mixed"). Suspend's state-preservation (AC-K-FSM-2a): seed `ClubCredit::factory()->create(['profile_id' => $p->id])` then assert the reloaded credit is still `ClubCreditState::Active` with `->remaining->equals($credit->remaining)` (Money value-equality) — suspension writes ONLY `Profile.state`.
    - **OVERLAPPING-from-state corollary (group 5; reused by group 6).** When verbs SHARE a from-state (group 5: `active` surfaces lapse+cancel+deactivate, `lapsed` surfaces renew+cancel), the single-verb maps don't fit — use SETS: the visibility test maps each state to an `array $visibleVerbs` and loops `['lapse','renew','cancel','deactivate']` asserting `assertActionVisible`/`assertActionHidden` by `in_array($verb, $visibleVerbs, true)`; the reject-floor maps each verb to a `list<ProfileState>` of its legal-froms (cancel = `[Active, Lapsed]`) and `continue`s when `in_array($from, $legalFroms, true)`. `array $visibleVerbs` as a positional dataset param is PHPStan-fine (NOT the string-keyed `...$spread` trap).
    - **`renew` past-grace = the SOLE UI-reachable reject (design D5).** `RenewProfile`'s gate is compound (`lapsed` AND within 30d of `lapsed_at`); the visibility predicate can only see `state == lapsed`, so a past-grace renew is VISIBLE → driven through the page → domain rejects → `action_failed` danger. Test it like the catalog `action_failed` shape (`callAction('renew')->assertNotified(…action_failed)` + state unchanged + event count 0), but make it past-grace by **anchoring `lapsed_at` in the past** (`CarbonImmutable::now()->subDays(31)`) — NOT `$this->travelTo()`, which fails PHPStan-max inside a Pest `it()` closure (`$this` binds to `Pest\PendingCalls\TestCall`, not the TestCase → `method.notFound`). Anchoring the domain time-field is the equivalent past-grace condition with no test-now mutation/cleanup. `cancel` is AUDIT-ONLY (no event) yet STILL throws `IllegalProfileTransition` out-of-band, so it belongs in the reject-floor `toThrow` set; its terminal soft-delete (AC-K-FSM-13) is asserted by `Profile::query()->whereKey($id)->exists()` (Profile has no `SoftDeletes` scope — the row simply persists with `state = cancelled`).
