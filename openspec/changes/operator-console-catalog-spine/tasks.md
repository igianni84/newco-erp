**Standing rules per task:** the five Quality Commands green at the end (format · test_filter · test · type_check · lint); `openspec validate operator-console-catalog-spine --strict` green; **no protected files touched** (`spec/**`, `openspec/specs/**`, `CLAUDE.md`, `RALPH.md`, `ralph.sh`, `.claude/**` — tests under `tests/` are NOT protected); **no new domain logic** — this change adds **zero** Catalog Actions/events/migrations and **no** Composer dependency (`git diff main -- composer.json composer.lock` stays empty; no `database/migrations/*` added).

**The one law (ADR 2026-06-19): the console READS module models and WRITES only through module domain actions — never `$model->save()`.** Resources bind **read-only** to `\App\Modules\Catalog\Models\*`. Every mutation is `app(<CatalogAction>::class)->handle(...)`. The shipped `NoEloquentWriteInOperatorPanelRule` (PHPStan, scoped to `Modules/OperatorPanel/Filament/`) and the `ModuleBoundariesTest` `{Models, Actions}` carve-out already cover the whole OperatorPanel namespace — **expect NO boundary amendment**; if one seems necessary, STOP and re-read the carve-out before widening it (design L9).

**Verify every Filament/framework API in `vendor/` before writing it** (lessons.md) — the Filament 5 `Resource` / `CreateRecord::handleRecordCreation(array $data): Model` / `ViewRecord::getHeaderActions()` / `Filament\Actions\Action` builder / `Filament\Forms\Components\*` (incl. the ordered-multi-input for Composite — repeater or `Select::multiple`) / `Filament\Notifications\Notification` / infolist + table builders, and the Livewire/Filament **test helpers** (`Livewire::test`, `callAction`, `assertHasFormErrors`, `assertActionExists`, `assertCanSeeTableRecords`). Authenticate with `actingAs(Operator::factory()->create(), 'operator')`; `Operator` = `\App\Modules\OperatorPanel\Models\Operator`.

**Exact shipped Catalog actions this console invokes (signatures verified in code — do not invent or rename; re-confirm in `app/Modules/Catalog/Actions/` before use). For EACH of the six entities `Submit<E>ForReview::handle(<E> $e): <E>`, `Reject<E>Review::handle(<E> $e, string $notes): <E>`, `Activate<E>::handle(<E> $e): <E>`, `Retire<E>::handle(<E> $e): <E>`, `Reopen<E>::handle(<E> $e): <E>` exist.** Create signatures (the per-entity divergence):
- `CreateFormat::handle(string $name, string $sizeLabel, int $volumeMl): Format`
- `CreateCaseConfiguration::handle(string $name, int $unitsPerCase, string $packagingType): CaseConfiguration` — **no breakability**
- `CreateProductVariant::handle(int $productMasterId, string $variantIdentifier, ?int $vintageYear = null, bool $nonVintage = false, ?TranslatableText $tastingNotes = null): ProductVariant`
- `CreateProductReference::handle(int $productVariantId, int $formatId): ProductReference` — **no app-layer dedup; duplicate `(variant,format)` throws `Illuminate\Database\UniqueConstraintViolationException`**
- `CreateSellableSku::handle(int $productReferenceId, int $caseConfigurationId, string $commercialName, ?string $marketingCopy = null): SellableSku`
- `CreateCompositeSku::handle(array $productReferenceIds): CompositeSku` — `list<int>` bundle order; **`< 2 distinct` throws `InsufficientCompositeConstituents` (RuntimeException, localized)**, producer-agnostic

**Domain-enforced, console-surfaced (never reimplemented — design L4):** the SoD floor (`ApprovalGovernance`), the within-catalog activation-cascade gate (Variant←Master; Reference←Variant+Format; Sellable←Reference+CaseConfig; Composite←every constituent), the illegal-from-state guard, and the retire reference-integrity block (Reference by active Sellable/Composite SKU; Case Configuration by active Sellable SKU) all surface by the wrapper catching the thrown `RuntimeException` and rendering a Filament **danger notification** (messages already localized in `lang/en/catalog.php`: `approval.*`, `gate.parent_not_active`, `lifecycle.*`, `retirement.blocked_by_active_references`). The console re-checks **none** of them. **None of the six binds a producer** — no producer picker, no Producer-gate handling.

**Event names (verbatim, product-catalog spec):** `*Created` on create; `*Activated` on `reviewed→active`; `*Retired` on `active→retired`; submit/reject/reopen record **no** event. Families: `ProductVariant*`, `Format*`, `ProductReference*`, `CaseConfiguration*`, `SellableSKU*`, `CompositeSKU*` (SKU upper-case). Every recorded write carries `actor_role: newco_ops` + the operator id via `ActorContext` — assert it.

**i18n (design L7):** per-entity copy in `lang/{en,it}/operator_console.php` (`operator_console.<entity>.{label,plural_label,columns,fields,actions,affordance,notifications}`, mirroring the Master shape); reuse `lang/en/catalog.php` for domain rejection copy; add the console-owned **Product Reference duplicate** key (no domain copy exists); EN baseline + per-key IT fallback (DEC-127); **no hardcoded user-facing string** in any Filament class.

**PG17 (knowledge/testing/rules.md):** every test that drives a DB-touching action is verified on **PostgreSQL 17** before the task is marked done — SQLite-green is necessary, never sufficient. Run the full suite as `php -d memory_limit=-1 vendor/bin/pest`, PHPStan as `php -d memory_limit=-1 vendor/bin/phpstan analyse`. Record the PG run in `progress.md` at close.

**Scope guard (proposal slice-boundary):** the six spine entities **only** — NO Master behaviour change beyond the retrofit, NO update/field-edit, NO bulk import, NO cascade-retire for the six, NO producer picker on the six, NO `DeleteAction`/`EditAction`, NO authority-tier RBAC, NO Parties/A–E/SPA surface.
