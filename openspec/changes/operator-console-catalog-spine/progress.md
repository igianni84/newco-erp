(consolidated reusable patterns ÔÇö read first each iteration)

- **The operator-console kit ÔÇö extracted in task 1.1; tasks 2.1ÔÇô5.2 REUSE it verbatim.** It lives under `app/Modules/OperatorPanel/Filament/Console/` ÔÇö deliberately NOT under `Filament/{Resources,Pages,Widgets}`, because Filament's `discoverResources/discoverPages/discoverWidgets` scan only those three dirs (1.1 of the predecessor repointed them there), so an **abstract** base Resource/Page placed in `Console/` is never auto-discovered (and never instantiated as a real page). Three pieces:
  - `Console/Concerns/SurfacesDomainActions` (**trait**): the operator-action wrapper. `surfaceLifecycleOutcome(Closure $run, string $successTitle)` (now an INSTANCE method ÔÇö it reads `$this->i18nKey()` for the `notifications.action_failed` danger title; renders success/danger `Notification`; catches `RuntimeException` by **base type** so nothing is imported from a module's `Exceptions` namespace ÔÇö the {Models, Actions} surface holds). `lifecycleAction(string $verb, string $successKey, Closure $invoke, ?array $form, ?string $confirmationKey): Action` ÔÇö the factory that builds one uniform lifecycle action: label = `operator_console.<i18nKey>.actions.`.`Str::snake($verb)` (so the camelCase action id `retireCascade` resolves the existing `retire_cascade` key), success notification = `ÔÇŽnotifications.<successKey>`, optional reject-notes form, optional `ÔÇŽ<confirmationKey>` confirmation affordance. `recordOf(class-string<TModel>, Model): TModel` ÔÇö narrows the page record to the concrete model. `abstract i18nKey(): string`.
  - `Console/OperatorConsoleViewRecord` (**abstract base View**, extends `Filament\Resources\Pages\ViewRecord`, `use SurfacesDomainActions`): builds the FIVE uniform header actions (submit ┬Ě reject[notes] ┬Ě activate[`affordance.second_actor`] ┬Ě retire ┬Ě reopen) from `abstract lifecycleInvocations(): array<string, Closure(Model, string): mixed>`.
  - A **per-entity View page** extends the base, sets `protected static string $resource`, returns its `operator_console.<entity>` root from `i18nKey()` and its 5 invocations from `lifecycleInvocations()`. Entity-specific extras override `getHeaderActions()` and **spread `parent::getHeaderActions()`** (e.g. Master appends only `retireCascade`). The producer picker / cascade are Master-only extensions, never in the base (design L6).
- **The typing pattern that keeps the kit PHPStan-`max` clean with NO shared action interface (verified):** the Catalog actions share no interface and each `handle()` takes its CONCRETE model (param e.g. `$master`), so a dynamic `app($classString)->handle($record)` cannot type-check (the action resolves to `object`/`mixed`). Instead the per-entity subclass supplies **typed invoke-closures**: `fn (Model $record, string $notes) => app(<ConcreteAction>::class)->handle($this->recordOf(<Model>::class, $record)[, $notes])`. `recordOf` narrows `Model`Ôćĺconcrete via `instanceof $class` on a `class-string<TModel>` (phpstan narrows the return to `TModel`); `app(<Literal>::class)` returns the concrete action (literal class-string Ôćĺ template-bound), so `->handle(<concrete>)` is clean. Give every invoke-closure the uniform `(Model $record, string $notes)` signature (`$notes` is `''` for the form-less verbs) ÔÇö a uniform `string $notes` avoids the `array<string,mixed>` annotation friction inline closures otherwise need; the factory extracts notes from Filament's `$data` once.
- **`Action::form()`'s component base is `Filament\Schemas\Components\Component`, NOT `Filament\Forms\Components\Component` (the latter does NOT exist in Filament 5).** A `@param array<int, Component>` for an action form MUST reference `Filament\Schemas\Components\Component` (`Textarea`Ôćĺ`Field`Ôćĺ`Schemas\Components\Component`). Pint's `fully_qualified_strict_types` turns a bare/`Forms\` `Component` docblock into an import of the **non-existent** `Filament\Forms\Components\Component` Ôćĺ phpstan `class.notFound`; import the Schemas one explicitly. (Ôćĺ lessons.md)
- **Filament action `data` injection is ALWAYS an array** ÔÇö `->action(fn (Model $record, array $data) => ÔÇŽ)` is safe for form-LESS actions too: `Action::resolveDefaultClosureDependencyForEvaluationByName('data')` returns `getData(): array` = `[]` when no form is mounted (verified in `vendor/filament/actions/src/Action.php`). So one uniform factory closure can always request `$data` and narrow `notes` from it (`is_string($data['notes'] ?? null) ? ÔÇŽ : ''`).
