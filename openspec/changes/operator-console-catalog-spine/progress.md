(consolidated reusable patterns — read first each iteration)

- **The operator-console kit — extracted in tasks 1.1 + 1.2; tasks 2.1–5.2 REUSE it verbatim.** It lives under `app/Modules/OperatorPanel/Filament/Console/` — deliberately NOT under `Filament/{Resources,Pages,Widgets}`, because Filament's `discoverResources/discoverPages/discoverWidgets` scan only those three dirs (1.1 of the predecessor repointed them there), so an **abstract** base Resource/Page placed in `Console/` is never auto-discovered (and never instantiated as a real page/resource). Five pieces:
  - `Console/Concerns/SurfacesDomainActions` (**trait**): the operator-action wrapper. `surfaceLifecycleOutcome(Closure $run, string $successTitle)` (now an INSTANCE method — it reads `$this->i18nKey()` for the `notifications.action_failed` danger title; renders success/danger `Notification`; catches `RuntimeException` by **base type** so nothing is imported from a module's `Exceptions` namespace — the {Models, Actions} surface holds). `lifecycleAction(string $verb, string $successKey, Closure $invoke, ?array $form, ?string $confirmationKey): Action` — the factory that builds one uniform lifecycle action: label = `operator_console.<i18nKey>.actions.`.`Str::snake($verb)` (so the camelCase action id `retireCascade` resolves the existing `retire_cascade` key), success notification = `…notifications.<successKey>`, optional reject-notes form, optional `…<confirmationKey>` confirmation affordance. `recordOf(class-string<TModel>, Model): TModel` — narrows the page record to the concrete model. `abstract i18nKey(): string` (INSTANCE — pages).
  - `Console/OperatorConsoleViewRecord` (**abstract base View**, extends `Filament\Resources\Pages\ViewRecord`, `use SurfacesDomainActions`): builds the FIVE uniform header actions (submit · reject[notes] · activate[`affordance.second_actor`] · retire · reopen) from `abstract lifecycleInvocations(): array<string, Closure(Model, string): mixed>`.
  - `Console/OperatorConsoleResource` (**abstract base Resource**, extends `Filament\Resources\Resource`) — the read-only conventions (task 1.2): `getModelLabel/getPluralModelLabel` resolve `operator_console.<entity>.{label,plural_label}` off `abstract protected static function i18nKey(): string` (STATIC here — labels/columns are static); `lifecycleStateColumn(): TextColumn` (the badge, rendering `$record->getAttribute('lifecycle_state')` via `instanceof BackedEnum ? (string) $state->value : ''` — type-clean at phpstan max, NO `Catalog\Enums` import) + `versionColumn(): TextColumn`. Adds NO mutating action. A subclass sets `$model`/`$recordTitleAttribute`, supplies `i18nKey()` + its own `form/table/infolist/getPages`, and spreads `static::lifecycleStateColumn(), static::versionColumn()` into its columns (interleave if a column sits between them, as Master's producer does). Master-only producer picker / cascade stay on the subclass (design L6).
  - `Console/OperatorConsoleCreateRecord` (**abstract base Create**, extends `Filament\Resources\Pages\CreateRecord`) — the write-through create template (task 1.2): `$canCreateAnother=false`; `handleRecordCreation()` FULLY overrides Filament's default `new Model($data); $record->save()` (which the no-Eloquent-write rule would fail) and delegates to `abstract createViaAction(array $data): Model` wrapped in the rejection catch — a localized domain `RuntimeException` → `data.<createRejectionField()>` form error (Master dedup, Composite `<2`). `abstract createRejectionField(): string`. **PR is the one special case:** its `createViaAction` catches the framework `Illuminate\Database\UniqueConstraintViolationException` itself and throws a `ValidationException` (NOT a RuntimeException → sails through the base catch) with a console-owned key, so the raw SQL never renders (design L5).
  - A **per-entity View page** extends the base, sets `protected static string $resource`, returns its `operator_console.<entity>` root from `i18nKey()` and its 5 invocations from `lifecycleInvocations()`. Entity-specific extras override `getHeaderActions()` and **spread `parent::getHeaderActions()`** (e.g. Master appends only `retireCascade`). A **per-entity Create page** supplies just `createViaAction` (narrow form state → `app(<CreateAction>)->handle(...)`; a malformed payload throws `InvalidArgumentException` = LogicException, which propagates past the base's RuntimeException catch — a 500, not a form error) + `createRejectionField`. The producer picker / cascade are Master-only extensions, never in the base (design L6).
- **The typing pattern that keeps the kit PHPStan-`max` clean with NO shared action interface (verified):** the Catalog actions share no interface and each `handle()` takes its CONCRETE model (param e.g. `$master`), so a dynamic `app($classString)->handle($record)` cannot type-check (the action resolves to `object`/`mixed`). Instead the per-entity subclass supplies **typed invoke-closures**: `fn (Model $record, string $notes) => app(<ConcreteAction>::class)->handle($this->recordOf(<Model>::class, $record)[, $notes])`. `recordOf` narrows `Model`→concrete via `instanceof $class` on a `class-string<TModel>` (phpstan narrows the return to `TModel`); `app(<Literal>::class)` returns the concrete action (literal class-string → template-bound), so `->handle(<concrete>)` is clean. Give every invoke-closure the uniform `(Model $record, string $notes)` signature (`$notes` is `''` for the form-less verbs) — a uniform `string $notes` avoids the `array<string,mixed>` annotation friction inline closures otherwise need; the factory extracts notes from Filament's `$data` once.
- **`Action::form()`'s component base is `Filament\Schemas\Components\Component`, NOT `Filament\Forms\Components\Component` (the latter does NOT exist in Filament 5).** A `@param array<int, Component>` for an action form MUST reference `Filament\Schemas\Components\Component` (`Textarea`→`Field`→`Schemas\Components\Component`). Pint's `fully_qualified_strict_types` turns a bare/`Forms\` `Component` docblock into an import of the **non-existent** `Filament\Forms\Components\Component` → phpstan `class.notFound`; import the Schemas one explicitly. (→ lessons.md)
- **Filament action `data` injection is ALWAYS an array** — `->action(fn (Model $record, array $data) => …)` is safe for form-LESS actions too: `Action::resolveDefaultClosureDependencyForEvaluationByName('data')` returns `getData(): array` = `[]` when no form is mounted (verified in `vendor/filament/actions/src/Action.php`). So one uniform factory closure can always request `$data` and narrow `notes` from it (`is_string($data['notes'] ?? null) ? … : ''`).
- **In the read-only `<Entity>Resource`, name the create-action in PROSE backticks, NEVER `{@see \App\Modules\Catalog\Actions\Create<Entity>}`.** Pint's `fully_qualified_strict_types` rewrites a FQ `@see` into a short `@see` + a real `use` import — pulling a Catalog *Action* into a Resource that uses it only in a docblock (an unused import; allowed by the {Models, Actions} carve-out so it's harmless to tests, but untidy and not auto-removed by Pint's preset). The Resource's only Catalog import should be the *Model*. The Create *page* legitimately imports + `@see`s the action (it calls it). Verified on 2.1 Format (the action import had to be hand-removed); applies to 2.2–4.1.
- **Reference-integrity retire-block test scaffold (2.2 Case Config; reused by PR 3.2).** The blocking active referencer is stood up **directly via the model factory** (`SellableSku::factory()->create(['case_configuration_id' => $cc->id, 'lifecycle_state' => Active])`), NOT through the `CreateSellableSku`+activate chain — the `RetirementReferenceIntegrityGate` reads only `lifecycle_state`, so a factory-active SKU is a legitimate fixture, and the factory bypasses the Create action so **no `SellableSKUCreated` muddies the event count** (proven shape: `tests/Feature/Modules/Catalog/RetirementCascadeTest.php`). The factory auto-builds the SKU's `product_reference_id` (irrelevant to a Case-Config retire gate, which only reads SKUs by `case_configuration_id`). `RetirementReferenceIntegrityViolation extends RuntimeException`, so the kit's `surfaceLifecycleOutcome` catches it by base type → the `…notifications.action_failed` danger title (body = the domain's `catalog.retirement.blocked_by_active_references`, already shipped — **no console-owned key needed**). Block test asserts: `assertNotified(action_failed)` + entity stays `active` + 0 `*Retired` + the SKU unaffected; unblock half flips the SKU to `Retired` (a closed reference does not block) → retire succeeds + 1 `*Retired`. For PR 3.2, the same shape with the SKU referencing the **PR** (`product_reference_id`) plus the Composite-SKU constituent-junction referencer (`CompositeSku::factory()->hasAttached($pr, ['position' => 1], 'constituents')`).
- **Spine-entity reuse confirmed PURE on 2.1 (Format).** Adds exactly: `FormatResource extends OperatorConsoleResource` (`i18nKey()='format'`, `$model`/`$recordTitleAttribute`, scalar create form, columns name/size_label/volume_ml + `static::lifecycleStateColumn()`/`versionColumn()`, infolist, `getPages()`); `ListFormats` (header create-link); `ViewFormat extends OperatorConsoleViewRecord` (`i18nKey()` + 5 typed invocations, NO `getHeaderActions()` override → exactly the kit's 5 actions, NO cascade); `CreateFormat extends OperatorConsoleCreateRecord` (`createRejectionField()='name'` safety-net + `createViaAction()` narrowing → `CreateFormatAction::handle`). Zero base/kit change, zero boundary amendment. The scope-guard "no cascade-retire for the six" is pinned by `assertActionDoesNotExist('retireCascade')` on the view page.
