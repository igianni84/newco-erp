# Progress — parties-enhanced-kyc-threshold

## Codebase Patterns
(consolidated reusable patterns — read first each iteration)

- **Run the FULL suite as `php -d memory_limit=2G vendor/bin/pest`.** `php artisan test` re-spawns Pest as a child process that does NOT inherit an outer `-d memory_limit` flag, and the ~1889-test suite exhausts the 128M CLI default during result collection (a `TestResult.php` fatal, not a test failure). Filtered runs (`php artisan test --filter=Name`) fit in 128M fine.
- **Read RAW DB scalars with the DB FACADE `DB::table('...')->value('col')` — NOT `Model::query()->...->value('col')`.** Eloquent's `->value()` hydrates a model and RE-APPLIES casts, so on a cast column it hands back the *enum/Carbon instance*, not the raw scalar (`->toBe('enhanced_kyc_threshold')` then fails: enum ≠ string). The DB facade bypasses casts → the raw stored value (the enum backing string; a PG bigint as a numeric string). Also prefer `->value('col')` over `->first()->prop`: PHPStan (max) flags `DB::table(...)->first()->reason` as property access on `stdClass|null`; `->value('col')` returns the scalar directly (the `ClubCreditSchemaTest`/`ComplianceReviewModelTest` idiom) — no nullable-object access, no suppression. Assert money/bigint columns with `->toEqual` (PG returns bigint as a numeric string), never `->toBe`. A closure `$row = fn () => DB::table('t')->where('id', $id)` gives a fresh builder per read when asserting several columns.
- **Value-set CHECK shape follows nullability.** A NOT-NULL enum-backed column → plain PG-only `CHECK (col IN (...))`; an additive-nullable column → `CHECK (col IS NULL OR col IN (...))` (the `2026_06_17_000001_add_compliance_to_parties` variant). Both derive the accepted set from `Enum::cases()` (guarded by `DB::getDriverName() === 'pgsql'`) so the constraint can never drift; on SQLite the enum cast is the floor.
- **Console i18n = console-chrome vs domain-value split, and `operator_console.php` is `array_replace_recursive([main-blocks], [sections-overlay])`.** Console CHROME (column headers, action/notification labels, and per-feature groups like `holds`, `compliance_reviews`) lives in the MAIN `customer` block of `lang/{en,it}/operator_console.php`; its `sections.*` HEADING keys live in a SECOND `customer` block that the file's `array_replace_recursive` merges in — so a feature group goes in the main block while its section heading goes in the overlay block (both land under `customer`). Guard console-chrome keys by front-loading their suffixes into `customerConsoleKitKeys()` in `CustomerConsoleI18nTest` (gives EN-complete + IT-distinct + IT⊆EN for free, ahead of the resolving surface — the holds/kyc-sanctions precedent); `sections.*` headings are conventionally left out of that list (verified by the view render). Enum VALUE labels rendered on a READ surface (a review's `reason`/`threshold_kind`) are DOMAIN vocabulary → they live in the module domain-copy file (`lang/en/parties.php`, `parties.compliance_review.*`), EN-only (there is no `lang/it/parties.php`; per-key EN fallback, DEC-127), guarded by a dedicated enum-iterating test. Enums carry NO `label()` method (SanctionsStatus/HoldType precedent); a Filament Select renders operands value→value (no per-value i18n — "domain data, not UI chrome"), but a read-only DISPLAY of an enum value maps `->value` through the domain-label keys.
- **Narrow `trans()`'s `string|array` return per-group, not per-element.** `trans('a.b.group', [], 'en')` is `string|array`; `assert(is_array($group))` narrows the whole variable so `array_keys($group)` is clean (PHPStan max). Accessing `$group['sub']` afterwards is still `mixed` → resolve each sub-group with its OWN `trans('a.b.group.sub', [], 'en')` + assert (the `CustomerConsoleI18nTest` idiom), never `array_keys($group['sub'])`.
- **Compare two `Money` with `≥` via `$a->minus($b)->minorUnits >= 0` — `Money` has NO comparison operator** (only `plus`/`minus`/`negate`/`equals`/`toPayload`). `minus()` throws `InvalidArgumentException` on a currency mismatch, so this `≥` is **fail-closed on currency** (a cross-currency figure throws rather than comparing raw minor units — invariant 6), and stays integer-minor-unit-only (no floats). Inclusive `≥` = `>= 0`; strict `>` = `> 0`. Hold thresholds as `int` minor-unit `const`s (a `Money` object is not a compile-time constant) and build `Money::of(self::X_MINOR, Currency::EUR)` at the comparison.
- **A scheduled module console command = 3 wiring points + a 5-part test recipe.** Command class lives in the module's `Console/` (e.g. `app/Modules/Parties/Console/ScanEnhancedKycThresholds.php`), NOT `app/Console/Commands` — so Laravel auto-discovery never sees it → register it explicitly in `bootstrap/app.php` `->withCommands([...])` AND schedule it in `routes/console.php` (`Schedule::command('sig')->daily()`; the `events:sweep` precedent). Iterate a table with `foreach (Model::query()->lazyById() as $row)` — id-cursor pagination is immune to the offset drift a plain `chunk()` suffers when the loop MUTATES rows (a scan flips a flag), and it types cleanly (`LazyCollection<int, Model>` → each `$row` is the model, `$row->id` is `int` via the model's `@property int $id`). Console/scheduler tick actor = `System` **for free** (no operator authenticated → `ActorContext` natural resolution; there is ZERO `runAs()` in app code — do NOT add one). Test under `DatabaseMigrations` (the command runs through `Artisan::call()` against the live connection at tx-level 0, so a per-row `DB::transaction` really commits — the `SweepTest` precedent): (1) behaviour via `expect(Artisan::call('sig'))->toBe(0)` + state asserts; (2) registration `expect(Artisan::all())->toHaveKey('sig')`; (3) schedule `app(Kernel::class)->bootstrap(); collect(app(Schedule::class)->events())->sole(fn (Event $e) => is_string($e->command) && str_contains($e->command, 'sig'))` then `expect($entry->expression)->toBe('0 0 * * *')` (`->daily()` == midnight). A per-caller fake collaborator (keyed on the loop id) proves the scan escalates ONE row and leaves siblings clean — the property a single-subject action test can't express.
- **Force a mid-transaction throw WITHOUT Mockery (the repo uses zero Mockery) — bind a throwing anonymous subclass of the concrete collaborator.** `app()->bind(ConcreteAction::class, fn () => new class(app(Dep1::class), app(Dep2::class)) extends ConcreteAction { public function handle(...): X { throw new RuntimeException('probe'); } })`. The overridden `handle` signature must mirror the parent EXACTLY (larastan max). Bind it, call the orchestrator, `expect(fn () => ...)->toThrow(...)`, then assert nothing persisted → proves the outer `DB::transaction` rolled back every prior write. Pairs with `DatabaseMigrations` (real outermost tx) for an invariant-grade atomicity proof; under `RefreshDatabase` the same works via savepoint rollback but is less faithful to the nested-tx shape.
