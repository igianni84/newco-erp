  - Test hint: feature — call the action; assert one row with `reason=enhanced_kyc_threshold`, the `threshold_kind`, `tripped_amount_minor` via `->toEqual`; `DomainEvent::count()` unchanged.

- [ ] 4.2 **`EvaluateEnhancedKycThreshold` action** (design D1/D2/D8; spec — *Enhanced-KYC Threshold Detection*) — `handle(int $customerId)`: in one `DB::transaction`, `lockForUpdate()` re-read the Customer; **no-op if `enhanced_kyc_flag` already set**; else read totals via the injected `CustomerTransactionTotalsReader`; breach = `largestSingleTransaction ≥ Money::of(1_000_000,'EUR')` **OR** `trailingTwelveMonthCumulative ≥ Money::of(5_000_000,'EUR')` (inclusive; named constants). On breach: (a) set `enhanced_kyc_flag=true` + `enhanced_kyc_at=now`; (b) `CreateComplianceReview` with the tripping `threshold_kind` (single wins if both) + amount; (c) record `CustomerEnhancedKycReviewRequired` via `DomainEventRecorder` (actor from `ActorContext`); (d) call `RecordCustomerScreening($id, SanctionsStatus::UnderReview, ScreeningTriggerSource::AmlThreshold)` (reused unchanged — nested tx/savepoint; no completion event). Non-`Create*` → **add to the exact-set whitelist in `tests/Feature/Modules/Parties/ComplianceIndependenceTest.php`**.
  - Acceptance: first breach performs all four writes atomically; second call (still above threshold) is a pure no-op; a sub-threshold Customer does nothing; `sanctions_status` becomes `under_review` + `trigger_source=aml_threshold`; **no** `CustomerRescreening*` event fires at initiation.
  - Test hint: feature (`DatabaseMigrations`) with a fake totals reader — single-tx breach and cumulative breach each assert flag+`enhanced_kyc_at`, one review row (right `threshold_kind`), one `CustomerEnhancedKycReviewRequired`, `sanctions_status=under_review`/`trigger_source=aml_threshold`; assert the recorded event-set excludes `CustomerRescreening*`; re-run → counts unchanged (idempotent); force a mid-tx throw → nothing persists (atomic); update the whitelist test so the exact-set assertion stays green.
