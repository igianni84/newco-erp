# Progress — parties-core

## Codebase Patterns
(consolidated reusable patterns — read first each iteration; the `catalog-product-spine` archive is the worked precedent for every idiom below)

- **Spine DB-entity creation template** (catalog task 2.1 → repeated here per entity). Each evented entity = `parties_*` migration + `Models\X` + `Database\Factories\Parties\XFactory` + `Events\XCreated` + `Actions\CreateX`.
  - **Migration:** `$table->id()`; entity cols; each state column `string` + `->default(Enum::Birth->value)`; `version` `unsignedInteger()->default(1)`; `timestampsTz()`. After `Schema::create`, a **driver-guarded** `CHECK` per enum column (`if (DB::getDriverName()==='pgsql')`, values from `Enum::cases()`, constraint `parties_X_<col>_check`) — verbatim the `domain_events.actor_role` / `catalog_*` idiom. SQLite skips the CHECK (cast + default carry the floor).
  - **Model:** `$table='parties_X'`; `$guarded=[]` (the action is the sole writer); `casts()` maps every enum/`Money`/`TranslatableText`/int column; full `@property` block; **typed `newFactory(): XFactory` override** (factories live under `Database\Factories\Parties\`, off the `App\Models` convention — the explicit return type fixes Larastan inference).
  - **Factory:** `protected $model = X::class`; bypasses the action (NO dedup, NO event) — pure fixture for standing up prerequisites cheaply.
  - **Event:** `final` class, untyped `const NAME` / `const ENTITY_TYPE`, static `payload(X): array` (PII-free). One class per `*Created` event under `Events/` (the module's public surface).
  - **Action:** `__construct(DomainEventRecorder $recorder, ActorContext $actor)`; pre-checks (uniqueness/required-ref) before/inside the tx; `DB::transaction(fn () => …)` inserts then `$this->recorder->record(name: XCreated::NAME, module: Module::Parties->value, actorRole: $this->actor->role(), actorId: $this->actor->actorId(), entityType: XCreated::ENTITY_TYPE, entityId: (string) $x->id, payload: XCreated::payload($x))`. The recorder's `NotInTransactionException` guard keeps write+emit atomic.
- **Two deliberate event asymmetries** (design D7) — `CreateSupplier` and the Account leg of `CreateCustomer` record **NO** event (the PRD §15 names none). Do not invent `SupplierCreated`/`AccountCreated`. The integration test asserts `entity_type IN ('Supplier','Account')` count is 0. **Worked at 2.2:** an event-silent action needs **no** `DomainEventRecorder`/`ActorContext` constructor deps (drop them — keep only `DB::transaction`).
- **Minimal entity** (Supplier, 2.2 — the worked example) — an entity the PRD gives no event *and* no lifecycle DROPS three Spine legs: `version`, the `status` column, and the `*Created` event. Migration is `id` + identity cols + `party_type` + `timestampsTz` only; model has a single `party_type` cast and an `@property` block without version/status. (Account, 2.x, is event-silent but NOT minimal — it keeps `status`/`version`.)
- **Party-type marker column idiom** — the `party_type` column takes a driver-guarded `CHECK` from the full `PartyType::cases()` (all three markers) **and NO `->default()`**: a marker is fixed explicitly by the action at creation, unlike a `status`/`state` column which gets a `->default(Enum::Birth->value)`. Same migration shape at Supplier (2.2, fixes `supplier`) and Customer (4.1, fixes `customer`). Proven on PG: the CHECK rejects an out-of-domain value, accepts the in-domain marker.
- **PII-free `CustomerCreated`** (design D7) — the Customer holds email/name/phone/DOB; the payload carries `customer_id`, `party_type`, `status`, `preferred_currency`, `preferred_locale`, `originating_club_id` (null) and **omits** all PII. Test: `expect($event->payload)->not->toHaveKey('email')` (and `name`/`phone`/`date_of_birth`).
- **All references are within Module K** — within-module FKs and Eloquent relations are allowed; **no** cross-module reference in this slice. `ModuleBoundariesTest` + `ModulePersistenceConventionsTest` (`$table = 'parties_*'`) stay green unamended. **FK idiom worked at Club (3.1) + the two-FK case at ProducerAgreement (3.2):** `$table->foreignId('x_id')->constrained(table:'parties_*', indexName:'<short_fk>')` — default RESTRICT (correct for a *referenced* parent like Producer/Club; `->cascadeOnDelete()` only for an *owned* child, none in this spine); short FK/index names (`parties_pa_*`) for PG's 63-char limit (`parties_producer_agreements` is the longest table — names fit but the short-explicit rule is followed). A **required + an optional FK coexist**: the optional one is `->nullable()->constrained(...)`. Within-module `belongsTo` (`$this->belongsTo(Producer::class,'producer_id')`, `@return BelongsTo<Producer,$this>`) is allowed + arch-green; an **optional** belongsTo uses the SAME `@return BelongsTo<Club,$this>` annotation (nullability lives in the column / `@property Club|null`).
- **Money & i18n** — Club `fee` is the only money field (worked at 3.1; Club is the FIRST production `MoneyCast` user): columns `integer('fee_minor')->nullable()` + `string('fee_currency',3)->nullable()` (verbatim the `MoneyCastTest` shape — `integer`, signed, never a float). Event payload via `$x->fee?->toPayload()` → `{minor_units, currency}` or null. **Test traps (PHPStan max + jsonb):** read raw money columns with `DB::table()->value('fee_minor')` (NOT `->first()->fee_minor` — that's `stdClass|null`); assert a nested payload money key as a whole via `expect($payload['fee'])->toEqual(Money::of(...)->toPayload())` (order-insensitive → survives PG jsonb reorder, AND avoids offset-on-`mixed`). Producer `description` is `TranslatableText` (one json column, English fallback; assert through the cast, never byte-compare jsonb). **Localized rejections** live in `lang/en/parties.php` (group file, dotted keys, `:placeholder`s; modelled on `lang/en/catalog.php`) — created at 3.1 for `MissingClubProducer`; reused for `DuplicateCustomerEmail` (4.1) / `DuplicateProfileForClub` (5.1). **`{@see}`-import trap (2.2/3.1):** a migration that only *creates* money columns must reference `Money`/`MoneyCast` in plain prose, never `{@see}` (Pint imports the FQN; `no_unused_imports` keeps it if the FQN is in prose too).
- **Date columns** (first at ProducerAgreement 3.2 `term_start`/`term_end`; reused by Customer `date_of_birth` 4.1) — `$table->date('x')->nullable()` + cast **`'immutable_date'`** (the parallel of the substrate's `immutable_datetime`), `@property CarbonImmutable|null`. Action accepts typed `?CarbonInterface` (tests pass `CarbonImmutable::parse('Y-m-d')`). Cross-engine: a PG `date` column returns plain `Y-m-d` (NO `+00` suffix — date, not timestamptz, so trap 4 does not apply); assert through the cast (`?->toDateString()`), never the raw column; event payload serialises dates as ISO `Y-m-d` strings. **Exact-key-set payload assertion** (`expect(array_keys($event->payload))->toEqualCanonicalizing([...])`) pins the PII-free contract against silent widening — reuse for the strict `CustomerCreated` PII guard.
- **Profile partial-unique** (design D8) — `(customer_id, club_id) WHERE state NOT IN ('rejected','cancelled','inactive')` via raw `DB::statement` (portable PG17 + SQLite; verify both) + an app pre-check throwing localized `DuplicateProfileForClub`. Inert in the spine (all Profiles born `applied`) but correct for the deferred lifecycle.
- **Enum-test convention** — `tests/Unit/Modules/Parties/Enums/EnumsTest.php`, no `RefreshDatabase`; per enum map `cases()` → `name=>value` and assert verbatim + order-sensitive; `->toHaveCount(n)` where "exactly N cases" is a spec rule (`AccountType` length 1; `ProfileState` length 9).
- **PG17 gate** — every DB-touching task verified on local PostgreSQL 17 before done (`knowledge/testing/rules.md`); record the run here at task 6.2.
