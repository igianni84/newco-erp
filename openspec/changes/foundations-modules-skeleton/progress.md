# Progress тАФ foundations-modules-skeleton

## Codebase Patterns
(consolidated reusable patterns тАФ read first each iteration)

- **Module registry is the single source for "the nine modules".** `App\Modules\Module` (string-backed enum, `app/Modules/Module.php`): case name = namespace segment (`namespace()` тЖТ `App\Modules\{CaseName}`), backing value = snake_case table prefix (`->value`, e.g. `catalog` тЖТ table prefix `catalog_`, design D6), `letter()` = spec letter (`0,K,A,D,S,B,C,E,Admin`). Every architecture/conformance test MUST iterate `Module::cases()` тАФ never a hardcoded module list тАФ so module-set drift fails loudly.
- **Unit tests need no Laravel boot.** `tests/Pest.php` binds `Tests\TestCase` only to `Feature`; `tests/Unit/**` runs on plain PHPUnit `TestCase`. Pure-PHP tests (enums, value objects) belong in `tests/Unit/`. Pest `it()/expect()/->and()` globals work in both suites regardless of the binding.
- **Strong verbatim map assertion.** `expect($builtMap)->toBe([...])` uses strict `===` (order- + type-sensitive): asserts completeness, values, AND declaration order in one shot. Build the map by iterating `Module::cases()` keyed by `->name`, compare against the literal in spec order.
- **Quality loop is green-clean today:** `vendor/bin/pint <files>` тЖТ `php artisan test <path>` тЖТ `php artisan test` тЖТ `vendor/bin/phpstan analyse` (level max, 0 baseline) тЖТ `vendor/bin/pint --test`. App files follow Laravel idiom: no `declare(strict_types=1)`, Pint laravel preset (no spaces around `.` concatenation).
- **Providers are DERIVED from the registry, not listed.** `bootstrap/providers.php` spreads `array_map(fn (Module $m) => $m->providerClass(), Module::cases())` after the two platform providers тАФ so adding/removing a module never edits the composition root by hand. `Module::providerClass()` owns the wiring-seam FQCN convention (`{namespace}\Providers\{Name}ServiceProvider`); both the composition root and the conformance test consume it (single source of truth). The composition root is boundary-exempt by design (D3) тАФ it references module provider FQCNs on purpose. Each module's standard provider `extends Illuminate\Support\ServiceProvider` (NOT Filament's `PanelProvider` тАФ that's the separate `AdminPanelProvider`, design D5) with empty `register()/boot()` = the seam F2+ fills.
- **Assert boot-time wiring via `app()->getLoadedProviders()`** тАФ an FQCN-keyed array (`[Provider::class => true]`), Laravel API verified at `Application.php:1477` (`providerIsLoaded(string)` at 1488 is the single-key variant). Non-deferred providers (no `DeferrableProvider`) are eagerly registered, so empty seam providers appear there. Use `expect($loaded)->toHaveKeys($all)` for completeness and `->not->toHaveKey($fabricated)` for the negative (Pest `Mixins/Expectation.php:628/661`; `not->toHaveKey` via `OppositeExpectation::__call`). A non-existent provider FQCN in `bootstrap/providers.php` fails LOUD: the app can't boot тЖТ the whole Feature suite errors.
- **Relocating/renaming any `App\**` class тЖТ run `composer dump-autoload` in the same iteration.** The `App\` PSR-4 map resolves the *new* path at runtime for free, but Composer's non-authoritative classmap (`vendor/composer/autoload_classmap.php`) keeps a STALE entry pointing the OLD FQCN at the deleted file. `findFile()` checks the classmap before PSR-4 and returns the stale path without an existence check тЖТ `class_exists('Old\FQCN')` tries to `include` a missing file тЖТ PHP warning тЖТ PHPUnit/Pest escalates it to a test failure. `composer dump-autoload` regenerates the classmap (drops the stale entry); it touches only `vendor/composer/autoload_*.php` тАФ never `composer.json`/`composer.lock`, and `vendor/` is gitignored, so zero commit/dependency churn. Verify with `php -r "require 'vendor/autoload.php'; var_dump(class_exists('Old\FQCN'));"` тЖТ must be `false` with no warning. Filament panel introspection for tests: `Filament\Facades\Filament::getPanel('admin')->getId()` (`FilamentManager.php:372` тЖТ `PanelRegistry::get($id, $isStrict=true)`; `Panel::getId()` via `HasId`).
- **The Architecture suite is boot-free, registry-located, and asserted by set-equality.** `tests/Architecture/` is its own `<testsuite>` in `phpunit.xml` so `php artisan test` runs it by default (the test-count delta is the proof) тАФ but it is NOT bound to `Tests\TestCase` in `tests/Pest.php`: arch/conformance/convention tests need no Laravel container. Adding a phpunit `<testsuite>` is sufficient for default discovery тАФ Pest finds test files from the phpunit testsuite directories; the `->in()` binding only chooses the base TestCase class; `--filter` works across all suites unaffected. Locate the modules root with `dirname((string) (new ReflectionClass(Module::class))->getFileName())`, never `app_path('Modules')`: reflection is boot-free, drift-proof (it follows wherever the registry file lives), and immune to a prior Feature test having flushed the container. Type-clean at PHPStan level max (which analyzes `tests/`) requires guarding the false-unions: `(string)` on `getFileName(): string|false`, `?: []` on `scandir(): array|false`. Assert "exactly these, nothing more/less" by sorting both the scanned entries and the `Module::cases()`-derived expected list, then `toBe` (strict `===`) тАФ symmetric set-equality: a missing AND an extra entry both fail. Filter dot-entries (`! str_starts_with($entry, '.')`) so the `.`/`..` pseudo-entries and gitignored OS artifacts (`.DS_Store`) never cause a false red (D1 forbids `.gitkeep`, so no legitimate dot-entry exists). Each arch-test task (2.1тАУ2.4) carries its own **red-proof** (temp violating fixture тЖТ suite red тЖТ remove тЖТ green, both outputs recorded here).
- **Import-boundary arch tests use `expect($ns)->not->toUse([$forbidden])->ignoring([$allowed])` (pest-plugin-arch v4.0.2, verified in `vendor/pestphp/pest-plugin-arch/src/`).** Confirmed semantics (read from source + empirically probed, design D4 тАФ never from memory): (1) **`not->toUse([list])` fails if the target uses ANY namespace in the list** (NOT all) and the `ExpectationFailedException` message **names the exact `(source, target)` pair** ("Expecting 'App\Modules\Catalog' not to use 'App\Modules\Parties'.") тАФ so one expectation per source module localises the offender without per-pair loops. (2) **`->ignoring([list])` strips uses by namespace PREFIX** (`LayerFactory::make` тЖТ `str_starts_with($use, $exclude)`), so ignoring `App\Modules\Parties\Contracts` allows references to anything under it while a reference elsewhere in `App\Modules\Parties` still fails тАФ this is exactly how the Contracts/Events public-surface allow-list is expressed. (3) **Layer membership is also prefix-based** (`leaveByNameStart`); the nine module names are mutually non-prefixing, so `toUse(['App\Modules\Finance'])` never accidentally catches another module. Build BOTH the forbidden list and the ignore list from `Module::cases()` (filter out the source) тАФ never hardcode. Arch expectations are lazily verified on `__destruct`/next-interaction inside the `it()` closure, so a bare `expect(...)` chain in a `foreach` is evaluated per iteration; they register real assertions (no risky/no-assertion flag). `tests/Architecture/ModuleBoundariesTest.php` is the reference (task 2.2); 2.3 extends the SAME file with a platform-direction block (`App\Providers|Models|Http` must not use `App\Modules`).
