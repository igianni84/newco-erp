# Progress — foundations-modules-skeleton

## Codebase Patterns
(consolidated reusable patterns — read first each iteration)

- **Module registry is the single source for "the nine modules".** `App\Modules\Module` (string-backed enum, `app/Modules/Module.php`): case name = namespace segment (`namespace()` → `App\Modules\{CaseName}`), backing value = snake_case table prefix (`->value`, e.g. `catalog` → table prefix `catalog_`, design D6), `letter()` = spec letter (`0,K,A,D,S,B,C,E,Admin`). Every architecture/conformance test MUST iterate `Module::cases()` — never a hardcoded module list — so module-set drift fails loudly.
- **Unit tests need no Laravel boot.** `tests/Pest.php` binds `Tests\TestCase` only to `Feature`; `tests/Unit/**` runs on plain PHPUnit `TestCase`. Pure-PHP tests (enums, value objects) belong in `tests/Unit/`. Pest `it()/expect()/->and()` globals work in both suites regardless of the binding.
- **Strong verbatim map assertion.** `expect($builtMap)->toBe([...])` uses strict `===` (order- + type-sensitive): asserts completeness, values, AND declaration order in one shot. Build the map by iterating `Module::cases()` keyed by `->name`, compare against the literal in spec order.
- **Quality loop is green-clean today:** `vendor/bin/pint <files>` → `php artisan test <path>` → `php artisan test` → `vendor/bin/phpstan analyse` (level max, 0 baseline) → `vendor/bin/pint --test`. App files follow Laravel idiom: no `declare(strict_types=1)`, Pint laravel preset (no spaces around `.` concatenation).
- **Providers are DERIVED from the registry, not listed.** `bootstrap/providers.php` spreads `array_map(fn (Module $m) => $m->providerClass(), Module::cases())` after the two platform providers — so adding/removing a module never edits the composition root by hand. `Module::providerClass()` owns the wiring-seam FQCN convention (`{namespace}\Providers\{Name}ServiceProvider`); both the composition root and the conformance test consume it (single source of truth). The composition root is boundary-exempt by design (D3) — it references module provider FQCNs on purpose. Each module's standard provider `extends Illuminate\Support\ServiceProvider` (NOT Filament's `PanelProvider` — that's the separate `AdminPanelProvider`, design D5) with empty `register()/boot()` = the seam F2+ fills.
- **Assert boot-time wiring via `app()->getLoadedProviders()`** — an FQCN-keyed array (`[Provider::class => true]`), Laravel API verified at `Application.php:1477` (`providerIsLoaded(string)` at 1488 is the single-key variant). Non-deferred providers (no `DeferrableProvider`) are eagerly registered, so empty seam providers appear there. Use `expect($loaded)->toHaveKeys($all)` for completeness and `->not->toHaveKey($fabricated)` for the negative (Pest `Mixins/Expectation.php:628/661`; `not->toHaveKey` via `OppositeExpectation::__call`). A non-existent provider FQCN in `bootstrap/providers.php` fails LOUD: the app can't boot → the whole Feature suite errors.
- **Relocating/renaming any `App\**` class → run `composer dump-autoload` in the same iteration.** The `App\` PSR-4 map resolves the *new* path at runtime for free, but Composer's non-authoritative classmap (`vendor/composer/autoload_classmap.php`) keeps a STALE entry pointing the OLD FQCN at the deleted file. `findFile()` checks the classmap before PSR-4 and returns the stale path without an existence check → `class_exists('Old\FQCN')` tries to `include` a missing file → PHP warning → PHPUnit/Pest escalates it to a test failure. `composer dump-autoload` regenerates the classmap (drops the stale entry); it touches only `vendor/composer/autoload_*.php` — never `composer.json`/`composer.lock`, and `vendor/` is gitignored, so zero commit/dependency churn. Verify with `php -r "require 'vendor/autoload.php'; var_dump(class_exists('Old\FQCN'));"` → must be `false` with no warning. Filament panel introspection for tests: `Filament\Facades\Filament::getPanel('admin')->getId()` (`FilamentManager.php:372` → `PanelRegistry::get($id, $isStrict=true)`; `Panel::getId()` via `HasId`).
- **The Architecture suite is boot-free, registry-located, and asserted by set-equality.** `tests/Architecture/` is its own `<testsuite>` in `phpunit.xml` so `php artisan test` runs it by default (the test-count delta is the proof) — but it is NOT bound to `Tests\TestCase` in `tests/Pest.php`: arch/conformance/convention tests need no Laravel container. Adding a phpunit `<testsuite>` is sufficient for default discovery — Pest finds test files from the phpunit testsuite directories; the `->in()` binding only chooses the base TestCase class; `--filter` works across all suites unaffected. Locate the modules root with `dirname((string) (new ReflectionClass(Module::class))->getFileName())`, never `app_path('Modules')`: reflection is boot-free, drift-proof (it follows wherever the registry file lives), and immune to a prior Feature test having flushed the container. Type-clean at PHPStan level max (which analyzes `tests/`) requires guarding the false-unions: `(string)` on `getFileName(): string|false`, `?: []` on `scandir(): array|false`. Assert "exactly these, nothing more/less" by sorting both the scanned entries and the `Module::cases()`-derived expected list, then `toBe` (strict `===`) — symmetric set-equality: a missing AND an extra entry both fail. Filter dot-entries (`! str_starts_with($entry, '.')`) so the `.`/`..` pseudo-entries and gitignored OS artifacts (`.DS_Store`) never cause a false red (D1 forbids `.gitkeep`, so no legitimate dot-entry exists). Each arch-test task (2.1–2.4) carries its own **red-proof** (temp violating fixture → suite red → remove → green, both outputs recorded here).
- **Import-boundary arch tests use `expect($ns)->not->toUse([$forbidden])->ignoring([$allowed])` (pest-plugin-arch v4.0.2, verified in `vendor/pestphp/pest-plugin-arch/src/`).** Confirmed semantics (read from source + empirically probed, design D4 — never from memory): (1) **`not->toUse([list])` fails if the target uses ANY namespace in the list** (NOT all) and the `ExpectationFailedException` message **names the exact `(source, target)` pair** ("Expecting 'App\Modules\Catalog' not to use 'App\Modules\Parties'.") — so one expectation per source module localises the offender without per-pair loops. (2) **`->ignoring([list])` strips uses by namespace PREFIX** (`LayerFactory::make` → `str_starts_with($use, $exclude)`), so ignoring `App\Modules\Parties\Contracts` allows references to anything under it while a reference elsewhere in `App\Modules\Parties` still fails — this is exactly how the Contracts/Events public-surface allow-list is expressed. (3) **Layer membership is also prefix-based** (`leaveByNameStart`); the nine module names are mutually non-prefixing, so `toUse(['App\Modules\Finance'])` never accidentally catches another module. Build BOTH the forbidden list and the ignore list from `Module::cases()` (filter out the source) — never hardcode. Arch expectations are lazily verified on `__destruct`/next-interaction inside the `it()` closure, so a bare `expect(...)` chain in a `foreach` is evaluated per iteration; they register real assertions (no risky/no-assertion flag). `tests/Architecture/ModuleBoundariesTest.php` holds BOTH boundary-law directions: the module-privacy loop (task 2.2) and the platform-direction loop (task 2.3).
  - **CRITICAL — arrays go on the DEPENDENCY side, NEVER the SOURCE side under `not`.** `not->toUse([d1,d2,…])` (array dependency) = "source uses ANY of these" → fails on any listed dep (safe; this is 2.2's `$forbidden`). But `expect([s1,s2,…])->not->toUse(X)` (array source) negates the *aggregate* positive `toUse` — which `Blueprint::expectToUse` defines as "**ALL** sources use X" — so the `not` passes the instant ONE source is clean, **silently masking a real violation in another source**. Empirically proven (task 2.3 red-proof): with a violating `App\Providers` class present, `expect(['App\Providers','App\Models','App\Http'])->not->toUse('App\Modules')` stayed GREEN (because `App\Models` is clean). The fix is to LOOP and assert one single-source expectation per source: `foreach ($sources as $s) expect($s)->not->toUse(X);` — reads "this source uses none of X", fails on any one dirty source. The platform list is the one legitimate hardcoded array (it's the *complement* of the registry — non-module `App\*` roots — so `Module::cases()` cannot derive it); keep it a single named `$platformNamespaces` array and loop it.
  - **What counts as a detected "use":** `ObjectDependenciesDescription::make` collects every AST `Node\Name` that passes `class_exists`/`interface_exists`/`enum_exists`/`function_exists`/`trait_exists` after the NameResolver expands it to an FQCN. A **type reference** (typed property/param, `extends`, `new`, param/return type) is reliably collected; a bare `Foo::class` constant-fetch was observed NOT to register (use a real type reference in red-proof fixtures). The repository globs the PSR-4 dir live (`Symfony\Finder` over `app/Providers/*.php`), so NEW fixture files are scanned without `composer dump-autoload`.
