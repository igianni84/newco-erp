# Progress ‚Äî foundations-modules-skeleton

## Codebase Patterns
(consolidated reusable patterns ‚Äî read first each iteration)

- **Module registry is the single source for "the nine modules".** `App\Modules\Module` (string-backed enum, `app/Modules/Module.php`): case name = namespace segment (`namespace()` ‚Üí `App\Modules\{CaseName}`), backing value = snake_case table prefix (`->value`, e.g. `catalog` ‚Üí table prefix `catalog_`, design D6), `letter()` = spec letter (`0,K,A,D,S,B,C,E,Admin`). Every architecture/conformance test MUST iterate `Module::cases()` ‚Äî never a hardcoded module list ‚Äî so module-set drift fails loudly.
- **Unit tests need no Laravel boot.** `tests/Pest.php` binds `Tests\TestCase` only to `Feature`; `tests/Unit/**` runs on plain PHPUnit `TestCase`. Pure-PHP tests (enums, value objects) belong in `tests/Unit/`. Pest `it()/expect()/->and()` globals work in both suites regardless of the binding.
- **Strong verbatim map assertion.** `expect($builtMap)->toBe([...])` uses strict `===` (order- + type-sensitive): asserts completeness, values, AND declaration order in one shot. Build the map by iterating `Module::cases()` keyed by `->name`, compare against the literal in spec order.
- **Quality loop is green-clean today:** `vendor/bin/pint <files>` ‚Üí `php artisan test <path>` ‚Üí `php artisan test` ‚Üí `vendor/bin/phpstan analyse` (level max, 0 baseline) ‚Üí `vendor/bin/pint --test`. App files follow Laravel idiom: no `declare(strict_types=1)`, Pint laravel preset (no spaces around `.` concatenation).
- **Providers are DERIVED from the registry, not listed.** `bootstrap/providers.php` spreads `array_map(fn (Module $m) => $m->providerClass(), Module::cases())` after the two platform providers ‚Äî so adding/removing a module never edits the composition root by hand. `Module::providerClass()` owns the wiring-seam FQCN convention (`{namespace}\Providers\{Name}ServiceProvider`); both the composition root and the conformance test consume it (single source of truth). The composition root is boundary-exempt by design (D3) ‚Äî it references module provider FQCNs on purpose. Each module's standard provider `extends Illuminate\Support\ServiceProvider` (NOT Filament's `PanelProvider` ‚Äî that's the separate `AdminPanelProvider`, design D5) with empty `register()/boot()` = the seam F2+ fills.
- **Assert boot-time wiring via `app()->getLoadedProviders()`** ‚Äî an FQCN-keyed array (`[Provider::class => true]`), Laravel API verified at `Application.php:1477` (`providerIsLoaded(string)` at 1488 is the single-key variant). Non-deferred providers (no `DeferrableProvider`) are eagerly registered, so empty seam providers appear there. Use `expect($loaded)->toHaveKeys($all)` for completeness and `->not->toHaveKey($fabricated)` for the negative (Pest `Mixins/Expectation.php:628/661`; `not->toHaveKey` via `OppositeExpectation::__call`). A non-existent provider FQCN in `bootstrap/providers.php` fails LOUD: the app can't boot ‚Üí the whole Feature suite errors.
