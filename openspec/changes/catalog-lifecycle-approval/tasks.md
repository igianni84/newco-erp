# Tasks — catalog-lifecycle-approval

Read `proposal.md`, `design.md` (D1–D11) and the spec delta `specs/product-catalog/spec.md` before any task. **Standing rules per task:** the five Quality Commands green at the end (format · test_filter · test · type_check · lint — full suite is `php -d memory_limit=512M vendor/bin/pest`, NOT `php artisan test`); `openspec validate catalog-lifecycle-approval --strict` green; **no protected files touched**; **exactly one migration** in this whole change — `catalog_producer_states` (D3/D10); **no `lifecycle_state` migration** (all seven columns + their `CHECK`s ship from the spine) and **no new Composer dependency** (`git diff main -- composer.json composer.lock` stays empty). **Verify every framework/vendor API in `vendor/`/`app/` before writing it** (lessons.md) — in particular `DomainEventRecorder::record(...)` returns the `DomainEvent`, `AuditRecorder::record(...)` takes `before`/`after`/`authorizationBasis`, `ConsumerRegistry::register(name, class, mode=Inline)`, and `DomainEventConsumer::handle(DomainEvent)`. **Every DB-touching task is verified on a real PostgreSQL 17 before it is marked done** — SQLite-green is necessary, never sufficient (`knowledge/testing/rules.md`); consumer/delivery tests use **`DatabaseMigrations`, not `RefreshDatabase`** (the inline post-commit hook fires only at `transactionLevel 0` — D11). **Boundary & persistence law:** the consumer and gate read only the event payload (`producer_id`) and Catalog's own projection — **no `App\Modules\Parties\*` import/query** (invariant 10; `ModuleBoundariesTest` stays green); the Models stay persistence-only — the **transition Action is the sole writer of `lifecycle_state`** (D1), and no Action writes `producer_id` or a parent FK. **Scope guard:** NO Filament/operator-console UI; NO KYC model or gate on `ActivateProducer` (D6 — upstream); NO cross-module retirement-blocking references (within-catalog only — D8); NO demand-side Parties.
