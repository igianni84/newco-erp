## Codebase Patterns
(consolidated reusable patterns ‚Äî read first each iteration)

- **Pint auto-imports docblock `{@see \FQCN}` refs (`fully_qualified_strict_types`).** If you `{@see \App\Modules\‚Ä¶\SomeClass}` a class a LATER task creates, Pint rewrites it into a real `use` import ‚Üí PHPStan then breaks on the unknown class. **Rule:** reference a not-yet-existent class (a future task's Action/Consumer/Event) as plain backticked text ‚Äî `` `ProducerLifecycleProjector` (task 1.2) `` ‚Äî never `{@see \FQCN}`, until the class exists. Same-namespace `{@see ClassName}` / `{@see Class::$prop}` are safe (Pint leaves them, no import needed).
- **PG-only CHECK test idiom** (template: `tests/Feature/Platform/ActorRoleConstraintTest.php`): a raw `DB::table('t')->insert([...])` bypasses the Eloquent enum cast (the app-layer floor), so on PG the DB CHECK is the SOLE gate. Wrap the probe insert in `DB::transaction(...)` (a SAVEPOINT, testing-rule #5) so a PG constraint-abort stays isolated and the row-state check after the throw is valid. Assert BOTH halves of the engine asymmetry, never a vacuous skip: `pgsql` ‚Üí message `toContain('<named_constraint>')` + row absent; `sqlite` ‚Üí message `''` + row present (the cast, not a DB CHECK, is the floor). NB: `captureConstraintViolation()` in that file is a top-level (GLOBAL) function ‚Äî do NOT redeclare it in another test file (fatal on full-suite load); inline the try/catch or name a fresh helper.
- **New projection/read-model migration shape** (mirrors the spine): `$table->id()` + a plain `unsignedBigInteger('<fk>_id')` (NEVER a cross-module FK/relation ‚Äî invariant 10) + a `string('status')` whose value-set is enforced by the enum cast on both engines PLUS a PG-only `CHECK (‚Ä¶ IN (‚Ä¶))` derived from `Enum::cases()` (so it can never drift) + `timestampsTz()`. Explicit, <63-char index names. CHECK DDL guarded by `if (DB::getDriverName() === 'pgsql')`.
- **First-consumer registration seam** (`{Module}ServiceProvider::boot()`, design D4 ‚Äî established by `CatalogServiceProvider`): register on the shared `ConsumerRegistry` via **method injection** ‚Äî `public function boot(ConsumerRegistry $registry): void { $registry->register(EVENT_NAME, Consumer::class); }`. The registry is a singleton bound in `AppServiceProvider` (all `register()`s run before any `boot()`, so the binding exists), so the instance `boot()` gets is the same one `DomainEventRecorder` fans out against. **Cross-module consumers key off BARE event-name strings**, never the emitter's event class (invariant 10 + the stricter task rule): anchor the names as the consumer's own `public const` (single source of truth for the provider registration + the handler's `match`), docblock-noting they mirror the emitter's `::NAME` verbatim. A registration test asserts `consumersFor(NAME)->toContain(Consumer::class)` (without it, zero deliveries fan out ‚Äî silent gate failure).
- **Idempotent + order-tolerant consumer (watermark upsert)**: read the row `->where(fk)->lockForUpdate()->first()` (real lock on PG serializes concurrent inline-vs-sweep deliveries for the same key; no-op on SQLite, where the assert carries correctness ‚Äî design D2/D4). Null ‚Üí `create(‚Ä¶last_event_id = $event->id)`; else apply **only when `$event->id` STRICTLY advances `last_event_id`** (`<=` ‚Üí no-op), so replays + stale/out-of-order events never regress. The handler opens **no** transaction (the `InlineDeliveryExecutor` wraps it ‚Äî DB-work-only contract). Test the guard by calling `app(Consumer::class)->handle($staleEvent)` **directly** ‚Äî a re-`deliver()` is a no-op on a `done` row (ledger terminality), so it can't drive the watermark branch.
- **Reading a `mixed` jsonb payload value under phpstan-max**: `(int) $event->payload['k']` trips `cast.int` (can't blind-cast `mixed`). Narrow instead: `$v = $event->payload['k'] ?? null; if (! is_int($v)) throw new UnexpectedValueException(‚Ä¶);` ‚Äî a malformed envelope then fails loudly (delivery retries ‚Üí dead-letters), never silently coerced. (Verified empirically: Laravel's `array` cast round-trips a JSON integer back as a PHP `int`, so `is_int` holds on the happy path.)
