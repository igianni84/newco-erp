- **Pint auto-imports docblock `{@see \FQCN}` refs (`fully_qualified_strict_types`).** If you `{@see \App\Modules\‚Ä¶\SomeClass}` a class a LATER task creates, Pint rewrites it into a real `use` import ‚Üí PHPStan then breaks on the unknown class. **Worse, for a cross-module class:** auto-importing an EXISTING `App\Modules\<other>\‚Ä¶` FQCN adds a real cross-module `use` ‚Üí **breaches invariant 10 and breaks `ModuleBoundariesTest`** (verified-by-reasoning task 2.1: a Catalog exception that wanted to cite Module K's `IllegalProducerTransition` analogue). **Rule:** reference a not-yet-existent class (a future task's Action/Consumer/Event) AND any other-module class as plain backticked text / prose ‚Äî `` `ProducerLifecycleProjector` (task 1.2) ``, "Module K's `IllegalProducerTransition`" ‚Äî never `{@see \FQCN}`. Same-namespace `{@see ClassName}` / `{@see Class::$prop}` are safe (Pint leaves them, no import needed).
- **PG-only CHECK test idiom** (template: `tests/Feature/Platform/ActorRoleConstraintTest.php`): a raw `DB::table('t')->insert([...])` bypasses the Eloquent enum cast (the app-layer floor), so on PG the DB CHECK is the SOLE gate. Wrap the probe insert in `DB::transaction(...)` (a SAVEPOINT, testing-rule #5) so a PG constraint-abort stays isolated and the row-state check after the throw is valid. Assert BOTH halves of the engine asymmetry, never a vacuous skip: `pgsql` ‚Üí message `toContain('<named_constraint>')` + row absent; `sqlite` ‚Üí message `''` + row present (the cast, not a DB CHECK, is the floor). NB: `captureConstraintViolation()` in that file is a top-level (GLOBAL) function ‚Äî do NOT redeclare it in another test file (fatal on full-suite load); inline the try/catch or name a fresh helper.
- **New projection/read-model migration shape** (mirrors the spine): `$table->id()` + a plain `unsignedBigInteger('<fk>_id')` (NEVER a cross-module FK/relation ‚Äî invariant 10) + a `string('status')` whose value-set is enforced by the enum cast on both engines PLUS a PG-only `CHECK (‚Ä¶ IN (‚Ä¶))` derived from `Enum::cases()` (so it can never drift) + `timestampsTz()`. Explicit, <63-char index names. CHECK DDL guarded by `if (DB::getDriverName() === 'pgsql')`.
- **First-consumer registration seam** (`{Module}ServiceProvider::boot()`, design D4 ‚Äî established by `CatalogServiceProvider`): register on the shared `ConsumerRegistry` via **method injection** ‚Äî `public function boot(ConsumerRegistry $registry): void { $registry->register(EVENT_NAME, Consumer::class); }`. The registry is a singleton bound in `AppServiceProvider` (all `register()`s run before any `boot()`, so the binding exists), so the instance `boot()` gets is the same one `DomainEventRecorder` fans out against. **Cross-module consumers key off BARE event-name strings**, never the emitter's event class (invariant 10 + the stricter task rule): anchor the names as the consumer's own `public const` (single source of truth for the provider registration + the handler's `match`), docblock-noting they mirror the emitter's `::NAME` verbatim. A registration test asserts `consumersFor(NAME)->toContain(Consumer::class)` (without it, zero deliveries fan out ‚Äî silent gate failure).
- **Idempotent + order-tolerant consumer (watermark upsert)**: read the row `->where(fk)->lockForUpdate()->first()` (real lock on PG serializes concurrent inline-vs-sweep deliveries for the same key; no-op on SQLite, where the assert carries correctness ‚Äî design D2/D4). Null ‚Üí `create(‚Ä¶last_event_id = $event->id)`; else apply **only when `$event->id` STRICTLY advances `last_event_id`** (`<=` ‚Üí no-op), so replays + stale/out-of-order events never regress. The handler opens **no** transaction (the `InlineDeliveryExecutor` wraps it ‚Äî DB-work-only contract). Test the guard by calling `app(Consumer::class)->handle($staleEvent)` **directly** ‚Äî a re-`deliver()` is a no-op on a `done` row (ledger terminality), so it can't drive the watermark branch.
- **Reading a `mixed` jsonb payload value under phpstan-max**: `(int) $event->payload['k']` trips `cast.int` (can't blind-cast `mixed`). Narrow instead: `$v = $event->payload['k'] ?? null; if (! is_int($v)) throw new UnexpectedValueException(‚Ä¶);` ‚Äî a malformed envelope then fails loudly (delivery retries ‚Üí dead-letters), never silently coerced. (Verified empirically: Laravel's `array` cast round-trips a JSON integer back as a PHP `int`, so `is_int` holds on the happy path.)
- **Shared generic transition mechanism (the lifecycle FSM seam ‚Äî design D1/D2).** `LifecycleTransition::transition()` is `@template TModel of Model&HasLifecycleState`. To thread `TModel` through `DB::transaction` under phpstan-max with NO forbidden `@var`/assert override, operate on the **passed `$model`** (typed `TModel`), never the `firstOrFail()` result ‚Äî Larastan erases `$model->newQuery()->‚Ä¶->firstOrFail()` to base `Model`, dropping the generic AND the interface. So: `$model->newQuery()->whereKey($model->getKey())->lockForUpdate()->firstOrFail()` (result **discarded** ‚Äî it only acquires the `FOR UPDATE` lock) **then `$model->refresh()`** loads the authoritative locked state onto the typed instance; `return $model` threads `TModel`, so a per-entity `handle(ProductMaster): ProductMaster` keeps its concrete return with zero casts. The from-state is read AFTER refresh ‚Üí a stale caller snapshot is overwritten by DB truth before the assert (race-safe; the lock serialises on PG, no-op on SQLite).
- **Lifecycle read contract is a typed METHOD, not a magic `@property`.** `HasLifecycleState::lifecycleState(): LifecycleState` ‚Äî because Larastan's Eloquent property extension **shadows an interface's `@property`** through a `Model&Interface` intersection (`‚Üí "Access to undefined property TModel of ‚Ä¶::$lifecycle_state"`), but resolves a real interface method cleanly. Spine models opt in with a one-line read accessor (a pure getter ‚Äî NOT a transition method, so they stay persistence-only). Same trick will serve any future generic-over-models mechanism.
- **Audit action derived from the TABLE, never `Str::snake(classBasename)`.** `catalog.<entity>.<verb>` builds `<entity>` as `Str::singular(Str::after($model->getTable(), 'catalog_'))` ‚Äî the table is canonical snake and survives the SKU acronyms (`catalog_sellable_skus ‚Üí sellable_sku`), whereas `Str::snake('SellableSKU')` mangles to `sellable_s_k_u`. Verified across all 7 spine tables. - **Pint method-name trap (sharpens Patterns #1).** `fully_qualified_strict_types` rewrites a docblock `{@see lowercaseMethod()}` to `{@see CasedClass()}` when an imported class matches it case-insensitively ‚Äî it "corrected" `{@see lifecycleState()}` ‚Üí `{@see LifecycleState()}` (the imported enum). - **Reading an uncast `actor_id` (or any bigint) column cross-engine.** A `Builder::value('actor_id')` (or `->first()->actor_id`) on a column NOT in the model's `casts()` returns a PHP `int` on SQLite but a numeric **STRING** on PostgreSQL (PDO_PGSQL's text protocol). A `===` against an `ActorContext` int id (or any int) then silently fails ONLY on PG ‚Äî SQLite-green hides it. Normalize every such read through one helper: `is_numeric($raw) ? (int) $raw : null` (covers int, numeric-string, and null/absent in one expression; `is_numeric` also satisfies phpstan-max so the subsequent `(int)` is not a `cast.int` on `mixed`). This is THE reason a guard that reads prior actors from `domain_events`/`audit_records` MUST be PG-verified. (Verified empirically on PG17 task 2.3.)
- **Asserting a multi-key jsonb snapshot (`before`/`after`) cross-engine.** PG `jsonb` does NOT preserve key insertion order (it reorders), so a whole-array `->toBe(['a'=>‚Ä¶,'b'=>‚Ä¶])` on a 2-plus-key snapshot is order-sensitive (`===`) and can pass on SQLite yet fail on PG. Assert keys INDIVIDUALLY and order-independently: `$after = $row->after ?? []; expect($after['k'] ?? null)->toBe(‚Ä¶)` (the `?? []` narrows the nullable jsonb to an array so phpstan-max is happy AND a null row fails loudly; the inner `?? null` tolerates a missing key without an offset error). A SINGLE-key snapshot is safe with whole-array `toBe` (no ordering to disturb).
