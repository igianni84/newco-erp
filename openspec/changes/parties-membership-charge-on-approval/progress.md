- **Prove a docblock/seam-name change is behaviour-neutral** with `git diff <file> | grep -E '^[+-]' | grep -vE '^[+-]{3}' | grep -vE '^\s*[+-]\s*\*'` → empty output means every changed line is a `*` comment. Pair it with the Action's isolated contract test passing **unedited** + PHPStan resolving the docblock `{@see}` refs.
- **Docblock `{@see}` imports:** same-namespace Actions (`ApproveProfile`, `DeclineProfile`, `ActivateProducer`, `CloseClub`, `IssueClubCredit`) and already-imported classes need **no** new `use` — only a *cross-namespace* `{@see}` would force one (Pint import-boundary lesson 2026-06-20). - **Docs-first within a change:** a docblock may describe the change's **target** design (here: `ApproveProfile` invoking `ActivateProfile` synchronously) one task before the wiring lands, as long as no test asserts the wiring from prose — green stays green between tasks.
- **A red-green FSM-shape flip inverts EVERY observer in the SAME iteration** (one commit stays green): grep `ProfileState::Approved` + the Action name across `tests/` to enumerate the observers exhaustively BEFORE editing — the design's enumeration is a checklist, the grep is the proof it's complete. Split them into (a) *outcome observers* (assert state/events after the real Action → flip these), (b) *precondition helpers* (double-drove approve→activate → **delete** the now-illegal second call, no try/catch), (c) *factory-forced-state* datasets (never call the flipped Action → leave untouched), (d) *source-scan guards* (`SupplyLifecycleChainTest` allow-list, `ComplianceIndependenceTest` OC-write count → stay diff-free iff no Action/Event class is added and the literal `'…' =>` write count is unchanged). The isolated writer's own contract (`ProfileActivationTest`) stands unedited.
- **Bare-path `--filter` on `OperatorPanel/Parties/` reds 5 `*ConsoleI18nTest`** with "expected the shared operator-console sink scanner to be loaded" — `scanOperatorConsoleHardcodedSinks()` is declared in `ProductMasterConsoleI18nTest` (Catalog dir) and only loaded by the FULL suite (lesson 2026-06-20). Not a regression; verify these via `php -d memory_limit=2G vendor/bin/pest` (whole suite), never a bare Parties path.
- **Nested-Action atomicity:** an Action may invoke a sibling Action's `handle()` inside its own `DB::transaction` — the sibling opens a savepoint (nested tx), so `DomainEventRecorder`'s `transactionLevel() === 0` guard stays satisfied, and any throw rolls the whole outer tx back. The recorder threads only *explicitly-passed* causation, so a sibling recording with no causation stays a ROOT even when called inside another Action's tx.
