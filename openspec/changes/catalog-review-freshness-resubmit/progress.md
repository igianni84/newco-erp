- **Pest global-class import gotcha (task 1.2 → lessons.md 2026-07-02):** never `use RuntimeException;` (or any global class) in a Pest test file — the file is the global namespace, so it warns "use statement with non-compound name has no effect". Reference bare `RuntimeException::class` (repo convention, e.g. `CurrencyTest.php`).
- **Block-gate is live for ALL seven entities the moment it lands in the shared `ApprovalGovernance::guard()` (task 2.2)** — but it only *breaks* a test that actually activates a rejection-pending entity (reject-then-activate). Verified: the reject→activate-succeeds path was UNIQUE to `ProductMasterLifecycleTest`'s "not terminal" test; the other six spine `*LifecycleTest`s, all seven console tests and `DemoSeederTest` stop at the rejection (assert `%Activated% == 0`) or never reject before activating, so they stayed green with zero edits. When task 3.x adds the six per-entity `Resubmit*`, the block-gate is ALREADY enforcing on them — 3.2's cross-entity test only needs to prove the re-arm, not re-add the gate. Grep shape: `grep -rln "->reject(\|Reject.*Review\|latestGovernanceAction" tests/` then check each hit for an activate-after-reject.
- **Derive-from-audit "latest action" read (task 2.2, reused by 3.x/5.1):** `assertNotRejectionPending` mirrors `reviewerOf` exactly — `AuditRecord::query()->where('module', Module::Catalog->value)->where('entity_type', $entity)->where('entity_id', $entityId)->orderByDesc('id')->value('action')`, then `is_string($latest) && str_ends_with($latest, '.rejected')`. The `is_string` guard is load-bearing: a never-audited entity returns `null` from `->value()` and `str_ends_with(null, …)` would TypeError under PHP 8.5. Reads only the LATEST row, so a `.resubmitted`/`.reopened`/`.submitted` on top of a buried `.rejected` is NOT pending (proven by the block-gate tests).
- **SoD survives N rejection rounds — the reviewer stays the original submitter (task 2.3, reused by 3.2/console):** `reviewerOf` reads the latest audit action matching `LIKE '%.submitted'`, and `catalog.<e>.resubmitted` does NOT match that pattern (the char before `submitted` is `e`, not `.`). So no matter how many reject→re-submit rounds run, the derived reviewer stays the SINGLE original `.submitted` actor; only `creatorOf` (first `*Created` event) and the acting approver complete the lineage. The **clean multi-round fixture** is three operators: Creator creates + re-submits every round, Reviewer submits + rejects every round, a distinct Approver activates once — the final SoD (C≠R≠A) holds because re-submits/rejects perturb neither `creatorOf` (no event) nor `reviewerOf` (no `.submitted`). A blocked activate attempt rolls back whole (throw in `guard()` before the write), leaving no `.activated` row / event and the latest action still `.rejected`.
- **Per-row assertions over an Eloquent `->get()` under PHPStan max (task 2.3 → lessons.md 2026-07-02):** a BARE `$collection[$i]->prop` reds `property.nonObject` — Larastan types `Collection<int, Model>` offset access as `Model|null`, and a preceding `expect($c)->toHaveCount(n)` does NOT narrow it. `$collection[$i]->prop ?? default` is clean (the `??` guards the null-object read — that's why `$rows[0]->after ?? []` passed while bare `$rows[0]->actor_id` failed). Project to a plain array first (`->pluck('col')->all()`, `->map(fn (Model $m) => …)->all()`) and index that, or use `->sole()`/`->first()` for single-row reads.
