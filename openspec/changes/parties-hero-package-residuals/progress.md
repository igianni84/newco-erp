## Codebase Patterns
(consolidated reusable patterns ŌĆö read first each iteration)

- **Two mutants, not one ŌĆö and the second is the one that matters.** A mutant that reds the new pin *and* the pre-existing tests proves the pin **fires**, not that it is **needed** (delete the pin; the suite still catches that mutant). Necessity is proved by the weaker, realistic drift that the existing suite **passes** and only the new pin fails. Empirically here: moving `ApproveProfile`'s from-state guard below the whole gate block reds 13 tests (7 of them pre-existing) ŌĆö loud and uninformative; moving it just past `lockAndCountOccupiedSeats()` but **before** the gate's `if` leaves all 13 pre-existing tests green and reds exactly the 6 new dataset rows, each on `expect($statements)->toBeEmpty()`. Design R5 asks for the first; ask for the second too, or "the positive assertion discriminates nothing" stays an argument rather than a measurement. (`knowledge/testing/hypotheses.md` ŌĆö corollary added.)
- **A chained `expect(...)->and(...)` SHORT-CIRCUITS, so ONE MUTANT PER CONJUNCT ŌĆö and the assertion COUNT proves which conjuncts ran.** Pest evaluates the chain left to right and throws on the first failure, so a mutant breaking conjunct 1 never evaluates conjunct 2, leaving it unfalsified ŌĆö decoration a later iteration will delete. Count the independent ways the *claim* can break, not the assertions. Measured in 2.1 on the root-ness pair (`causation_id` null ┬Ę `correlation_id === event_id`): green baseline **12384** assertions; mutant A (`causationId:`) ŌåÆ 1 failure at the `toBeNull()` line, **12383** assertions (the chain aborted); mutant B (a shared `correlationId:`) ŌåÆ 1 failure at the `toBe()` line, **12384** (conjunct 1 ran and passed). Two numbers, and the short-circuit stops being a guess. Design each mutant so the other conjuncts stay green. **Not** the two-mutants corollary above: that is loud-vs-realistic drift against *one* assertion; this is one mutant per *independent* assertion. Task 3.2's `title` + `status` + `body` notification pin is three conjuncts. (`knowledge/testing/hypotheses.md` ŌĆö new hypothesis, 1/3.)
- **Pest's `toThrow($class, $message)` compares the message with `toBe`, i.e. EXACTLY** (`vendor/pestphp/pest/src/Mixins/Expectation.php:960`; the `assertStringContainsString` calls on :972/:978 serve the *other* arities). So *"raises reason X, never reason Y"* needs no second assertion ŌĆö passing the localized `cannot_approve` string already excludes `club_at_capacity`. Do not add a defensive `->not->toContain(...)`; it is dead weight.
- **`Factory::count(0)->create()` is safe** ŌĆö `Factory::make()` short-circuits on `$this->count < 1` and returns an empty collection (`vendor/laravel/framework/.../Factories/Factory.php:443`). It does **not** reach `range(1, 0)`, which in PHP counts *downwards* and would seed two rows. A capacity-`0` at-parity fixture therefore needs no `if ($occupied > 0)` guard.
- **An at-parity fixture must account for the SUBJECT's own seat, and capacity `0` is the only one that need not.** `wouldOversell()` is true at every occupancy when capacity is `0`, so one fixture serves every from-state. A *free-seat* fixture cannot be reused that way: `active` occupies a seat (`OCCUPYING_STATES = [Active, Suspended]`) and `lapsed` does not, so `capacity: 2, occupied: 1` is at parity for an `active` subject and below parity for a `lapsed` one. Cap well above the seeded occupancy (`5, 1`) when the row must read "a free seat" for both.
- **The only honest uncapped fixture is an explicit `null` in `by_club_id` beneath a CAPPED `default`.** The adapter resolves the override on `array_key_exists`, so presence wins and the null pins that one Club uncapped. A huge cap (`PHP_INT_MAX`) exercises the **capped** branch (`$capacity !== null`) of a Club that merely is not full ŌĆö a different code path, and not the claim.
- **Inline `config()->set(...)` beats a new global helper.** Pest `include`s every selected test file into one process while building the suite, so a second `function fooBar()` is a **fatal redeclare that kills the whole run** before any test executes. This file already declares `seatClubTo()` / `freeOneSeat()`; the new pin needed a `?int` capacity and a capped default, so it sets config inline ŌĆö the shape the file's own from-state test and `ProfileRenewalCapacityGateTest`'s dataset both already use. A helper would have bought nothing and risked everything.
