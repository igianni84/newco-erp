## Codebase Patterns
(consolidated reusable patterns â€” read first each iteration)

- **White-box concurrency-guard test (substrate).** A race a single-connection SQLite test DB cannot interleave through the public surface is tested deterministically by constructing the exact post-race row state and invoking the private guard method via reflection (`inlineInvokePrivate()` in `InlineDeliveryTest.php`, matching the Money/FxRate `ReflectionMethod` pattern â€” PHPStan-max clean, since phpstan analyses `tests/`). Make the discriminating assertion load-bearing and non-vacuous: re-invocation is caught via `RecordingConsumer::$handled` (empty vs `[event_id]`), resurrection via terminal `status` (the attempts value can be vacuous when stale+1 == DB value, so don't rely on it alone). Engine-agnostic â†’ identical on both lanes; the PG lane is the true-contention proof (`lockForUpdate()` = real `FOR UPDATE` on PG, documented no-op on single-writer SQLite via the base grammar's `compileLock`).
- **Query-builder UPDATE vs model UPDATE (casts).** A query-builder `->update([...])` runs NO model casts: pass an enum as its `->value` and an `immutable_datetime` column as a raw `CarbonImmutable`. The Carbon binding is formatted by `Connection::prepareBindings()` via `$grammar->getDateFormat()` â€” the SAME `Y-m-d H:i:s` the model cast uses (`Model::getDateFormat()` also reads the grammar) â€” so the column round-trips identically on both engines. Reach for the builder form when you need a conditional write (`where('status', â€¦)->update(â€¦)`) a model `->update()` cannot express. A builder `->update()` also returns the **affected-row count** â€” use it to branch (e.g. `$recorded === 0` â‡’ a sibling already won the row, so there is nothing to log/surface).
- **Pinning a connector-applied DB session setting (timezone / isolation / search_path).** A value Laravel's connector pushes to the session on connect (`config/database.php` `pgsql.timezone` â†’ `PostgresConnector::configureTimezone` issues `SET TIME ZONE 'UTC'`) is proven on two levels: (1) a **config-pin** â€” `expect(config('database.connections.pgsql.timezone'))->toBe('UTC')` â€” runs on BOTH lanes (config arrays load regardless of the active engine) and fails loudly if the key is dropped; (2) an **engine-guarded behavioural read-back** â€” `if (DB::getDriverName() === 'pgsql') expect(DB::scalar('show time zone'))->toBe('UTC')` â€” proves the connector *actually applied* it, runs only on the PG lane, and leaves the off-lane (SQLite) assertion positive (the config pin), never a vacuous skip. `DB::scalar('show time zone')` returns the live PG session zone (`UTC`). Home for connection-level config pins: `tests/Feature/EnvironmentTest.php` (mirrors its existing `database.connections.sqlite.database` pin + inline engine guard).
- **Assert a log was emitted at level X with context Y (PHPStan-max-clean facade spy).** `Log::fake()` does NOT exist on this framework â€” use the Mockery facade spy: `$log = Log::spy()` (typed `\Mockery\MockInterface`), trigger, then `$log->shouldHaveReceived('warning')` / `->shouldNotHaveReceived('error')`. To match the structured **context without pinning the message wording**, pass a closure as the SECOND arg: `$log->shouldHaveReceived('info', fn (string $message, array $context): bool => â€¦)` â€” Mockery treats it as a `withArgs` matcher and the 2-arg `shouldHaveReceived` returns `self` (no `->once()` chaining, which the `MockInterface` type would reject; the closure form asserts "â‰Ą1 matching call"). Three PHPStan-max gotchas: (1) the closure param needs a `/** @param array<string, mixed> $context */` docblock â€” PHPStan reads it on an inline closure/arrow-fn; (2) reading `$context['k']` yields `mixed`, so `=== 2` is fine but `str_contains((string) $x, â€¦)` trips `cast.string` â€” narrow with `is_string($x)` (a full closure + local var), never a `(string)` cast; (3) `Log::spy()` does NOT throw on un-asserted calls, so the non-spying failure tests (FailingConsumer) keep logging to the real test log (`storage/logs`, exists/writable) harmlessly.
