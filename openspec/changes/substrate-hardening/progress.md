# Progress — substrate-hardening

## Codebase Patterns
(consolidated reusable patterns — read first each iteration)

- **White-box concurrency-guard test (substrate).** A race a single-connection SQLite test DB cannot interleave through the public surface is tested deterministically by constructing the exact post-race row state and invoking the private guard method via reflection (`inlineInvokePrivate()` in `InlineDeliveryTest.php`, matching the Money/FxRate `ReflectionMethod` pattern — PHPStan-max clean, since phpstan analyses `tests/`). Make the discriminating assertion load-bearing and non-vacuous: re-invocation is caught via `RecordingConsumer::$handled` (empty vs `[event_id]`), resurrection via terminal `status` (the attempts value can be vacuous when stale+1 == DB value, so don't rely on it alone). Engine-agnostic → identical on both lanes; the PG lane is the true-contention proof (`lockForUpdate()` = real `FOR UPDATE` on PG, documented no-op on single-writer SQLite via the base grammar's `compileLock`).
- **Query-builder UPDATE vs model UPDATE (casts).** A query-builder `->update([...])` runs NO model casts: pass an enum as its `->value` and an `immutable_datetime` column as a raw `CarbonImmutable`. The Carbon binding is formatted by `Connection::prepareBindings()` via `$grammar->getDateFormat()` — the SAME `Y-m-d H:i:s` the model cast uses (`Model::getDateFormat()` also reads the grammar) — so the column round-trips identically on both engines. Reach for the builder form when you need a conditional write (`where('status', …)->update(…)`) a model `->update()` cannot express.
