# Progress — substrate-hardening

## Codebase Patterns
(consolidated reusable patterns — read first each iteration)

- **White-box concurrency-guard test (substrate).** A race a single-connection SQLite test DB cannot interleave through the public surface is tested deterministically by constructing the exact post-race row state and invoking the private guard method via reflection (`inlineInvokePrivate()` in `InlineDeliveryTest.php`, matching the Money/FxRate `ReflectionMethod` pattern — PHPStan-max clean, since phpstan analyses `tests/`). Make the discriminating assertion load-bearing and non-vacuous: re-invocation is caught via `RecordingConsumer::$handled` (empty vs `[event_id]`), resurrection via terminal `status` (the attempts value can be vacuous when stale+1 == DB value, so don't rely on it alone). Engine-agnostic → identical on both lanes; the PG lane is the true-contention proof (`lockForUpdate()` = real `FOR UPDATE` on PG, documented no-op on single-writer SQLite via the base grammar's `compileLock`).
- **Query-builder UPDATE vs model UPDATE (casts).** A query-builder `->update([...])` runs NO model casts: pass an enum as its `->value` and an `immutable_datetime` column as a raw `CarbonImmutable`. The Carbon binding is formatted by `Connection::prepareBindings()` via `$grammar->getDateFormat()` — the SAME `Y-m-d H:i:s` the model cast uses (`Model::getDateFormat()` also reads the grammar) — so the column round-trips identically on both engines. Reach for the builder form when you need a conditional write (`where('status', …)->update(…)`) a model `->update()` cannot express. A builder `->update()` also returns the **affected-row count** — use it to branch (e.g. `$recorded === 0` ⇒ a sibling already won the row, so there is nothing to log/surface).
- **Pinning a connector-applied DB session setting (timezone / isolation / search_path).** A value Laravel's connector pushes to the session on connect (`config/database.php` `pgsql.timezone` → `PostgresConnector::configureTimezone` issues `SET TIME ZONE 'UTC'`) is proven on two levels: (1) a **config-pin** — `expect(config('database.connections.pgsql.timezone'))->toBe('UTC')` — runs on BOTH lanes (config arrays load regardless of the active engine) and fails loudly if the key is dropped; (2) an **engine-guarded behavioural read-back** — `if (DB::getDriverName() === 'pgsql') expect(DB::scalar('show time zone'))->toBe('UTC')` — proves the connector *actually applied* it, runs only on the PG lane, and leaves the off-lane (SQLite) assertion positive (the config pin), never a vacuous skip. `DB::scalar('show time zone')` returns the live PG session zone (`UTC`). Home for connection-level config pins: `tests/Feature/EnvironmentTest.php` (mirrors its existing `database.connections.sqlite.database` pin + inline engine guard).
- **Assert a log was emitted at level X with context Y (PHPStan-max-clean facade spy).** `Log::fake()` does NOT exist on this framework — use the Mockery facade spy: `$log = Log::spy()` (typed `\Mockery\MockInterface`), trigger, then `$log->shouldHaveReceived('warning')` / `->shouldNotHaveReceived('error')`. To match the structured **context without pinning the message wording**, pass a closure as the SECOND arg: `$log->shouldHaveReceived('info', fn (string $message, array $context): bool => …)` — Mockery treats it as a `withArgs` matcher and the 2-arg `shouldHaveReceived` returns `self` (no `->once()` chaining, which the `MockInterface` type would reject; the closure form asserts "≥1 matching call"). Three PHPStan-max gotchas: (1) the closure param needs a `/** @param array<string, mixed> $context */` docblock — PHPStan reads it on an inline closure/arrow-fn; (2) reading `$context['k']` yields `mixed`, so `=== 2` is fine but `str_contains((string) $x, …)` trips `cast.string` — narrow with `is_string($x)` (a full closure + local var), never a `(string)` cast; (3) `Log::spy()` does NOT throw on un-asserted calls, so the non-spying failure tests (FailingConsumer) keep logging to the real test log (`storage/logs`, exists/writable) harmlessly.

(2) `InlineDeliveryExecutor::attempt()` now `return`s an `AttemptOutcome` (`return DB::transaction(fn (): AttemptOutcome …)` — the framework types `transaction()` generically as the closure's return; `Skipped` on the locked-status bail, `Delivered` on the `done` flip, else the catch returns `recordFailure()`'s outcome). (3) `recordFailure()` now returns `AttemptOutcome` and **logs**: it branches on the builder `->update()` affected-row count — `0` ⇒ a sibling won the row ⇒ `Skipped`, no log; otherwise `Log::warning` for a still-retryable failure or `Log::error` on the `$attempts >= $maxAttempts` dead-letter transition, each with context `{delivery_id, consumer, attempts, error}`. (5) `SweepCommand::handle()` logs `Log::info` `events:sweep complete: swept=%d failed=%d` (swept = delivered + failed) with a `{swept, failed}` context, still returns `self::SUCCESS`.
