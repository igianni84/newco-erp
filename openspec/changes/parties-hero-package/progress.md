## Codebase Patterns
(consolidated reusable patterns ‚Äî read first each iteration)

- **The K read-port seam, now shipped three times.** A Module-K read of a deferred module is three files + one line: `Contracts/<X>Reader.php` (interface, one method), `Reads/<Adapter>.php` (launch stand-in), and a plain `$this->app->bind(Interface::class, Adapter::class)` in `PartiesServiceProvider::register()` ‚Äî never `singleton`. Precedents: `CustomerTransactionTotalsReader`‚Üí`Null‚Ä¶` (RM-02, Module S), `PartyComplianceStatusReader`‚Üí`Database‚Ä¶`, and now `HeroPackageCapacityReader`‚Üí`ConfigHeroPackageCapacityReader` (Module A). The adapter's docblock carries *why it is a stand-in* and *what replaces it*.
- **PHPStan max analyses `tests/`, and it audits your fakes.** An in-test anonymous-class fake of a `?int`-returning contract that never returns `null` on any path draws `return.unusedType` ("never returns null so it can be removed from the return type"). Do **not** narrow the fake's signature to silence it ‚Äî the flag means the fake is too weak to prove the consumer handles the null arm. Make it answer both arms (`$clubId === 1 ? null : $clubId * 10`). **Tasks 2.2 / 2.4 / 3.2 / 6.1 all bind fake capacity readers ‚Äî they will hit this.**
- **`env()` yields a string; cast at the adapter boundary, once.** `config('parties.hero_package.capacity.default')` is `mixed`. The house normalisation idiom is `is_numeric($x) ? (int) $x : <safe default>` (`Catalog\Lifecycle\ApprovalGovernance::roleCount()` is the precedent). For capacity the safe default is `null` ‚áí uncapped, so a malformed value dark-launches rather than garbage-casting.
- **A per-id config override must be keyed on `array_key_exists`, not truthiness / `??`.** `by_club_id[$clubId] ?? $default` would silently discard an explicit `null` override (pin one Club uncapped beneath a capped default) and fall through to the default. Presence wins.
- **The uncapped test default is the reason the suite stays green.** No `PARTIES_HERO_PACKAGE_CAPACITY` in `phpunit.xml`, and no `.env.testing` exists in this repo at all ‚áí `default = null` ‚áí every one of the 2221 pre-existing tests runs against unchanged behaviour. Any task tempted to set that env var globally breaks the whole suite; bind a fake reader or `config()->set()` per-test instead.
