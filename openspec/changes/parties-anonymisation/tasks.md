## 3. Anonymisation core

- [ ] 3.1 **`AnonymisedPlaceholders` deterministic helper** (design D1) — a pure value-object/helper deriving per-Customer-unique placeholders from the Customer id: `email → anonymised+{id}@anonymised.invalid`, `name → "Anonymised Customer {id}"`, `phone → null`, `date_of_birth → null`, and Address personal-field placeholders. Deterministic, never `random`/`faker`.
  - Acceptance: same id → same placeholders (reproducible); two ids → distinct emails (UNIQUE-safe); no randomness.
  - Test hint: unit test (no `RefreshDatabase`) — `expect(AnonymisedPlaceholders::for(7)->email)->toBe('anonymised+7@anonymised.invalid')`; two ids yield distinct emails; calling twice for the same id is equal.

- [ ] 3.2 **`AnonymiseCustomer` action — gate + overwrite + `anonymised_at`** (design D1/D2/D4; spec — *Customer Anonymisation*, *Anonymisation Hold Precedence*, *Customer Address*) — one `DB::transaction` + `lockAndRefresh` + operator floor: (a) **Hold-precedence gate** — read `PartyComplianceStatusReader::forCustomer(...)`, throw the localized blocked-exception iff an active `compliance` Hold covers the Customer (never touch the `Hold` model directly); (b) overwrite Customer PII + every scoped Address's personal fields with `AnonymisedPlaceholders`; (c) set `anonymised_at`; (d) **idempotent** — an already-anonymised Customer is a no-op. Must NOT write `status` or record any status event (orthogonality).
  - Acceptance: on an un-held Customer → PII + Address personal fields overwritten (deterministic), `anonymised_at` set, Profiles/Addresses **not** deleted, `status` unchanged; on a `compliance`-held Customer → localized throw, nothing changed; a second call → no-op; anonymising a `closed` Customer leaves it `closed`.
  - Test hint: feature tests — happy path asserts overwritten email/name/dob + Address fields + `anonymised_at` set + Profile row still present + `status` unchanged + no `CustomerClosed` event; `compliance` Hold → `expect(fn () => AnonymiseCustomer::run($id))->toThrow(...)` + Customer untouched; two Customers anonymised → both succeed, distinct emails (UNIQUE-safe); idempotent second call → unchanged.
