- **Form-less, visibility-gated lifecycle verb (the cheap path for a bare-`int $id` Action).** When a domain Action takes only the entity `int $id` (no operands), the verb is the kit's `lifecycleAction($verb, $successKey, $invoke)` with `$form` left null — `$invoke = fn (Model $record, string $notes) => app(SomeAction::class)->handle($this->recordOf(Customer::class, $record)->id)` (`$notes` unused). To gate it to a legal from-state, chain `->visible(fn (): bool => $this->predicate($this->recordOf(Customer::class, $this->getRecord())))` onto the returned Action — the `->visible()` closure reads `$this->getRecord()` (the page record), evaluated per-record. Extract each predicate to a `private` helper so the magic-string token + the complement-of-guard rationale (design D4) live in ONE place. Predicate a STATE enum through its model CAST VALUE, NEVER an import (design D5): use `=== null || ->value === 'token'` when a prior `=== null` narrows the operand (non-nullsafe), or `?->value === 'token'` when the column is independently nullable (nullsafe — and PHPStan-max clean, no `nullsafe.neverNull`, because the cast is genuinely `Enum|null`). Because the predicate is the EXACT COMPLEMENT of the Action's domain from-state guard, a rejected transition is unreachable through the surface — prove its reject with a domain `toThrow` + `assertActionHidden`, never an `action_failed` the hidden verb can't raise (the Filament hidden-action landmine). The Action stays the sole writer; the console never re-checks the from-state.

- **Page header-action visibility-test API (Filament 5.6.7, pinned against installed source — NOT memory).** The header-action twin of the Holds table's `assertTableActionVisible/Hidden`. Prove a verb's reject via a domain `expect(fn () => app(Action::class)->handle($id))->toThrow(...)` + `assertActionHidden('verb')`, NEVER an `assertNotified(action_failed)` the page can't raise.
