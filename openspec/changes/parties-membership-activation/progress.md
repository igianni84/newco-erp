## Codebase Patterns
(consolidated reusable patterns — read first each iteration)

- **Additive-nullable-timestamp migration (no CHECK).** Plain `timestampTz()->nullable()` columns need NO PG-only value-set CHECK — there is no enum to pin (the type is the domain); the `immutable_datetime` cast carries the typed-read floor on both engines. Simpler than `2026_06_17_000001` (which adds CHECKs because its columns are enum-backed). `down()` is a bare `dropColumn([...])` — nothing to drop first.
- **Isolated PG17 reversibility proof.** To prove ONE migration's `down()`/`up()` on PG without rolling back the whole chain: `migrate:fresh` → `migrate:rollback --path=<migration>` (runs only that down()) → assert columns gone + table intact (`information_schema.columns` count + `to_regclass(...) IS NOT NULL`) → `migrate --path=<migration>` (re-up) → assert columns back. (`--path` rollback prints a cosmetic "0001_..._create_users_table Migration not found" line — the absent default users migration — ignore it; the column count delta is the truth.)
- **Null-safe round-trip assertion for nullable datetime casts.** Assert `expect($read->col)->toBeInstanceOf(CarbonImmutable::class)` (proves non-null) AND `->and($read->col?->format('Y-m-d H:i:s'))->toBe('...')` (value round-trip). The `?->` keeps PHPStan-max happy on a `CarbonImmutable|null` property without `assert()`/`@var`/casts; the `->format()` read-through is testing-rule #4 (never byte-compare a raw `timestamptz` — it renders `+00` on PG). Use fixed no-microsecond moments so the round-trip is deterministic across engines.
- **The arch OOM hits the full suite on SQLite too, not just PG.** `php artisan test` (full) fataled at the default 128 MB `memory_limit` on SQLite (pest-plugin-arch docblock parsing). Run the FULL suite on either engine with `php -d memory_limit=512M vendor/bin/pest`. (A single filtered file runs fine at default memory.)
- **An EVENT-SILENT transition Action injects NO recorder/actor — mirror `RecordKycRejected`, not `ActivateProducer`.** A from-state-guarded Action that writes only `state`/`status` and records NO domain event (the column change IS the audit record — § 15.2 audit-only transitions) takes the bare shape of `RecordKycRejected`: NO constructor, NO `DomainEventRecorder`/`ActorContext` dependency, just `handle()` → `DB::transaction` → `lockForUpdate` re-read → from-state guard → `update([...])`. `DeclineProfile` (`applied → rejected`) is this shape. An EVENTED transition (`ApproveProfile` conditionally records `OriginatingClubLocked`; the upcoming `ActivateProfile`/`ActivateCustomer` record their `*Activated`) injects recorder+actor and mirrors `ActivateProducer`. Decide by "does it record an event?", not by surface symmetry — injecting an unused recorder is dead code the codebase deliberately avoids. (Resolves the task's generic "each inject DomainEventRecorder + ActorContext" against the real precedent.)
- **ACTION-guard narrowing topology (now CONFIRMED, was predicted in T1.3).** A new demand-side transition ACTION trips exactly three guard slices, each narrowed in the SAME task: (1) `SupplyLifecycleChainTest`'s exact-set whitelist (`$transitions->toEqualCanonicalizing([...])` ~L323) — ADD the Action to a `$demandSideTransitions` group; (2) `ComplianceIndependenceTest`'s forbidden-Action `not->toContain` list (~L150-156) — REMOVE the now-shipped name, KEEP the still-deferred ones (`LockOriginatingClub`/`SetOriginatingClub` stay forbidden forever — the OC lock is an in-tx side-effect, never an Action); (3) if the Action writes `originating_club_id`, `ComplianceIndependenceTest`'s OC write-scan (~L168) — add an `elseif ($name === '<Action>')` branch admitting exactly 1 `'originating_club_id' =>` write (and assert it is NOT `=> null`, i.e. a set, never a reset). Event CLASSES (T1.3) trip a DIFFERENT slice (the `glob(Events/*.php)` existence loop) — orthogonal. Keep the Action's docblock free of the literal `'originating_club_id' =>` token so the substr_count write-scan stays exact.
- **`originating_club_id` reads need loose `toEqual` (uncast bigint FK — testing trap 6).** Direct Eloquent column re-reads of `originating_club_id` (`Customer::findOrFail(...)->originating_club_id`) use `->toEqual($club->id)`, not `->toBe(...)` — an uncast bigint FK reads back as a numeric string on PG, a PHP int on SQLite. Event-payload ids stay `->toBe(...)` (they decode from jsonb as reliable PHP ints — trap 3).
- **Deferred cross-module-trigger seam-proof: assert on the EVENT FILE NAME + `class_exists(...)->toBeFalse()`, never file contents.** When an evented Action ships the within-module *writer* of a transition whose production *trigger* is a deferred other-module event (e.g. `ActivateProfile` ships `approved → active`; the `MembershipFeePaid` listener is a Module-E seam — design L5), the seam-proof test asserts: (1) `class_exists(<Action>::class)->toBeTrue()` (the writer exists); (2) `class_exists('App\Modules\<OtherModule>\Events\<TriggerEvent>')->toBeFalse()` AND `class_exists('App\Modules\<ThisModule>\Events\<TriggerEvent>')->toBeFalse()` (zero-invention — no fabricated contract); (3) the `glob(Events/*.php)` basename list `->not->toContain('<TriggerEvent>')`. **Assert on the file NAME, never `file_get_contents` substring** — the Action's and the event's docblocks legitimately *mention* the seam in prose (analogous to the OC write-scan's docblock-token caveat). A listener can only type-hint an event class that does not exist, so the absent contract also forecloses the listener.
- **Event-payload `*_at` moment with no backing column → `CarbonImmutable::now()->toIso8601String()`.** When an event payload must carry a timestamp the entity has no column for (e.g. `OriginatingClubLocked.locked_at` — the lock surface is `originating_club_id` alone), use the codebase's single-moment idiom `CarbonImmutable::now()` (as `LiftHold`/`RecordKycVerified`/`RecordCustomerScreening` do for their `*_at` writes), serialized with `->toIso8601String()`. **Test it by freezing:** `CarbonImmutable::setTestNow($moment)` in the test body + `afterEach(fn () => CarbonImmutable::setTestNow())` to reset (the `SweepTest` idiom), then assert the exact ISO string inside the `->toBe([...])` snapshot. **To pin which of two same-typed sources a payload field reads,** set the OTHER source to a different value in the fixture (Customer `originating_club_id = 999` vs triggering Profile `club_id = 3` → assert `club_id => 3` proves the payload reads the triggering Profile's club, not the Customer's OC).
