# Tasks — operator-console-catalog-master

Read `proposal.md`, `design.md` (L1–L10), the delta spec `specs/operator-console/spec.md`, and the ADR `decisions/2026-06-19-operator-console-read-binding-write-through-actions.md` before any task. **Standing rules per task:** the five Quality Commands green at the end (format · test_filter · test · type_check · lint); `openspec validate operator-console-catalog-master --strict` green; **no protected files touched** (`spec/**`, `openspec/specs/**`, `CLAUDE.md`, `RALPH.md`, `ralph.sh`, `.claude/**` — tests under `tests/` are NOT protected); **no new domain logic** — this change adds **zero** Catalog Actions/events/migrations and **no Composer dependency** (`git diff main -- composer.json composer.lock` stays empty; no `database/migrations/*` added).

**The one law (ADR 2026-06-19): the console READS module models and WRITES only through module domain actions — never `$model->save()`.** Resources bind **read-only** to `\App\Modules\Catalog\Models\ProductMaster`; the producer display/picker reads `\App\Modules\Catalog\Models\ProducerState` (`catalog_producer_states`) — Catalog's **own** projection, never Module K. Every mutation is `app(<CatalogAction>::class)->handle(...)`. Task 1.2 builds the architecture test that enforces this; it is built **first** so it guards every later task.

**Verify every Filament/framework API in `vendor/` before writing it** (lessons.md) — in particular the Filament 5 `Resource` / `CreateRecord::handleRecordCreation(array $data): Model` / `Filament\Actions\Action`/`->action()`/`->form()`/`->requiresConfirmation()` / `Filament\Notifications\Notification` / infolist + table builder APIs, and the Livewire/Filament **test helpers** (`Filament\Actions\Testing`, `livewire()`/`Livewire::test`, `callAction`, `assertHasFormErrors`). Use `actingAs($operator, 'operator')` (the `operator` guard) to authenticate; `Operator` is `\App\Modules\OperatorPanel\Models\Operator`.

**Exact shipped Catalog actions this console invokes (signatures verified in code — do not invent or rename; re-confirm in `app/Modules/Catalog/Actions/` before use):**
- `CreateProductMaster::handle(string $name, int $producerId, string $appellation, string $region, ?TranslatableText $wineryStory = null, string $productType = 'wine'): ProductMaster`
- `SubmitProductMasterForReview::handle(ProductMaster $master): ProductMaster` — `draft → reviewed`, **audit-only (no domain event)**
- `RejectProductMasterReview::handle(ProductMaster $master, string $notes): ProductMaster` — stays `reviewed`, **audit-only**
- `ActivateProductMaster::handle(ProductMaster $master): ProductMaster` — `reviewed → active`; subject to `ApprovalGovernance` (distinct-actor floor) + `ProducerActivationGate`; records `ProductMasterActivated`
- `RetireProductMaster::handle(ProductMaster $master): ProductMaster` — `active → retired`, preserves active children; records `ProductMasterRetired`
- `ReopenProductMaster::handle(ProductMaster $master): ProductMaster` — `retired → reviewed`, **audit-only**
- `RetireProductMasterCascade::handle(ProductMaster $master): ProductMaster` — Master + active descendants, parent-before-child

**Domain-enforced, console-surfaced (never reimplemented — design L5/L6):** the Creator→Reviewer→Approver / no-self-approval floor lives in `ApprovalGovernance`; the Producer-active gate in `ProducerActivationGate` (reads `ProducerState`). The console calls the action and renders any thrown domain exception as a Filament **danger notification**; it does not re-check actor identity or producer state itself.

**Event names (verbatim, product-catalog spec):** `ProductMasterCreated`, `ProductMasterActivated`, `ProductMasterRetired`. Submit / reject / reopen record **no** domain event (audit-only). Coin **no** new event. Every recorded write carries `actor_role: newco_ops` + the operator id via `ActorContext` — assert it.

**i18n (design L8):** new copy in a new `lang/en/operator_console.php` + `lang/it/operator_console.php` group; reuse `lang/{en,it}/catalog.php` for domain rejection copy; EN baseline + per-key IT fallback (DEC-127); **no hardcoded user-facing string** in any Filament class.

**PG17 (knowledge/testing/rules.md):** every test that drives a DB-touching action is verified on **PostgreSQL 17** before the task is marked done — SQLite-green is necessary, never sufficient. Record the PG run in `progress.md` at close.

**Scope guard (proposal slice-boundary):** ProductMaster **only** — NO other spine entity resource, NO bulk import, NO enrichment-metadata update, NO Edit page / field-editing, NO `DeleteAction`, NO authority-tier RBAC, NO Parties/Producer/Consumer surface, NO shared base-resource abstraction (design L9). Build the one `ProductMasterResource` and its pages/actions, nothing more.
