(consolidated reusable patterns ŌĆö read first each iteration)

- **Parties state-enum idiom (sibling templates `KycStatus`/`HoldType`):** a `string`-backed `enum` in `app/Modules/Parties/Enums/`, **no `declare(strict_types=1)`** (no sibling has it), case names PascalCase, backing values the lowercase persisted token. A class-level docblock cites the PRD ┬¦ + the verbatim FSM. Domain rules live as **one-line predicate methods** (`clears()`, `autoLiftable()`, here `isActive()`/`isTerminal()`) each with its own docblock ŌĆö these are the readable from-state guards the writer Actions call. Test in a domain-grouped `tests/Unit/Modules/Parties/Enums/{Domain}EnumsTest.php` (Pest `it()` style) asserting the caseŌåÆvalue map **verbatim + order-sensitive via `toBe([...])`** + `toHaveCount(n)`, a `from()` round-trip, the predicate truth tables, and `from('bogus')->toThrow(ValueError::class)`.
- **Full-suite invocation ŌĆö use `php -d memory_limit=-1 vendor/bin/pest`, NOT `php -d memory_limit=-1 artisan test`.** The `artisan test` wrapper (laravel/pao) spawns the real test process *without* inheriting the outer `-d memory_limit=-1`, so it OOMs at the 128 MB default (fatal in `filament/.../routes/web.php` during `setUp()`). Invoking `vendor/bin/pest` directly keeps the run in one unlimited-memory process. Quality cmd `php artisan test` (root CLAUDE.md) is nominal; this is the memory-safe equivalent.
- **Parties migration idiom (templates `2026_06_15_000003_create_parties_clubs_table.php` / `ŌĆ”000007_create_parties_profiles_table.php`):** rich class docblock citing the PRD ┬¦/design; `Schema::create` with within-module `foreignId(...)->constrained(table: 'ŌĆ”', indexName: '<short>_fk')` (+ explicit `->restrictOnDelete()` for a shared parent); Money via two columns `{key}_minor` (`integer`) + `{key}_currency` (`string(3)`) ŌĆö never a float; `timestampTz`/`timestampsTz` for datetimes; a value-set `state`/`status` column as `string` + the enum cast, with a **PG-only CHECK** (`if (DB::getDriverName() === 'pgsql')`) whose accepted set derives from `Enum::cases()` so it can't drift; any **partial unique index** via raw `DB::statement('CREATE UNIQUE INDEX <name> ON <t> (<cols>) WHERE <pred>')` run on **BOTH** engines (NOT driver-guarded ŌĆö identical SQLite/PG syntax), with the predicate literal derived from the enum (`Enum::Case->value`). Index names < 63 chars (PG limit). `down()` = bare `Schema::dropIfExists`.
- **Raw-insert schema test for a migration whose model lands later (template `tests/Feature/Platform/EventDeliveriesSchemaTest.php`):** name it `{Entity}SchemaTest.php`, `uses(RefreshDatabase::class)`, a top-level `function {entity}Row(int $parentId, array $overrides = [])` helper (UNIQUELY named ŌĆö top-level test funcs are global, collide across files). Prove: `Schema::hasColumns`; **partial unique index by NAME** via `Schema::hasIndex($t, '<index_name>')` (getIndexes() exposes name+cols but NOT the predicate on either engine, so name is the portable proof ŌĆö a bad predicate would abort the migration and red the file); constraint rejection via `expect(fn () => DB::transaction(fn () => DB::table($t)->insert(...)))->toThrow(QueryException::class)` (the `DB::transaction` wrap = SAVEPOINT, so PG's transaction-abort stays isolated and a post-throw count check is valid); a NOT-NULL/no-default floor by `unset($row['col'])` then expecting a throw; FK orphan rejection with a non-existent parent id (SQLite enforces FKs here).
- **Don't down-test a Laravel migration.** The base `Illuminate\Database\Migrations\Migration` declares **no** `up()`/`down()` (the migrator calls them by reflection), so a `require`d migration is `mixed` ŌåÆ `instanceof Migration` narrows to a type that *still* lacks `up()`/`down()` ŌåÆ PHPStan max reds either way (`method.nonObject`, then `method.notFound`). No `parties_*` sibling down-tests; `down()` is a trivial `Schema::dropIfExists` (drops the table and, with it, every dependent index). Assert the forward/up state only; treat `down()` correctness as structural. Cost me two PHPStan cycles before reverting.
- **Scoped within-module `hasOne` relation + nullable-accessor test idiom (1.4):** a state-filtered single-child relation is `return $this->hasOne(Related::class)->where('col', Enum::Case->value);` with declared return `HasOne` + docblock `@return HasOne<Related, $this>` ŌĆö larastan 3.x preserves the relation generic THROUGH the `->where()` `__call` forward (no precedent needed; PHPStan max stays 0). Because a scoped `hasOne` genuinely resolves to `null`, the class `@property-read` is `Related|null $rel` (contrast a required `belongsTo`, which larastan types non-null, so `ProfileTest` can chain `->customer->is(...)` directly). **The null type then bites the TEST:** Pest's `expect($rel)->not->toBeNull()` does NOT narrow `$rel` for PHPStan max, so a following `$rel->method()`/`$rel->prop` reds with `method.nonObject`/`property.nonObject`. Use the suite-wide `?->` idiom (`$rel?->is($x)`, `$rel?->state`) for the chained accesses ŌĆö it's analyzer-safe AND stays strict (the explicit `->not->toBeNull()` is the behavioral check; a surprise null makes `?->ŌĆ”` yield null and the following `toBeTrue()`/`toBe()` fail).
