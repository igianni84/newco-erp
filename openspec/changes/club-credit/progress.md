## Codebase Patterns
(consolidated reusable patterns — read first each iteration)

- **Parties state-enum idiom (sibling templates `KycStatus`/`HoldType`):** a `string`-backed `enum` in `app/Modules/Parties/Enums/`, **no `declare(strict_types=1)`** (no sibling has it), case names PascalCase, backing values the lowercase persisted token. A class-level docblock cites the PRD § + the verbatim FSM. Domain rules live as **one-line predicate methods** (`clears()`, `autoLiftable()`, here `isActive()`/`isTerminal()`) each with its own docblock — these are the readable from-state guards the writer Actions call. Test in a domain-grouped `tests/Unit/Modules/Parties/Enums/{Domain}EnumsTest.php` (Pest `it()` style) asserting the case→value map **verbatim + order-sensitive via `toBe([...])`** + `toHaveCount(n)`, a `from()` round-trip, the predicate truth tables, and `from('bogus')->toThrow(ValueError::class)`.
- **Full-suite invocation — use `php -d memory_limit=-1 vendor/bin/pest`, NOT `php -d memory_limit=-1 artisan test`.** The `artisan test` wrapper (laravel/pao) spawns the real test process *without* inheriting the outer `-d memory_limit=-1`, so it OOMs at the 128 MB default (fatal in `filament/.../routes/web.php` during `setUp()`). Invoking `vendor/bin/pest` directly keeps the run in one unlimited-memory process. Quality cmd `php artisan test` (root CLAUDE.md) is nominal; this is the memory-safe equivalent.
- **Parties migration idiom (templates `2026_06_15_000003_create_parties_clubs_table.php` / `…000007_create_parties_profiles_table.php`):** rich class docblock citing the PRD §/design; `Schema::create` with within-module `foreignId(...)->constrained(table: '…', indexName: '<short>_fk')` (+ explicit `->restrictOnDelete()` for a shared parent); Money via two columns `{key}_minor` (`integer`) + `{key}_currency` (`string(3)`) — never a float; `timestampTz`/`timestampsTz` for datetimes; a value-set `state`/`status` column as `string` + the enum cast, with a **PG-only CHECK** (`if (DB::getDriverName() === 'pgsql')`) whose accepted set derives from `Enum::cases()` so it can't drift; any **partial unique index** via raw `DB::statement('CREATE UNIQUE INDEX <name> ON <t> (<cols>) WHERE <pred>')` run on **BOTH** engines (NOT driver-guarded — identical SQLite/PG syntax), with the predicate literal derived from the enum (`Enum::Case->value`). Index names < 63 chars (PG limit). `down()` = bare `Schema::dropIfExists`.
- **Raw-insert schema test for a migration whose model lands later (template `tests/Feature/Platform/EventDeliveriesSchemaTest.php`):** name it `{Entity}SchemaTest.php`, `uses(RefreshDatabase::class)`, a top-level `function {entity}Row(int $parentId, array $overrides = [])` helper (UNIQUELY named — top-level test funcs are global, collide across files). Prove: `Schema::hasColumns`; **partial unique index by NAME** via `Schema::hasIndex($t, '<index_name>')` (getIndexes() exposes name+cols but NOT the predicate on either engine, so name is the portable proof — a bad predicate would abort the migration and red the file); constraint rejection via `expect(fn () => DB::transaction(fn () => DB::table($t)->insert(...)))->toThrow(QueryException::class)` (the `DB::transaction` wrap = SAVEPOINT, so PG's transaction-abort stays isolated and a post-throw count check is valid); a NOT-NULL/no-default floor by `unset($row['col'])` then expecting a throw; FK orphan rejection with a non-existent parent id (SQLite enforces FKs here).
- **Don't down-test a Laravel migration.** The base `Illuminate\Database\Migrations\Migration` declares **no** `up()`/`down()` (the migrator calls them by reflection), so a `require`d migration is `mixed` → `instanceof Migration` narrows to a type that *still* lacks `up()`/`down()` → PHPStan max reds either way (`method.nonObject`, then `method.notFound`). No `parties_*` sibling down-tests; `down()` is a trivial `Schema::dropIfExists` (drops the table and, with it, every dependent index). Assert the forward/up state only; treat `down()` correctness as structural. Cost me two PHPStan cycles before reverting.
