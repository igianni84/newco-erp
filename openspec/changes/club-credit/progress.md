- **Parties state-enum idiom (sibling templates `KycStatus`/`HoldType`):** a `string`-backed `enum` in `app/Modules/Parties/Enums/`, **no `declare(strict_types=1)`** (no sibling has it), case names PascalCase, backing values the lowercase persisted token. Domain rules live as **one-line predicate methods** (`clears()`, `autoLiftable()`, here `isActive()`/`isTerminal()`) each with its own docblock вЂ” these are the readable from-state guards the writer Actions call. Test in a domain-grouped `tests/Unit/Modules/Parties/Enums/{Domain}EnumsTest.php` (Pest `it()` style) asserting the caseв†’value map **verbatim + order-sensitive via `toBe([...])`** + `toHaveCount(n)`, a `from()` round-trip, the predicate truth tables, and `from('bogus')->toThrow(ValueError::class)`.
- **Full-suite invocation вЂ” use `php -d memory_limit=-1 vendor/bin/pest`, NOT `php -d memory_limit=-1 artisan test`.** The `artisan test` wrapper (laravel/pao) spawns the real test process *without* inheriting the outer `-d memory_limit=-1`, so it OOMs at the 128 MB default (fatal in `filament/.../routes/web.php` during `setUp()`). Invoking `vendor/bin/pest` directly keeps the run in one unlimited-memory process. Quality cmd `php artisan test` (root CLAUDE.md) is nominal; this is the memory-safe equivalent.
- **Parties migration idiom (templates `2026_06_15_000003_create_parties_clubs_table.php` / `вЂ¦000007_create_parties_profiles_table.php`):** rich class docblock citing the PRD В§/design; `Schema::create` with within-module `foreignId(...)->constrained(table: 'вЂ¦', indexName: '<short>_fk')` (+ explicit `->restrictOnDelete()` for a shared parent); Money via two columns `{key}_minor` (`integer`) + `{key}_currency` (`string(3)`) вЂ” never a float; `timestampTz`/`timestampsTz` for datetimes; a value-set `state`/`status` column as `string` + the enum cast, with a **PG-only CHECK** (`if (DB::getDriverName() === 'pgsql')`) whose accepted set derives from `Enum::cases()` so it can't drift; any **partial unique index** via raw `DB::statement('CREATE UNIQUE INDEX <name> ON <t> (<cols>) WHERE <pred>')` run on **BOTH** engines (NOT driver-guarded вЂ” identical SQLite/PG syntax), with the predicate literal derived from the enum (`Enum::Case->value`). Index names < 63 chars (PG limit). `down()` = bare `Schema::dropIfExists`.
- **Raw-insert schema test for a migration whose model lands later (template `tests/Feature/Platform/EventDeliveriesSchemaTest.php`):** name it `{Entity}SchemaTest.php`, `uses(RefreshDatabase::class)`, a top-level `function {entity}Row(int $parentId, array $overrides = [])` helper (UNIQUELY named вЂ” top-level test funcs are global, collide across files). Prove: `Schema::hasColumns`; **partial unique index by NAME** via `Schema::hasIndex($t, '<index_name>')` (getIndexes() exposes name+cols but NOT the predicate on either engine, so name is the portable proof вЂ” a bad predicate would abort the migration and red the file); constraint rejection via `expect(fn () => DB::transaction(fn () => DB::table($t)->insert(...)))->toThrow(QueryException::class)` (the `DB::transaction` wrap = SAVEPOINT, so PG's transaction-abort stays isolated and a post-throw count check is valid); a NOT-NULL/no-default floor by `unset($row['col'])` then expecting a throw; FK orphan rejection with a non-existent parent id (SQLite enforces FKs here).
- **Don't down-test a Laravel migration.** The base `Illuminate\Database\Migrations\Migration` declares **no** `up()`/`down()` (the migrator calls them by reflection), so a `require`d migration is `mixed` в†’ `instanceof Migration` narrows to a type that *still* lacks `up()`/`down()` в†’ PHPStan max reds either way (`method.nonObject`, then `method.notFound`). No `parties_*` sibling down-tests; `down()` is a trivial `Schema::dropIfExists` (drops the table and, with it, every dependent index). Assert the forward/up state only; treat `down()` correctness as structural. Cost me two PHPStan cycles before reverting.
- **Scoped within-module `hasOne` relation + nullable-accessor test idiom (1.4):** a state-filtered single-child relation is `return $this->hasOne(Related::class)->where('col', Enum::Case->value);` with declared return `HasOne` + docblock `@return HasOne<Related, $this>` вЂ” larastan 3.x preserves the relation generic THROUGH the `->where()` `__call` forward (no precedent needed; PHPStan max stays 0). Because a scoped `hasOne` genuinely resolves to `null`, the class `@property-read` is `Related|null $rel` (contrast a required `belongsTo`, which larastan types non-null, so `ProfileTest` can chain `->customer->is(...)` directly). **The null type then bites the TEST:** Pest's `expect($rel)->not->toBeNull()` does NOT narrow `$rel` for PHPStan max, so a following `$rel->method()`/`$rel->prop` reds with `method.nonObject`/`property.nonObject`. Use the suite-wide `?->` idiom (`$rel?->is($x)`, `$rel?->state`) for the chained accesses вЂ” it's analyzer-safe AND stays strict (the explicit `->not->toBeNull()` is the behavioral check; a surprise null makes `?->вЂ¦` yield null and the following `toBeTrue()`/`toBe()` fail).
- **A NEW non-`Create` Parties Action MUST be registered in `SupplyLifecycleChainTest`'s scope-guard whitelist (NEW, 2.1):** `tests/Feature/Modules/Parties/SupplyLifecycleChainTest.php` runs an EXHAUSTIVE `expect($transitions)->toEqualCanonicalizing([...])` over every Action whose class name does NOT start with `Create` (`! str_starts_with($name, 'Create')`, globbed from the filesystem). So any new Action named `Issue*`/`Apply*`/`Forfeit*`/`Restore*` (anything non-`Create*`) lands in that set and **reds the exact-match** until added to one of the `$вЂ¦Transitions` arrays + spread into the assertion. Add ONLY the Actions that EXIST this iteration (the glob is filesystem-truth вЂ” pre-listing a not-yet-created sibling over-specifies and fails the other way). I added a `$clubCreditWriters = ['IssueClubCredit']` array; **the next Club Credit Action tasks each append their writer here** вЂ” 3.1 `ApplyClubCredit`, 4.1 `ForfeitClubCredit`, 4.2 `RestoreClubCredit`. (Design L9's grep checked only the FORBIDDEN-name lists and missed this allow-list, so this is routine guard-test maintenance, not a realignment вЂ” and it's the SAME maintenance every prior Action slice did, per that test's own comments.)
- **`Club.fee` is `Money|null` в†’ never pass it straight to `Money::equals()`/`minus()` in a test (NEW, 2.1):** `@property Money|null $fee` means `$club->fee` / `$profile->club->fee` is nullable, and PHPStan max reds `argument.type` when it reaches a `Money`-typed param (and `expect()->not->toBeNull()` does NOT narrow вЂ” the suite-wide trap). For issuance/redemption tests, build an explicit non-null `$fee = Money::of(вЂ¦)` LOCAL, pass it to `Club::factory()->create(['fee' => $fee])`, and assert against that local вЂ” it both satisfies the analyzer AND makes "amount = THE Club's fee" provable (you gave the Club that exact fee). The `IssueClubCredit` Action narrows the same nullable via a control-flow `if ($fee === null) throw вЂ¦` (the fee-null guard) before the create, so `$fee` is `Money` at the call site вЂ” no cast/assert needed.
