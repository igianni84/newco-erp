- **The non-catalog console recipe (§1 confirmed) — reuse the kit where it is lifecycle-agnostic, diverge where the catalog FSM is encoded (ADR 2026-06-20).** `ProducerResource extends OperatorConsoleResource` for the label helpers (off `static i18nKey() => 'producer'`) + `versionColumn()`, but does **NOT** call `lifecycleStateColumn()` — Producer's state lives in `status` (a `ProducerStatus`) plus a separate nullable `kyc_status` (`KycStatus|null`), so the resource supplies its **own** `statusColumn()` + `kycStatusColumn()` (private static helpers) rendering the enum via `getStateUsing` + `instanceof BackedEnum ? (string) $state->value : ''` — type-clean at phpstan-max, **no `Parties\Enums` import** (the nullable kyc column renders `''` for NULL). List = name · region · country · status · kyc_status · `versionColumn()`; infolist = identity attrs + the translatable `description` + the operated-Clubs read. `ListProducers` = the header create-LINK page (copy `ListFormats`, NOT a `CreateAction`). The OperatorPanel import surface stays exactly **{`Parties\Models\Producer`}** on the Resource (the Create page adds `Parties\Actions\CreateProducer`).
- **`getPages()` coupling gotcha (drove the §1 slice shape).** A Filament Resource's `getPages()` eagerly references each page class (`Pages\CreateProducer::route()` autoloads the class), and the panel calls `getPages()` at **registration** — so the resource cannot boot until **all three** page classes (`ListProducers`, `CreateProducer`, `ViewProducer`) exist. The §1 test pins `getPages()` keys `== ['index','create','view']`. Therefore the read-surface section (§1) must **also scaffold** `CreateProducer` + `ViewProducer`, or the whole operator panel fails to boot and every Filament test reds. §1 created `CreateProducer` with its **real** `createViaAction` (the form on the Resource + the create test land in §2) and `ViewProducer` as a **bare read-only `ViewRecord`** (the `SurfacesDomainActions` trait + the activate/retire + KYC header actions land in §3/§4). `discoverResources` scans `Filament/Resources` recursively, so the new `Resources/Parties/` sub-namespace is auto-discovered with no panel-provider change.
- **Operated-Clubs read with NO `Club` import:** `fn (Producer $record): string => $record->clubs->pluck('display_name')->implode(', ')` — keeps the resource's cross-module surface at exactly `{Producer}`; phpstan-clean (`Collection<int,Club>::pluck → Collection<int,mixed>::implode → string`). The translatable description renders `fn (Producer $record): ?string => $record->description?->resolve(app()->getLocale())` (mirror catalog `tasting_notes`).
- **Create write-through (§1+§2 confirmed):** `CreateProducer extends OperatorConsoleCreateRecord`; page/action class-name collision → `use App\Modules\Parties\Actions\CreateProducer as CreateProducerAction` (D6). `createViaAction` narrows `array<string,mixed>` (string guards → `InvalidArgumentException` = LogicException, propagates past the base's `RuntimeException` catch), wraps `description` as `TranslatableText::of(['en' => $description])` when present else `null`, calls `app(CreateProducerAction::class)->handle(name, region, country, appellation, description, website)`. `createRejectionField() => 'name'` is the inert safety net (Producer ships no create guard — D6). The Parties Actions take args (not the model), unlike catalog. **The create FORM lives on `ProducerResource::form()`, NOT the page** — Filament's `CreateRecord` inherits the Resource form (mirrors catalog `FormatResource`); the page supplies only `createViaAction`/`createRejectionField`. The form is `name`/`region`/`country` required + `appellation`/`website` optional `TextInput` + `description` optional `Textarea`, all labelled `operator_console.producer.fields.*`. It exposes **neither `status` nor `kyc_status`** — both FSMs are born by the action (`draft` / NULL) and advance only through the view-page actions (D6). Tested in `ProducerCreateConsoleTest` (3 tests): valid submit → `draft`+NULL+1 PII-free `ProducerCreated` (payload keys are exactly the structural set — no party email/phone), `assertFormFieldDoesNotExist('status'|'kyc_status')`, two same-name Producers both succeed.
- **i18n incremental fill:** the `operator_console.producer.*` block grows per section. §1 added `label`/`plural_label` (EN; IT omits → per-key EN fallback), `columns.{name,region,country,status,kyc_status,version}`, `fields.{appellation,website,description,clubs}`, `actions.create`. §2 added `fields.{name,region,country}` (the required create inputs — re-using the same words as `columns.*`, like catalog). §3/§4 add `actions.{activate,retire,require_kyc,…}` + `notifications.*`. The §5 kit-key completeness test will then enumerate the full contract.
