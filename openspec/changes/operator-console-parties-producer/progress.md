The §1 test pins `getPages()` keys `== ['index','create','view']`. Therefore the read-surface section (§1) must **also scaffold** `CreateProducer` + `ViewProducer`, or the whole operator panel fails to boot and every Filament test reds. §1 created `CreateProducer` with its **real** `createViaAction` (the form on the Resource + the create test land in §2) and `ViewProducer` as a **bare read-only `ViewRecord`** (the `SurfacesDomainActions` trait + the activate/retire + KYC header actions land in §3/§4). `discoverResources` scans `Filament/Resources` recursively, so the new `Resources/Parties/` sub-namespace is auto-discovered with no panel-provider change.
- **Operated-Clubs read with NO `Club` import:** `fn (Producer $record): string => $record->clubs->pluck('display_name')->implode(', ')` — keeps the resource's cross-module surface at exactly `{Producer}`; phpstan-clean (`Collection<int,Club>::pluck → Collection<int,mixed>::implode → string`). The translatable description renders `fn (Producer $record): ?string => $record->description?->resolve(app()->getLocale())` (mirror catalog `tasting_notes`).
- **Create write-through (real in §1, tested in §2):** `CreateProducer extends OperatorConsoleCreateRecord`; page/action class-name collision → `use App\Modules\Parties\Actions\CreateProducer as CreateProducerAction` (D6). `createViaAction` narrows `array<string,mixed>` (string guards → `InvalidArgumentException` = LogicException, propagates past the base's `RuntimeException` catch), wraps `description` as `TranslatableText::of(['en' => $description])` when present else `null`, calls `app(CreateProducerAction::class)->handle(name, region, country, appellation, description, website)`. `createRejectionField() => 'name'` is the inert safety net (Producer ships no create guard — D6). The Parties Actions take args (not the model), unlike catalog.
- **i18n incremental fill:** the `operator_console.producer.*` block grows per section. §1 added `label`/`plural_label` (EN; IT omits → per-key EN fallback), `columns.{name,region,country,status,kyc_status,version}`, `fields.{appellation,website,description,clubs}`, `actions.create`. §2 adds the create-form field labels; §3/§4 add `actions.{activate,retire,require_kyc,…}` + `notifications.*`.
