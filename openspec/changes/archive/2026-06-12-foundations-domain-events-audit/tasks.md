## 4. Delivery Execution

- [x] 4.1 `App\Platform\Events\InlineDeliveryExecutor` + post-commit wiring (design D5): `deliver(array $domainEventIds)` loads `pending` due deliveries ordered by `domain_event_id`, resolves each consumer from the container, runs handler + status flip to `done` (attempts+1) **inside one DB transaction per delivery**; on throw — rollback handler txn, record failure separately (attempts+1, `available_at = now + backoff(attempts)`, `last_error`; `failed` at max attempts), continue with the next delivery. Wire the recorder: ONE `DB::afterCommit` hook per transaction handing recorded ids to the executor (verify `Illuminate\Foundation\Testing\DatabaseTransactionsManager` afterCommit-under-RefreshDatabase semantics in vendor FIRST; fall back per design D5 if they differ, recording the resolution in `progress.md`).
  - Acceptance: delta scenarios proven — inline happy path (commit → handler ran → `done`, attempts 1); R4 fan-out isolation (two consumers, first throws: its row in retry state with `last_error`, sibling `done`, emitter's committed rows intact); done-is-terminal (re-running the executor over a `done` row does not re-invoke the handler); causal order (three events one transaction → consumer invoked in `id` order); exactly-once for DB effects (handler's write and its `done` flip commit together — assert both present after, or neither mid-failure); happy + failure tested; quality commands green.
  - Test hint: `tests/Feature/Platform/InlineDeliveryTest.php`; test consumers as invokable classes writing marker rows to `cache` (DB-work-only rule) + a throwing consumer; invocation order captured in a static/array sink; if the afterCommit hook proves untestable under `RefreshDatabase`, call `deliver()` explicitly in these tests and cover the hook in one targeted non-wrapped test per design D5.
- [x] 4.2 `php artisan events:sweep` + schedule + tunables (design D6): command selects due deliveries (`pending` AND `available_at` null-or-past) ordered `(consumer, domain_event_id)` and executes through the 4.1 path; `config/events.php` with `sweep.max_attempts` (5), `sweep.backoff_base_seconds` (30, exponential ×2), `sweep.backoff_cap_seconds` (3600); schedule entry in `routes/console.php` at `everyThirtySeconds()` with `withoutOverlapping()` (verify the Laravel 13 scheduling + sub-minute idiom in vendor; commands registered per design D1's note).
