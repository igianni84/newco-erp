# Proposal — foundations-money-i18n-flags

## Why

This is the third and final F1 foundations change (F1 3/3) — the Phase-1 platform primitives the prior two F1 changes explicitly deferred here. The skeleton change (F1 1/3) and the domain-events-audit change (F1 2/3) both name `foundations-money-i18n-flags` as their successor, and the substrate code already forward-references it: `DomainEventRecorder` documents that "a float in an FX field is a caller bug **the F1 3/3 value objects will make unrepresentable**" — but those value objects do not yet exist (`app/Platform/Money` is absent, `lang/` is absent, Pennant is not installed). A 2026-06-13 audit found this change was skipped with no recorded decision; authoring it now IS that decision (descoping was the alternative — rejected: see Impact).

Build Workplan Phase 1 (`spec/05-release/Build_Workplan_v0.3-MVP.md` § 2, "Key decisions to make") lists exactly these four concerns as **Phase-1 platform concerns**, not module work: the multi-currency dual-record schema pattern (D18 FLOOR, "full pattern"), the i18n schema pattern (D2, six-locale-ready), **(i)** the feature-flag infrastructure that gates NFT/on-chain (D12), and **(iii)** the `actor_role` audit envelope. F2 (Module 0 Catalog + Module K Parties) depends on all of them: Catalog is i18n-keyed (DEC-064), Module K's Club Credit is a monetary entity, and both emit `actor_role`-stamped events — building F2 without these primitives would force each module to reinvent them ad-hoc, eroding the boundary. This change decides nothing the spec or an existing ADR has not already decided.

## What Changes

- **Money value objects** (`App\Platform\Money`, owned code, no new dependency): `Currency` (ISO 4217 code + minor-unit exponent; launch set EUR-base/USD/GBP/CHF/JPY per DEC-037, JPY exponent 0, fail-closed on unknown codes), `Money` (immutable integer minor units + `Currency`; no float construction path; minor-units arithmetic with a same-currency guard; persists/rehydrates without precision loss), `FxRate` (exact decimal string, never a float), and `DualCurrencyAmount` (the D18 dual-record *representation*: customer-currency `Money` + EUR-equivalent `Money` + locked `FxRate` + rate timestamp, serialising to the DEC-169 payload shape). These make the substrate's documented payload discipline — minor-units money, decimal-string FX — *unrepresentable to violate*, fulfilling the F1 2/3 forward-reference.
- **i18n foundation** (`App\Platform\I18n` + `lang/` + `config/i18n.php`): a single-source-of-truth supported-locale registry of the six launch locales (`en, it, fr, de, ja, zh_Hans`) with English as the final fallback; `lang/` translation-file scaffolding for all six (English authored, others MAY stagger per AC-0-XM-4); a reusable `TranslatableText` value object + Eloquent cast for the DEC-064 "i18n-keyed JSON per attribute" pattern with per-attribute English fallback (DEC-127 item 4) and application-layer locale validation; and remediation of `resources/views/welcome.blade.php` (debt S3) into a minimal localized placeholder carrying no hardcoded user-facing copy.
- **Feature-flag infrastructure** (Laravel Pennant — first-party, latest stable `^1.23`): install + config + the `features` table migration; the re-scoped **EXT-1** NFT/on-chain feature flag defined **default OFF** (gating mint/burn + custodial wallet + on-chain recovery + Bottle-Page chain-link — NOT the launch-ready serialization workflow); a reusable accessor; and the documented convention that the non-serialized (NS) path is the universal fallback when the flag is off.
- **Actor-context resolver seam** (`App\Platform\Events\ActorContext`): a reusable resolver supplying the `actor_role` (+ optional `actor_id`) for the current execution context, so emitters obtain "who is acting" from one canonical seam instead of hardcoding it. The `ActorRole` enum and the NOT-NULL envelope column already exist (F1 2/3); this adds only the resolver. It defaults to `system` and reads NO authentication state — the operator/producer/customer wiring is the identity/auth ADR's job (gate before Module K). This is carry (iii)'s "Admin-Panel arm" reduced to the one gate-safe seam buildable today.
- **Docs & forward-reference cleanup**: update `docs/event-substrate.md` and the `DomainEventRecorder` docblock from "the F1 3/3 value objects *will* make…" to present tense; refresh the `GUIDE.md` F1 status line; extend `CONTEXT.md` with the resolved glossary terms (Money, Currency, FX Rate, Dual-Currency Amount, Supported locale, Translatable text, Feature flag / EXT-1, Actor context).

## Capabilities

### New Capabilities

- `money`: the money discipline value objects — `Money` (integer minor units + ISO 4217 currency, float-unrepresentable), `Currency` (code + minor-unit exponent, launch set, fail-closed), `FxRate` (exact decimal string) and the `DualCurrencyAmount` dual-record representation (D18). The reusable primitives every monetary field and financial-event payload (Module K Club Credit, Module E events, Module S/A prices) is built from.
- `i18n`: the internationalisation foundation — the six-locale supported set with English fallback, the no-hardcoded-strings discipline (incl. the `welcome.blade.php` remediation), and the reusable i18n-keyed-JSON translatable-attribute primitive with per-attribute English fallback and application-layer locale validation.
- `feature-flags`: the feature-flag infrastructure (Laravel Pennant) and the NFT/on-chain EXT-1 flag (default OFF) that decouples the on-chain surfaces from the launch critical path, with the NS path as the universal fallback.

### Modified Capabilities

- `event-substrate`: gains one requirement — **Actor Context Resolution** — the reusable resolver that supplies `actor_role`/`actor_id` for the current context. No existing event-substrate requirement changes; the envelope already mandates `actor_role`, this adds the canonical way to populate it (system-default until the auth ADR).

## Impact

- **Code:** `app/Platform/Money/**` (new: `Currency`, `Money`, `FxRate`, `DualCurrencyAmount`, `MoneyCast`), `app/Platform/I18n/**` (new: `SupportedLocale`, `TranslatableText`, `TranslatableTextCast`), `app/Platform/Features/**` (new: the EXT-1 feature definition + accessor), `app/Platform/Events/ActorContext.php` (new), `config/i18n.php` + `config/pennant.php` (new/published), `lang/**` (new: six locales), `database/migrations/` (Pennant `features` table), `resources/views/welcome.blade.php` (remediated), `tests/Unit/Platform/**` + `tests/Feature/Platform/**` (new). The boundary arch test needs **no amendment** — `App\Platform` already covers these sub-namespaces by name-prefix.
- **Docs:** `docs/event-substrate.md` (forward-reference → present tense + the new VOs as the payload-building path), `docs/development.md` (Pennant installed version), `GUIDE.md` (F1 status line), `CONTEXT.md` (glossary terms), `docs/INDEX.md` if a new doc is added.
- **Dependencies:** **one added** — `laravel/pennant` `^1.23` (latest stable, verified resolvable against Laravel v13.15.0 on 2026-06-13; first-party Laravel package, framework-conventional, no ADR required per the stack ADR's posture). composer.json/composer.lock churn is limited to Pennant + its transitive needs. The PHP constraint is NOT bumped here (`composer.json` `php ^8.3`→`^8.5` is staged in `substrate-hardening`). The Money VOs add no dependency (owned code, consistent with the substrate ADR's no-new-heavyweight-deps posture; `brick/money`/`moneyphp` considered and rejected — our needs are minimal and a dep would need its own ADR).
- **Slice boundary — deliberately NOT in this change (declared future homes):**
  - **Operator/producer/customer actor resolution** (mapping an authenticated principal to `actor_role`) → the identity/auth ADR + Module K. `ActorContext` ships the seam returning `system`; it reads no auth state, so it does not step through the auth gate.
  - **The full request-locale resolution chain** (DEC-127: `Accept-Language` → sticky cookie → English, manual switcher, Bottle-Page render) → Module B / the consumer frontend. This change ships the supported-locale registry + the per-attribute fallback *primitive*, not the request-time render pipeline.
  - **The consumer/producer frontend stack** → the TanStack ADR (Module S gate). `welcome.blade.php` becomes a minimal localized placeholder; it decides no frontend stack and adds no routing.
  - **Module E FX policy** (snapshot time, buffer %, refresh cadence, per-leg lock moments — DEC-038/DEC-169) → Module E. `DualCurrencyAmount` is pure representation; it carries no policy.
  - **Flipping EXT-1 on** (the NFT/on-chain workstream) → gated on an external blockchain-expert review; the flag ships OFF.
  - **Any module entity, column or migration using these primitives** (Club Credit, Catalog translatable attributes, Offer/Allocation prices, the ~30 Module E financial-event payloads) → F2+ module changes. This change ships the primitives they will use, not their usage.
  - **The PHP `^8.3`→`^8.5` composer-constraint bump** → `substrate-hardening` (already staged; keeping it there avoids a merge collision).
- **Traceability gaps (deliberate):** of Workplan Phase 1's "Key decisions" list, this change closes the money pattern (D18), the i18n pattern (D2), the feature-flag infrastructure (D12, carry i) and the `actor_role` envelope's reusable seam (carry iii). The remaining Phase-1 items — observability, hosting/EU residency, the frontend platform shell, secrets management — stay with the gated ADRs (hosting → staging gate; frontend → Module S gate) exactly as the skeleton change recorded; they are not regressed here. The audit's stale "next: foundations-money-i18n-flags" pointers in the two archived proposals are left as historical record (they were correct when written); the *live* forward-references (GUIDE.md, docs/event-substrate.md, the code docblock) are updated to present tense by this change.
