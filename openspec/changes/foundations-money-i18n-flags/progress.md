# Progress — foundations-money-i18n-flags

## Codebase Patterns
(consolidated reusable patterns — read first each iteration)

- **Fixed-set value type → string-backed enum.** When a VO's domain is a closed set whose members carry derived data, realise it as a `string`-backed enum (the cases ARE the single-source-of-truth set; value-equality and `from`/`tryFrom` are native). Per-case data via an **exhaustive `match`** (no `default` → PHPStan-max-clean, and a new case fails loudly until handled). Used by `Currency` (code→`minorUnitExponent()`); reuse for `SupportedLocale` (2.1).
- **Fail-closed VO factory.** `public static function of(string $x): self { return self::tryFrom($x) ?? throw new InvalidArgumentException("…names the valid set + the one-line fix…"); }` — the PHP 8 throw-expression keeps it a one-liner. Exact match, no case-folding (a non-canonical input is a caller bug → reject). Money/FX/locale VOs all follow this.
- **No `declare(strict_types=1)`** anywhere in `app/Platform` — match the house style (ActorRole, DeliveryStatus, NotInTransactionException all omit it). Pint enforces the rest.
- **Exception choice:** SPL `InvalidArgumentException` for "bad argument to a VO factory" with an intentful message; custom `Runtime`/SPL subclass (cf. `NotInTransactionException`) only when a named domain guard recurs across call-sites.
- **Verbatim set pin** (mirror `tests/Unit/Platform/EnumsTest.php`): loop `$values[$case->name] = $case->value;` then `expect($values)->toBe([...])` — order-sensitive, so any case/value drift fails first. Use for every closed enum set.
- **Test placement:** pure VO/enum tests → `tests/Unit/Platform/<Sub>/…Test.php` (Pest binds the Laravel `TestCase` only `->in('Feature')`, so Unit tests need no app/DB — fine for pure logic). DB-touching tests (casts: `MoneyCast` 1.5, `TranslatableTextCast` 2.3, Pennant 3.x) live under `tests/Feature/Platform/…` and opt in per-file with `uses(RefreshDatabase::class)` (Pest.php leaves `RefreshDatabase` commented out globally).
- **Immutable composite VO** (the `Money` template — reuse for `FxRate` 1.3, `DualCurrencyAmount` 1.4): `private function __construct(public readonly … )` (promoted readonly props are the public read surface — the cast/consumers read `$m->minorUnits`, no accessor boilerplate) + a static `of(…)` factory as the SOLE construction path. Derivations/arithmetic return a **new** instance (never mutate); `equals(self)` is value equality (`===` works directly on enum legs — singletons); `toPayload()` returns the serialise shape with an `@return array{…}` shape annotation (PHPStan-max needs it so callers index keys typed). **Rehydrate by composing the public factories** — `Money::of($p['minor_units'], Currency::of($p['currency']))` — NOT a dedicated `fromPayload()` (would be dead code; the cast reads two columns, the recorder never rehydrates). Preconditions (same-currency, EUR-leg) are private `assert*()` guards throwing SPL `InvalidArgumentException` with an intentful message (mirrors `Currency::of()`). No `final` (house style), no `declare(strict_types=1)`.
- **Structural-invariant test via reflection** (the non-vacuous "no float path" proof): `new ReflectionMethod(Cls::class, 'of')->getParameters()[0]->getType()` → `expect(...)->toBeInstanceOf(ReflectionNamedType::class)` for the runtime guard, then `assert($type instanceof ReflectionNamedType)` to narrow `ReflectionType|null` for PHPStan-max, then `expect($type->getName())->toBe('int')->and($type->allowsNull())->toBeFalse()`. Also assert the constructor `isPrivate()` (so `of()` is the only path). `ReflectionMethod`/`ReflectionNamedType`/`InvalidArgumentException` are global — no `use` in a no-namespace Pest file. **`tests/` IS under PHPStan max** (`phpstan.neon` paths include `tests`) — reflection narrowing is mandatory.

## [2026-06-13 09:30] — 1.2 Money
- Implemented `App\Platform\Money\Money` — an immutable VO of `int $minorUnits` + `Currency`, composing the 1.1 enum. Private constructor + promoted `public readonly` props; static `of(int, Currency)` is the SOLE construction path (no float overload, no `fromMajorUnits` — YAGNI). `plus`/`minus`/`negate` return new instances on the integer minor units; `plus`/`minus` route through a private `assertSameCurrency()` guard (throws SPL `InvalidArgumentException` naming both currencies, pointing cross-currency math at DualCurrencyAmount/Module E). `equals()` value equality (minor units + currency). `toPayload()` → `['minor_units' => int, 'currency' => string]` with an `@return array{…}` shape annotation. Negatives valid (Club Credit/refunds).
- **Design call:** rehydration composes the public factories (`Money::of($p['minor_units'], Currency::of($p['currency'])`) — no dedicated `fromPayload()` (the round-trip test and the future `MoneyCast` both reconstruct via `of()`, and a `fromPayload` would be dead code: the cast reads two columns, the recorder persists the array verbatim and never rehydrates).
- Files changed: `app/Platform/Money/Money.php` (new), `tests/Unit/Platform/Money/MoneyTest.php` (new, 8 tests / 23 assertions).
- Quality loop: **green** — pint clean · MoneyTest 8/8 · full suite **166/166** (was 158, +8) · phpstan **0** @ max · pint `--test` clean · `openspec validate --strict` valid. No `composer.json`/`.lock` churn (`git diff main` empty — owned code).
- **Acceptance:** all five Money delta scenarios covered — no float path (reflection: `of()` first param is non-nullable `int`, constructor private — non-vacuous), minor-units round-trip, cross-currency `plus`/`minus` throw, payload shape (`toBe` + `toBeInt`/`toBeString`), negative representable with sign preserved — plus add/subtract/negate and value-equality happy paths.
- **Learnings for future iterations:**
  - `Money` is the template for `FxRate` (1.3) and `DualCurrencyAmount` (1.4) — see the new "Immutable composite VO" + "Structural-invariant test via reflection" Codebase Patterns at the top. `DualCurrencyAmount` will hold two `Money` legs + an `FxRate`; serialise its customer leg as `amount` and EUR leg as `eur_equivalent_amount` (DEC-169 keys) — it builds its own payload from `$money->minorUnits`/`$money->currency->value`, NOT by calling `Money::toPayload()` (different key names).

The readonly props are the cast's read surface — no accessor methods were added.
  - PHPStan max analyses `tests/`; any reflection on `getType()` must narrow `ReflectionType|null` (via `assert(... instanceof ReflectionNamedType)`) before `getName()`.
