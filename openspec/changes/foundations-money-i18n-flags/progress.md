# Progress ‚Äî foundations-money-i18n-flags

## Codebase Patterns
(consolidated reusable patterns ‚Äî read first each iteration)

- **Fixed-set value type ‚Üí string-backed enum.** When a VO's domain is a closed set whose members carry derived data, realise it as a `string`-backed enum (the cases ARE the single-source-of-truth set; value-equality and `from`/`tryFrom` are native). Per-case data via an **exhaustive `match`** (no `default` ‚Üí PHPStan-max-clean, and a new case fails loudly until handled). Used by `Currency` (code‚Üí`minorUnitExponent()`); reuse for `SupportedLocale` (2.1).
- **Fail-closed VO factory.** `public static function of(string $x): self { return self::tryFrom($x) ?? throw new InvalidArgumentException("‚Ä¶names the valid set + the one-line fix‚Ä¶"); }` ‚Äî the PHP 8 throw-expression keeps it a one-liner. Exact match, no case-folding (a non-canonical input is a caller bug ‚Üí reject). Money/FX/locale VOs all follow this.
- **No `declare(strict_types=1)`** anywhere in `app/Platform` ‚Äî match the house style (ActorRole, DeliveryStatus, NotInTransactionException all omit it). Pint enforces the rest.
- **Exception choice:** SPL `InvalidArgumentException` for "bad argument to a VO factory" with an intentful message; custom `Runtime`/SPL subclass (cf. `NotInTransactionException`) only when a named domain guard recurs across call-sites.
- **Verbatim set pin** (mirror `tests/Unit/Platform/EnumsTest.php`): loop `$values[$case->name] = $case->value;` then `expect($values)->toBe([...])` ‚Äî order-sensitive, so any case/value drift fails first. Use for every closed enum set.
- **Test placement:** pure VO/enum tests ‚Üí `tests/Unit/Platform/<Sub>/‚Ä¶Test.php` (Pest binds the Laravel `TestCase` only `->in('Feature')`, so Unit tests need no app/DB ‚Äî fine for pure logic). DB-touching tests (casts: `MoneyCast` 1.5, `TranslatableTextCast` 2.3, Pennant 3.x) live under `tests/Feature/Platform/‚Ä¶` and opt in per-file with `uses(RefreshDatabase::class)` (Pest.php leaves `RefreshDatabase` commented out globally).
- **Immutable composite VO** (the `Money` template ‚Äî reuse for `FxRate` 1.3, `DualCurrencyAmount` 1.4): `private function __construct(public readonly ‚Ä¶ )` (promoted readonly props are the public read surface ‚Äî the cast/consumers read `$m->minorUnits`, no accessor boilerplate) + a static `of(‚Ä¶)` factory as the SOLE construction path. Derivations/arithmetic return a **new** instance (never mutate); `equals(self)` is value equality (`===` works directly on enum legs ‚Äî singletons); `toPayload()` returns the serialise shape with an `@return array{‚Ä¶}` shape annotation (PHPStan-max needs it so callers index keys typed). **Rehydrate by composing the public factories** ‚Äî `Money::of($p['minor_units'], Currency::of($p['currency']))` ‚Äî NOT a dedicated `fromPayload()` (would be dead code; the cast reads two columns, the recorder never rehydrates). Preconditions (same-currency, EUR-leg) are private `assert*()` guards throwing SPL `InvalidArgumentException` with an intentful message (mirrors `Currency::of()`). No `final` (house style), no `declare(strict_types=1)`.
- **Structural-invariant test via reflection** (the non-vacuous "no float path" proof): `new ReflectionMethod(Cls::class, 'of')->getParameters()[0]->getType()` ‚Üí `expect(...)->toBeInstanceOf(ReflectionNamedType::class)` for the runtime guard, then `assert($type instanceof ReflectionNamedType)` to narrow `ReflectionType|null` for PHPStan-max, then `expect($type->getName())->toBe('int')->and($type->allowsNull())->toBeFalse()` (use `'string'` for FxRate). Also assert the constructor `isPrivate()` (so `of()` is the only path). `ReflectionMethod`/`ReflectionNamedType`/`InvalidArgumentException` are global ‚Äî no `use` in a no-namespace Pest file. **`tests/` IS under PHPStan max** (`phpstan.neon` paths include `tests`) ‚Äî reflection narrowing is mandatory.
- **String-validated scalar VO** (the `FxRate` variant of *Immutable composite VO* ‚Äî for a VO wrapping ONE scalar string that must round-trip bit-for-bit): private ctor + a single `public readonly string $value` + sole `of(string)` factory + `__toString()` returning `$value` (the `(string)` cast is the ergonomic read the test pins; `$value` is the structured read surface consumers index). Validate in `of()` against a class-const regex anchored `\A‚Ä¶\z` (**NOT** `^‚Ä¶$` ‚Äî `$` leniently matches before a trailing `\n`, so `"1.0842\n"` would wrongly pass) and test `preg_match($pat, $s) !== 1` (one comparison covers both the `0` no-match and the `false` error returns ‚Äî PHPStan-max clean); on failure throw SPL `InvalidArgumentException` with an intentful message naming the accepted grammar. **Store the accepted string verbatim ‚Äî never normalise** (no trim, no reformat, no canonicalising trailing zeros): that IS the bit-for-bit / no-precision-drift guarantee (invariant 5, refunds at the locked rate). Well-formedness only ‚Äî economic/semantic validity (positivity, bounds) stays OUT as downstream policy. FxRate grammar = `/\A\d+(\.\d+)?\z/` (non-negative plain decimal; no sign, exponent or whitespace).
- **Policy-free composite VO + purity guard** (the `DualCurrencyAmount` extension of *Immutable composite VO* ‚Äî for a bundle that sits next to a module boundary and must carry data only, never policy): (1) **build the payload from the parts' read surfaces, not a composed VO's `toPayload()`** ‚Äî the keys differ (`DualCurrencyAmount` emits `amount`/`eur_equivalent_amount` from `$money->minorUnits`; `Money::toPayload()` emits `minor_units`), so compose the shape by hand and pin it with an exact `@return array{‚Ä¶}` + an order-sensitive `->toBe([‚Ä¶])` test (DEC-169 key order). (2) **An immutable date/time leg is typed `DateTimeImmutable`** (a `Carbon\CarbonImmutable` literal satisfies it; a *mutable* `Carbon`/`DateTime` is a TypeError ‚Äî correct, it would break the immutable bundle) and serialised in **full ISO 8601** via `->format(DateTimeImmutable::ATOM)` ‚Äî do **NOT** truncate to `Y-m-d`: truncation silently imposes a snapshot-granularity policy the VO must not own (design D3 landmine). Mirror the reflection guard (param `[n]` type is `'DateTimeImmutable'`, non-nullable) to make immutability non-vacuous. (3) **Prove "carries no policy" structurally:** `$m = get_class_methods($vo::class); sort($m); expect($m)->toBe([...allow-list...])` ‚Äî from a test's (external) scope `get_class_methods` returns only PUBLIC methods (verified), so a private ctor/`assert*` guard is excluded and the allow-list stays the public surface (`['of','toPayload']`); any `deriveRate()/convert()/snapshot()/refresh()` would break it. A construction precondition (EUR-leg) is a private instance `assert*()` called **from the ctor body** (runs after promoted-prop binding) ‚Üí covers every construction path, stays off the public allow-list.
- **Two-column Eloquent cast** (the `MoneyCast` template ‚Äî reuse for `TranslatableTextCast` 2.3): a `CastsAttributes` mapping a VO across N real columns derived **by convention from the cast key** (`'price' => MoneyCast::class` ‚áí `price_minor` + `price_currency`; suffix the key, no per-column wiring). Verified vendor contract (Laravel 13 `HasAttributes`): `set()` returning an **array** `[col => value]` is spread into the model's attributes (`normalizeCastClassResponse` ‚Üí `array_replace`, lines 2029/1269) ‚Äî the multi-column write; the cast-key attribute itself is never persisted. `get()` receives the **full raw `$attributes`** array and reads its real columns from there (`$attributes["{$key}_minor"]`), so reload-from-DB (not the just-set model ‚Äî its `classCastCache` holds the live VO) is what exercises `get()`. **Generic: `@implements CastsAttributes<TVo, mixed>`** ‚Äî `TGet=TVo` so the owning model's property surfaces typed (`?TVo`), but **`TSet=mixed`** because Eloquent passes through whatever was assigned (it does not enforce the generic), so the `! $value instanceof TVo` runtime guard is load-bearing; typing `TSet=TVo` makes PHPStan flag that guard `instanceof.alwaysTrue`. Fail-closed on a non-VO `set()` and on a corrupt `get()` column (mirrors `Currency::of()`). **Reading `mixed` `$attributes` at PHPStan max:** `(int) $mixed`/`(string) $mixed` are forbidden (`cast.int`/`cast.string`) ‚Äî narrow with `is_numeric`/`is_string` first (their true-branch types cast cleanly), throwing on the else (corrupt money data must not be silently coerced ‚Äî invariant 6).
- **Enum-as-SSOT, config derives** (`SupportedLocale` 2.1 ‚Äî for any closed registry that needs both a typed anchor AND a `config()` surface): the **enum is the single source of truth** (cases are compile-time constants; config can't supply them), and `config/<x>.php` *derives* its array form from the enum (`'supported' => SupportedLocale::values()`, `'fallback' => SupportedLocale::fallback()->value`) rather than restating the set ‚Äî so the two can't drift. Add a `values(): list<string>` helper = `array_map(static fn (self $c): string => $c->value, self::cases())` (PHPStan-max infers `list<string>` cleanly; annotate `@return list<string>`). **Critical: `config/` is NOT in `phpstan.neon` paths** (only `app`/`database`/`routes`/`tests`) ‚Äî a config file that references app code (a typo'd method, wrong arity) is invisible to PHPStan, so its ONLY static-correctness guard is a tiny **Feature** config-pin test (`expect(config('x.supported'))->toBe(Enum::values())`; Feature, because Pest boots the Laravel TestCase only `->in('Feature')` so `config()`/the container resolve ‚Äî no `RefreshDatabase`, no DB). Referencing an app class from config is `config:cache`-safe: the static call runs at cache-build time and returns a plain array `var_export` serialises. Two validation surfaces on the enum: `isSupported(string): bool` (predicate) + `assertSupported(string): self` (fail-closed factory, mirrors `Currency::of()` body ‚Äî exact match, no case-folding, throws `InvalidArgumentException` naming the set).
- **Throwaway-table Feature test** (the DB-touching test shape ‚Äî `MoneyCast` 1.5; reuse for `TranslatableTextCast` 2.3, Pennant 3.x): `uses(RefreshDatabase::class)`; build the fixture table in `beforeEach` with `Schema::dropIfExists(...)` + `Schema::create(...)` (DDL inside the per-test transaction is rolled back on SQLite + PG; the dropIfExists is a belt-and-braces guard); a **named** fixture `Model` at the file bottom (anonymous classes can't carry `@property`) with `protected $table`/`$guarded=[]`/`casts()` mirroring `AuditRecord`. **Type the PK `@property int $id`** and pass `$model->id` (an `int`) to `findOrFail()` ‚Äî passing a `mixed` key (e.g. `getKey()`) makes Larastan return the `Collection<Model>|Model` union and every `->prop` access becomes `property.notFound`. Assert raw columns with **`toEqual`** (loose): an integer column reads back `int` on SQLite but `numeric-string` on PG, so `toBe(1999)` is non-portable; the VO round-trip (`equals()`) proves type fidelity exactly. Narrow `?Vo` from a reload with `expect(...)->toBeInstanceOf(...)` (runtime) + `assert($x instanceof Vo)` (PHPStan), same as the reflection tests. A pure-logic branch of the cast (a corrupt-column throw) can be tested by calling `(new Cast)->get($model, $key, null, [...corrupt...])` directly ‚Äî no DB, engine-independent.
- **`lang/` scaffolding ‚Äî PHP-array group files + native per-key fallback** (2.2; the i18n static-copy mechanism, distinct from `TranslatableText` 2.3 which is per-row content): static UI copy lives in **`lang/{locale}/{group}.php`** returning an array, referenced with **dotted keys** `__('group.key')` ‚Äî NOT JSON (`lang/{locale}.json`, keyed by the English sentence). Why PHP-array group files: (1) keys are stable identifiers (`welcome.headline`), decoupled from the English copy, so editing wording doesn't churn the other five locales' diffs or break lookups (JSON's key IS the English string); (2) **native per-key fallback** ‚Äî `Translator::get()` resolves the chain `[active-locale, fallback]` (verified: `Translator::localeArray()` = `array_filter([$locale, $this->fallback])`), so a key absent in `it` is served from `en` per key (DEC-127 final fallback) and partial coverage is first-class (AC-0-XM-4); (3) matches the framework's own `lang/{locale}/{auth,validation}.php`. **Author `en` complete** (it's the fallback baseline); the other five MAY stagger. **Test shape (Feature, no DB):** drive the per-locale filesystem assertion off `SupportedLocale::values()` (`is_dir(lang_path($locale))` + `file_exists(lang_path("{$locale}/<group>.php"))`) so a missing resource can't drift from the registry; pin per-locale resolution with `App::setLocale('it')` + `__('group.present_key')` differing from the `en` value; pin fallback **non-vacuously** with `Lang::has('group.en_only_key', 'it', false)` (3rd arg `$fallback=false` ‚Üí checks `it` ONLY; `has()` is throw-safe ‚Äî it disables missing-key handling internally) asserted `false`, then the resolved value `===` the `en` value. `trans('group', [], 'en')` returns the whole array but is typed `string|array` ‚Üí `assert(is_array($x))` before `array_keys()` (PHPStan max analyses tests/). Document the convention choice in `docs/` (task requires it; 5.1 references it). `lang_path()` global helper exists in Laravel 13.
- **Single-JSON-column Eloquent cast + `JsonSerializable` VO owning its round-trip** (the `TranslatableTextCast`/`TranslatableText` template 2.3 ‚Äî the ONE-column sibling of *Two-column Eloquent cast*; reuse for any VO persisted to a single JSON column): the VO implements `JsonSerializable` so `json_encode($vo)` emits the FLAT shape (`jsonSerialize()` returns the `array<string,string>` ‚Äî **without it `json_encode` emits the nested `{"translations":{‚Ä¶}}`, the wrong on-disk shape**) AND owns a static `fromJson(string): self` that decodes + narrows (`is_array($decoded)`, then per-entry `is_string($key) && is_string($value)`, throwing on either) + delegates to `of()`. Housing the `mixed`‚Üítyped narrowing in `fromJson()` keeps `of()`'s param clean `array<string,string>` (app call-sites stay typed) and makes the cast a thin adapter (DRY ‚Äî both the cast and tests round-trip through the same `fromJson`/`jsonSerialize` pair). **Cast plumbing (verified Laravel 13 `HasAttributes`):** for ONE column, `set()` returns the serialised **scalar** (or `null`) directly ‚Äî `normalizeCastClassResponse` (line 2031) wraps a non-array as `[$key => $value]`; `get()` reads the raw column from its **`$value` param** (the call sites at 774/907 pass the raw attribute as `$value`), NOT `$attributes[$key]` (that two-column read was MoneyCast-specific). `@implements CastsAttributes<Vo, mixed>` (TSet=mixed ‚Üí the `! $value instanceof Vo` guard is load-bearing; same as MoneyCast). **`json_encode($x, JSON_THROW_ON_ERROR)` narrows to `string` under PHPStan max** (the JsonThrow return-type extension) ‚Üí `set()` returns it as `?string` with no `assert`; pair with `JSON_UNESCAPED_UNICODE` for readable non-Latin on disk (escaping is round-trip-lossless either way). **DB:** `$table->json()` = SQLite text / PG `json` (order-preserving, NOT jsonb) ‚Üí the JSON string round-trips on both lanes; assert the decoded structure with `toEqual` (order-tolerant) + per-key `resolve()`/value checks for exactness (mirrors the *Throwaway-table Feature test* shape ‚Äî corrupt-column throw tested by calling `(new Cast)->get($m, $key, <corrupt>, [])` directly, engine-independent).
- **Pure-primitive resolution (reads no app/request state)** (`TranslatableText::resolve` 2.3): a VO that resolves a locale stays a PURE primitive ‚Äî given a locale it applies per-attribute fallback (`$map[$locale] ?? $map[$fallback] ?? null`), and a **null** input resolves to the registry fallback DIRECTLY, never `App::getLocale()`. Wiring the active request locale (the DEC-127 `Accept-Language`‚Üícookie‚ÜíEnglish pipeline) is the DEFERRED Module-B/frontend job; keeping it out means the VO reads no request/app state ‚Üí it is fully Unit-testable (Pest binds the Laravel `TestCase` only `->in('Feature')`, so a Unit VO test cannot boot facades ‚Äî `App::getLocale()` would have forced a Feature test or a split). READ-side is lenient (an unknown requested locale just falls back ‚Äî a stray request locale must not crash a render); WRITE-side (`of()`) is fail-closed (`SupportedLocale::assertSupported()` every key ‚Äî DEC-064 application-layer validation). Returns `null` (not an exception) when neither the requested locale nor the fallback is present ‚Äî partial coverage is first-class (AC-0-XM-4).
