## Codebase Patterns
(consolidated reusable patterns тАФ read first each iteration)

- **Fixed-set value type тЖТ string-backed enum.** When a VO's domain is a closed set whose members carry derived data, realise it as a `string`-backed enum (the cases ARE the single-source-of-truth set; value-equality and `from`/`tryFrom` are native). Per-case data via an **exhaustive `match`** (no `default` тЖТ PHPStan-max-clean, and a new case fails loudly until handled). Used by `Currency` (codeтЖТ`minorUnitExponent()`); reuse for `SupportedLocale` (2.1).
- **Fail-closed VO factory.** `public static function of(string $x): self { return self::tryFrom($x) ?? throw new InvalidArgumentException("тАжnames the valid set + the one-line fixтАж"); }` тАФ the PHP 8 throw-expression keeps it a one-liner. Exact match, no case-folding (a non-canonical input is a caller bug тЖТ reject). Money/FX/locale VOs all follow this.
- **No `declare(strict_types=1)`** anywhere in `app/Platform` тАФ match the house style (ActorRole, DeliveryStatus, NotInTransactionException all omit it). Pint enforces the rest.
- **Exception choice:** SPL `InvalidArgumentException` for "bad argument to a VO factory" with an intentful message; custom `Runtime`/SPL subclass (cf. `NotInTransactionException`) only when a named domain guard recurs across call-sites.
- **Verbatim set pin** (mirror `tests/Unit/Platform/EnumsTest.php`): loop `$values[$case->name] = $case->value;` then `expect($values)->toBe([...])` тАФ order-sensitive, so any case/value drift fails first. Use for every closed enum set.
- **Test placement:** pure VO/enum tests тЖТ `tests/Unit/Platform/<Sub>/тАжTest.php` (Pest binds the Laravel `TestCase` only `->in('Feature')`, so Unit tests need no app/DB тАФ fine for pure logic). DB-touching tests (casts: `MoneyCast` 1.5, `TranslatableTextCast` 2.3, Pennant 3.x) live under `tests/Feature/Platform/тАж` and opt in per-file with `uses(RefreshDatabase::class)` (Pest.php leaves `RefreshDatabase` commented out globally).
- **Immutable composite VO** (the `Money` template тАФ reuse for `FxRate` 1.3, `DualCurrencyAmount` 1.4): `private function __construct(public readonly тАж )` (promoted readonly props are the public read surface тАФ the cast/consumers read `$m->minorUnits`, no accessor boilerplate) + a static `of(тАж)` factory as the SOLE construction path. Derivations/arithmetic return a **new** instance (never mutate); `equals(self)` is value equality (`===` works directly on enum legs тАФ singletons); `toPayload()` returns the serialise shape with an `@return array{тАж}` shape annotation (PHPStan-max needs it so callers index keys typed). **Rehydrate by composing the public factories** тАФ `Money::of($p['minor_units'], Currency::of($p['currency']))` тАФ NOT a dedicated `fromPayload()` (would be dead code; the cast reads two columns, the recorder never rehydrates). Preconditions (same-currency, EUR-leg) are private `assert*()` guards throwing SPL `InvalidArgumentException` with an intentful message (mirrors `Currency::of()`). No `final` (house style), no `declare(strict_types=1)`.
- **Structural-invariant test via reflection** (the non-vacuous "no float path" proof): `new ReflectionMethod(Cls::class, 'of')->getParameters()[0]->getType()` тЖТ `expect(...)->toBeInstanceOf(ReflectionNamedType::class)` for the runtime guard, then `assert($type instanceof ReflectionNamedType)` to narrow `ReflectionType|null` for PHPStan-max, then `expect($type->getName())->toBe('int')->and($type->allowsNull())->toBeFalse()` (use `'string'` for FxRate). Also assert the constructor `isPrivate()` (so `of()` is the only path). `ReflectionMethod`/`ReflectionNamedType`/`InvalidArgumentException` are global тАФ no `use` in a no-namespace Pest file. **`tests/` IS under PHPStan max** (`phpstan.neon` paths include `tests`) тАФ reflection narrowing is mandatory.
- **String-validated scalar VO** (the `FxRate` variant of *Immutable composite VO* тАФ for a VO wrapping ONE scalar string that must round-trip bit-for-bit): private ctor + a single `public readonly string $value` + sole `of(string)` factory + `__toString()` returning `$value` (the `(string)` cast is the ergonomic read the test pins; `$value` is the structured read surface consumers index). Validate in `of()` against a class-const regex anchored `\AтАж\z` (**NOT** `^тАж$` тАФ `$` leniently matches before a trailing `\n`, so `"1.0842\n"` would wrongly pass) and test `preg_match($pat, $s) !== 1` (one comparison covers both the `0` no-match and the `false` error returns тАФ PHPStan-max clean); on failure throw SPL `InvalidArgumentException` with an intentful message naming the accepted grammar. **Store the accepted string verbatim тАФ never normalise** (no trim, no reformat, no canonicalising trailing zeros): that IS the bit-for-bit / no-precision-drift guarantee (invariant 5, refunds at the locked rate). Well-formedness only тАФ economic/semantic validity (positivity, bounds) stays OUT as downstream policy. FxRate grammar = `/\A\d+(\.\d+)?\z/` (non-negative plain decimal; no sign, exponent or whitespace).
- **Policy-free composite VO + purity guard** (the `DualCurrencyAmount` extension of *Immutable composite VO* тАФ for a bundle that sits next to a module boundary and must carry data only, never policy): (1) **build the payload from the parts' read surfaces, not a composed VO's `toPayload()`** тАФ the keys differ (`DualCurrencyAmount` emits `amount`/`eur_equivalent_amount` from `$money->minorUnits`; `Money::toPayload()` emits `minor_units`), so compose the shape by hand and pin it with an exact `@return array{тАж}` + an order-sensitive `->toBe([тАж])` test (DEC-169 key order). (2) **An immutable date/time leg is typed `DateTimeImmutable`** (a `Carbon\CarbonImmutable` literal satisfies it; a *mutable* `Carbon`/`DateTime` is a TypeError тАФ correct, it would break the immutable bundle) and serialised in **full ISO 8601** via `->format(DateTimeImmutable::ATOM)` тАФ do **NOT** truncate to `Y-m-d`: truncation silently imposes a snapshot-granularity policy the VO must not own (design D3 landmine). Mirror the reflection guard (param `[n]` type is `'DateTimeImmutable'`, non-nullable) to make immutability non-vacuous. (3) **Prove "carries no policy" structurally:** `$m = get_class_methods($vo::class); sort($m); expect($m)->toBe([...allow-list...])` тАФ from a test's (external) scope `get_class_methods` returns only PUBLIC methods (verified), so a private ctor/`assert*` guard is excluded and the allow-list stays the public surface (`['of','toPayload']`); any `deriveRate()/convert()/snapshot()/refresh()` would break it. A construction precondition (EUR-leg) is a private instance `assert*()` called **from the ctor body** (runs after promoted-prop binding) тЖТ covers every construction path, stays off the public allow-list.
- **Two-column Eloquent cast** (the `MoneyCast` template тАФ reuse for `TranslatableTextCast` 2.3): a `CastsAttributes` mapping a VO across N real columns derived **by convention from the cast key** (`'price' => MoneyCast::class` тЗТ `price_minor` + `price_currency`; suffix the key, no per-column wiring). Verified vendor contract (Laravel 13 `HasAttributes`): `set()` returning an **array** `[col => value]` is spread into the model's attributes (`normalizeCastClassResponse` тЖТ `array_replace`, lines 2029/1269) тАФ the multi-column write; the cast-key attribute itself is never persisted. `get()` receives the **full raw `$attributes`** array and reads its real columns from there (`$attributes["{$key}_minor"]`), so reload-from-DB (not the just-set model тАФ its `classCastCache` holds the live VO) is what exercises `get()`. **Generic: `@implements CastsAttributes<TVo, mixed>`** тАФ `TGet=TVo` so the owning model's property surfaces typed (`?TVo`), but **`TSet=mixed`** because Eloquent passes through whatever was assigned (it does not enforce the generic), so the `! $value instanceof TVo` runtime guard is load-bearing; typing `TSet=TVo` makes PHPStan flag that guard `instanceof.alwaysTrue`. Fail-closed on a non-VO `set()` and on a corrupt `get()` column (mirrors `Currency::of()`). **Reading `mixed` `$attributes` at PHPStan max:** `(int) $mixed`/`(string) $mixed` are forbidden (`cast.int`/`cast.string`) тАФ narrow with `is_numeric`/`is_string` first (their true-branch types cast cleanly), throwing on the else (corrupt money data must not be silently coerced тАФ invariant 6).
- **Throwaway-table Feature test** (the DB-touching test shape тАФ `MoneyCast` 1.5; reuse for `TranslatableTextCast` 2.3, Pennant 3.x): `uses(RefreshDatabase::class)`; build the fixture table in `beforeEach` with `Schema::dropIfExists(...)` + `Schema::create(...)` (DDL inside the per-test transaction is rolled back on SQLite + PG; the dropIfExists is a belt-and-braces guard); a **named** fixture `Model` at the file bottom (anonymous classes can't carry `@property`) with `protected $table`/`$guarded=[]`/`casts()` mirroring `AuditRecord`. **Type the PK `@property int $id`** and pass `$model->id` (an `int`) to `findOrFail()` тАФ passing a `mixed` key (e.g. `getKey()`) makes Larastan return the `Collection<Model>|Model` union and every `->prop` access becomes `property.notFound`. Assert raw columns with **`toEqual`** (loose): an integer column reads back `int` on SQLite but `numeric-string` on PG, so `toBe(1999)` is non-portable; the VO round-trip (`equals()`) proves type fidelity exactly. Narrow `?Vo` from a reload with `expect(...)->toBeInstanceOf(...)` (runtime) + `assert($x instanceof Vo)` (PHPStan), same as the reflection tests. A pure-logic branch of the cast (a corrupt-column throw) can be tested by calling `(new Cast)->get($model, $key, null, [...corrupt...])` directly тАФ no DB, engine-independent.
